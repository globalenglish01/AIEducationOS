LLM 并不是简单地把见过的句子背下来再复读。

这个模型能够捕捉 Token 之间复杂的上下文关系。

因此它经常能够生成自己从未见过的句子组合。

这也是为什么：

它看起来像是在“思考”。

实际上是在进行“统计泛化”。

为什么会出现“涌现能力”？

随着模型规模扩大，

某些复杂任务（例如代码生成、多步推理、跨语言迁移等）的表现，

有时会出现比简单线性外推更快的提升。

这种现象通常被称为：

Emergent Ability（涌现能力）

例如：

小模型：

1B参数

可能只能完成基础补全任务。

更大的模型：

100B+

可能在代码生成、翻译、总结等任务上表现出显著提升。

需要注意的是：

目前学术界对于“涌现能力”的成因仍存在讨论。

部分研究认为：

这可能既与模型规模增长有关，

也可能与评测指标本身的非线性特征有关。

因此：

“涌现”更适合作为一种经验观察现象，

而不是已经完全解释清楚的理论结论。

Part 6：动手 Demo（可运行代码）

下面用一个真实的小型语言模型体验 LLM 的生成过程。

安装依赖：

Bash

pip install transformers torch

示例代码：

Python

from transformers import pipeline, set_seed

generator = pipeline(
    "text-generation",
    model="gpt2"
)

set_seed(42)

result = generator(
    "今天天气",
    max_length=20,
    do_sample=True,
    temperature=0.7
)

print(result[0]["generated_text"])

关键代码解释

Python

pipeline(
    "text-generation",
    model="gpt2"
)

加载一个真实语言模型。

Python

do_sample=True

开启概率采样。

如果关闭采样，

模型会更倾向于选择当前概率最高的 Token。

Python

temperature=0.7

控制随机度。

Temperature 越高：

探索越强。

Temperature 越低：

输出越稳定。

真实模型内部发生了什么？

输入文本

↓

Tokenization

↓

Transformer计算

↓

得到所有Token的概率(Logits)

↓

Softmax

↓

概率分布

↓

Sampling

↓

输出Token

其中：

Logits 可以理解为模型对所有候选 Token 的“原始偏好分数”。

Softmax 会把这些分数转换成概率。

Sampling 再根据这些概率选出下一个 Token。

运行结果

可能得到：

今天天气很好，适合出门散步。

也可能得到：

今天天气不错，但晚上可能会下雨。

结果可能不同。

因为：

LLM 本来就是概率模型。

Part 14：本章小结

LLM 看起来像在思考，本质上是在做逐 Token 的概率预测。

AI 工程的关键不是寻找“最聪明的模型”，而是学会为模型提供正确的上下文、事实来源和评估机制。

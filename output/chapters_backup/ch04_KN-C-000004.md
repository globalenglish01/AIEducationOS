Part 5：技术本质深讲

幻觉到底是什么

很多人把幻觉理解成：

“模型答错了。”

这太浅了。

更准确的定义：

模型生成了事实上错误、不存在或与上下文矛盾的内容，并以高置信度呈现。

例如：

不存在的论文

不存在的API

不存在的人名

不存在的数据

都属于幻觉。

幻觉为什么必然存在

回忆第1章：

LLM 的核心任务：

预测下一个Token

而不是：

验证事实

模型优化目标：

maximize P(next token)

而不是：

maximize factual correctness

这就是根本矛盾。

原理图

          用户问题
               │
               ▼
      ┌────────────────┐
      │  LLM概率预测器 │
      └────────────────┘
               │
     ┌─────────┼─────────┐
     ▼         ▼         ▼

正确答案   错误答案   编造答案

     ▲         ▲         ▲
     │         │         │

都可能具有高概率

注意：

模型只关心：

哪个词更像人类会说的话

而不关心：

哪个词更符合现实世界

幻觉来源一：训练数据噪音

训练数据来自互联网。

互联网并不是真理。

里面包含：

错误文章

谣言

过期信息

虚构内容

模型学到的知识天然带噪声。

幻觉来源二：知识截止

假设模型训练截止于：

2025年6月

你问：

2026年世界杯冠军是谁？

模型没有数据。

理论上应该回答：

不知道

但实际上经常会：

猜一个最合理的答案

这就是知识截止幻觉。

幻觉来源三：概率补全机制

最核心的一种。

假设问题：

请列出某篇冷门论文的第三作者

模型记忆模糊。

此时仍然要继续生成 Token。

于是出现：

合理猜测
↓
概率补全
↓
虚构细节

最终形成完整答案。

三种典型幻觉

1. 事实幻觉

编造不存在的信息。

例如：

不存在的论文

不存在的公司

不存在的法律条文

最常见。

2. 指令幻觉

忽略 Prompt 要求。

Prompt：

只回答YES或NO

输出：

YES。

这里我详细解释一下……

模型按习惯输出。

不是按要求输出。

3. 上下文幻觉

最常见于 RAG。

文档：

公司成立于2018年

模型回答：

根据资料，公司成立于2016年

明明上下文给了正确答案。

模型却使用自己的记忆。

幻觉的第四种：推理幻觉

这一类经常被忽略。

模型引用的事实可能都是正确的。

但推理过程出了问题。

例如：

A销量高于B
B利润高于C

模型得出：

A利润高于C

这个结论并不成立。

因为：

销量和利润不是同一个维度。

这里的问题不是知识缺失。

而是逻辑推导错误。

这类错误通常被称为：

Reasoning Hallucination
（推理幻觉）

也是为什么：

RAG ≠ 解决所有幻觉

RAG 能补知识。

但不能保证推理正确。

一个必须记住的公式

流畅度 ≠ 准确度

或者：

Confidence ≠ Correctness

这是 AI 工程师必须建立的认知。

Part 6：动手 Demo（可运行代码）

下面用一个简单程序模拟“概率补全导致幻觉”。

Python

import random

# 模拟模型知识库
knowledge = {
    "Python作者": "Guido van Rossum",
    "Linux作者": "Linus Torvalds"
}

def fake_llm(question):
    """
    模拟一个简单LLM
    """
    if question in knowledge:
        return knowledge[question]

    # 不知道答案时开始“脑补”
    hallucinations = [
        "John Smith",
        "David Brown",
        "Michael Johnson",
        "Robert Wilson"
    ]

    return random.choice(hallucinations)

questions = [
    "Python作者",
    "Linux作者",
    "某篇不存在论文的第三作者"
]

for q in questions:
    print(f"问题: {q}")
    print(f"回答: {fake_llm(q)}")
    print("-" * 30)

关键代码解释

Python

if question in knowledge:

知识库里有答案。

直接返回。

Python

return random.choice(hallucinations)

知识库没有答案。

不返回“不知道”。

而是生成一个看起来合理的人名。

这就是幻觉的核心机制。

运行结果示例

问题: Python作者
回答: Guido van Rossum

问题: Linux作者
回答: Linus Torvalds

问题: 某篇不存在论文的第三作者
回答: David Brown

注意：

这里的问题本身不存在真实答案。

因此程序只能依靠概率补全生成一个“看起来合理”的名字。

真实 LLM 复杂一百万倍。

Part 14：本章小结

LLM 的目标不是寻找真相，而是生成统计上最合理的文本，因此幻觉是其结构性特征。

模型的自信程度与事实准确度没有必然关系，越自然流畅的回答，越需要验证来源。

从 L0 到 L1，你需要完成一个关键认知转变：

相信模型
      ↓
理解模型
      ↓
管理模型

这将决定你未来能否构建可靠的 AI 系统。

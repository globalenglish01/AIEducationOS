Part 6：动手 Demo（可运行代码）

下面实现一个最小版 Agent Loop。

Python

from dataclasses import dataclass, field

@dataclass
class AgentState:
    task: str
    history: list = field(default_factory=list)
    iteration: int = 0

def search_tool(keyword):
    return f"搜索结果：{keyword} 是一种 Agent 技术"

def fake_llm(state):
    """
    模拟 LLM 决策
    """

    if state.iteration == 0:
        return {
            "thought": "需要先搜索 Agent Loop",
            "action": "search",
            "action_input": "Agent Loop"
        }

    return {
        "thought": "信息已经足够，可以结束",
        "action": "finish",
        "action_input": None,
        "result": "Agent Loop 是驱动 Agent 多轮执行的核心机制。"
    }

def run_agent(task, max_iterations=5):

    state = AgentState(task=task)

    while state.iteration < max_iterations:

        decision = fake_llm(state)

        state.history.append({
            "thought": decision["thought"],
            "action": decision["action"]
        })

        if decision["action"] == "finish":
            return decision["result"]

        observation = search_tool(
            decision["action_input"]
        )

        state.history.append({
            "observation": observation
        })

        state.iteration += 1

    raise RuntimeError("达到最大迭代次数")

if __name__ == "__main__":

    result = run_agent(
        "解释 Agent Loop"
    )

    print(result)

关键代码解释

初始化状态：

Python

进入循环：

Python

模拟 LLM 决策：

Python

执行工具：

Python

    decision["action_input"]
)

达到完成条件：

Python

立即退出。

运行结果

Agent Loop 是驱动 Agent 多轮执行的核心机制。

整个执行过程：

 ↓

 ↓
Observation
 ↓

Part 8：这里容易踩坑

坑1：无限循环

真实案例：

Part 9：面试怎么答

L1：Agent Loop 的终止条件有哪些？为什么要同时设置多个？

回答框架

终止条件至少包括：

回答框架

不要简单截断。

Part 11：必背金句

[循环原则]：Agent 的能力来自循环，而不是单次调用。

[终止原则]：没有终止条件的 Agent，本质上是失控程序。

[状态原则]：AgentState 是 Agent 的记忆载体。

[审计原则]：历史记录只追加，不覆盖。

[生产原则]：计数器在手，Checkpoint 兜底，截断不能有。

[成本原则]：没有 max_iterations 的 Agent Loop，极有可能在生产环境中引发 Token 爆炸。

Part 14：本章小结

复杂任务无法依靠一次 LLM 调用完成，必须通过 Agent Loop 持续执行 Thought → Action → Observation 的迭代过程。

生产级 Agent Loop 不只是一个 while 循环，还必须具备 max_iterations、timeout、连续失败检测以及 Checkpoint 持久化能力。

从 L0 的 Prompt、L1 的 ReAct，到本章的 Agent Loop，你已经开始从“让模型回答问题”走向“让系统自主完成任务”。

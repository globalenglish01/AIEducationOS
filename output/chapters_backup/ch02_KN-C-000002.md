Part 6：动手Demo（可运行代码）

下面写一个最简单的 Token 估算器。

注意：这个示例仅用于理解 Token 与成本概念，并不能替代真实 Tokenizer。生产环境必须使用 tiktoken 或模型官方 Tokenizer。

Python

import re

def estimate_tokens(text):
    """
    粗略估算Token数量

    中文：
        约 1.5 Token/字

    英文：
        约 1 Token / 4字符

    注意：
        仅用于教学演示
        不可用于生产计费
    """

    chinese_chars = len(
        re.findall(r'[\u4e00-\u9fff]', text)
    )

    english_chars = len(
        re.findall(r'[A-Za-z]', text)
    )

    chinese_tokens = chinese_chars * 1.5

    english_tokens = english_chars / 4

    total = (
        chinese_tokens
        + english_tokens
    )

    return round(total)

def estimate_cost(
    input_tokens,
    output_tokens,
    input_price,
    output_price
):
    """
    价格单位：
    元 / 千Token

    示例价格仅用于演示
    实际以模型官方价格为准
    """

    return (
        input_tokens * input_price
        + output_tokens * output_price
    ) / 1000

text = """
Token 是 LLM 的基础计量单位。
Token is the basic unit of LLMs.
"""

tokens = estimate_tokens(text)

print("估算Token:", tokens)

cost = estimate_cost(
    input_tokens=tokens,
    output_tokens=300,
    input_price=0.008,
    output_price=0.016
)

print("预计成本(元):", round(cost, 4))

Part 14：本章小结

Token 是 LLM 世界里的统一计量单位，模型处理、上下文容量和 API 成本都建立在 Token 之上，而不是建立在字数或单词数之上。

理解 Token 的关键不是记住“中文约1.5 Token/字、英文约1.3 Token/词”，而是建立一个正确认知：模型看到的是 Token 序列，不是人类看到的文字。

从 L0 到 L1 的成长过程，本质上就是从“会调用模型”升级为“理解模型如何计费、如何计算、如何消耗资源”。

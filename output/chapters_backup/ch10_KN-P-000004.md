Part 6：动手 Demo（可运行代码）

下面实现一个极简 ReAct Agent。

功能：

用户问：
上海天气怎么样？

Agent：
解析城市
决定调用天气工具
获取结果
检查是否完成
返回答案

Python

import re
from typing import Dict

def weather_tool(city: str) -> Dict:

    weather_data = {
        "北京": "31℃ 晴天",
        "上海": "29℃ 多云",
        "东京": "27℃ 小雨"
    }

    return {
        "success": city in weather_data,
        "result": weather_data.get(city),
        "error": None if city in weather_data else "未知城市"
    }

def extract_city(question: str):

    for city in ["北京", "上海", "东京"]:
        if city in question:
            return city

    return None

def react_agent(question: str):

    state = {
        "question": question,
        "observation": None
    }

    max_iterations = 5

    for step in range(max_iterations):

        print(f"\n=== Step {step+1} ===")

        # Thought
        if state["observation"] is None:

            thought = "先识别用户要查询的城市"

            print("Thought:", thought)

            city = extract_city(question)

            action = {
                "tool": "weather_tool",
                "city": city
            }

            print("Action:", action)

            observation = weather_tool(city)

            print("Observation:", observation)

            state["observation"] = observation

        else:

            thought = "根据工具结果判断任务是否完成"

            print("Thought:", thought)

            if state["observation"]["success"]:

                print("\nFinal Answer:")
                print(state["observation"]["result"])

                return

            else:

                print("\nFinal Answer:")
                print("无法获取天气信息")

                return

    print("达到最大迭代次数")

关键代码解析

限制最大步数

Python

max_iterations = 5

避免死循环。

Thought阶段

Python

thought = "先识别用户要查询的城市"

决定下一步行为。

Action阶段

Python

action = {
    "tool": "weather_tool",
    "city": city
}

选择工具。

Observation阶段

Python

observation = weather_tool(city)

获取结果。

Observation写回状态

Python

形成闭环。

第二轮反思

Python

不是拿到结果立刻返回。

而是先观察，再决策。

这才符合 ReAct 的循环思想。

运行结果

Action:

Observation:

=== Step 2 ===

Part 8：这里容易踩坑

坑1：不设置 max_iterations

错误写法：

Python

while True:

    thought = llm()
    action = tool()

    state.append(action)

后果：

无限循环
无限Token
无限时间

正确写法：

Python

MAX_ITERATIONS = 15

for step in range(MAX_ITERATIONS):

    run_one_step()

raise Exception("Agent exceeded limit")

坑2：Observation 不写回上下文

错误代码：

Python

result = tool.execute()

# 没更新state

模型下一轮看不到结果。

后果：

不断重复同样动作

正确代码：

Python

result = tool.execute()

state["observation"] = result

形成闭环：

Action
 ↓
Observation
 ↓
State
 ↓
Thought

坑3：工具报错只记日志

错误写法：

Python

try:
    tool.execute()

except Exception:
    logger.error("failed")

LLM完全不知道失败原因。

正确写法：

Python

try:
    result = tool.execute()

except Exception as e:

    result = {
        "success": False,
        "error": str(e)
    }

写入：

Observation:
数据库连接失败

Agent 才能改变策略。

为什么容易犯？

因为传统开发者习惯：

日志也要给模型看

Part 11：必背金句

[先思考再行动]：每个 Action 都必须有 Thought 解释原因。

[观察驱动决策]：Observation 是下一轮 Thought 的输入。

[没有限步必死循环]：max_iterations 是 Agent 的安全带。

[工具结果必须可见]：Observation 不进上下文等于没执行。

[Agent本质是循环]：ReAct 不是一次回答，而是一系列决策。

**林雪的工作日记 · 第六个月第三周**
*（陈博面试后的一个下午）*

---

今天下午重新把测试平台的架构图画了一遍，这次是在纸上，不是白板。

画了大概一小时，改了三稿。

第一稿：画的是"功能结构"——四个Agent、Supervisor、中间件、存储层。整齐，但是平的。

第二稿：按"执行时间线"画——一个请求从进来到出去，经过哪些组件，在哪里停顿，在哪里决策。这张图有了动态感，但太长，很难在白板上十分钟内画完。

第三稿：混合。静态部分用方框表示组件，在关键决策点上标注"ConditionalEdge"，用虚线框表示中间件"包裹"的范围，用双横线表示"检查点持久化"发生的位置。

第三稿我觉得最好——它不只是"有什么组件"，也暗示了"数据怎么流、控制权怎么转移、什么地方可能失败"。

但我也意识到，第三稿是"三小时思考后"的结果，不是"白板前十分钟"能自然画出来的。

---

有一件事我后来想了很久：

面试里有两种"不知道"：

第一种：确实不知道这个概念，完全空白。

第二种：知道这个概念，但没有想清楚它在这个具体系统里怎么表示、怎么运转。

陈博问"中间件在架构图里怎么画"，我遇到的是第二种——我知道中间件是什么，知道它的作用，但没有想清楚它在架构图层面的视觉表示。

第二种"不知道"，是可以在面试里当场推导的。条件是：你要说出来你在想什么，让面试官看到你的推理过程，而不是沉默。

沉默对第一种不知道也是无效的。

所以规则是：无论哪种不知道，都要开口。
- 真的不知道：说"我不知道这个，但我的理解是……"
- 没想清楚：说"我知道这个，但让我在白板上边想边画……"

这两种回答，都比沉默或者假装知道要好。

---

*（注：林雪后来在某次内部分享里，把这个"两种不知道"的框架讲给了她后来公司的同事听。她说，对她自己来说，能把"没想清楚"和"完全不知道"区分开来，是她在面试准备过程里最大的认知改变之一。）*

### 第8章预告

**第8章：中间件栈——成本治理、预算控制与防护机制**

面试官刘工把林雪的简历放在一边，直接问：

"你说成本从二十五美元一天大幅下降，'大幅'是多少？什么导致了这个成本？你审查的时候是怎么发现的？"

三个连续问题，没有给回答第一个的时间。

林雪知道这个问题的难处不在于数字——"大幅"这个词本身就是她主动留下的缓冲，因为她没有精确的成本对比数字。

难处在于：面试官注意到了"大幅"这个模糊表述，在追问。

这是一个测试候选人如何处理"模糊数字被追问"的标准面试陷阱——正确的应对，不是补一个数字，而是解释清楚"大幅"背后的工程逻辑。

下一章，深入成本治理的细节：$25/天是怎么产生的，五大根因是什么，每个中间件的设计参数是怎么来的，以及面对"给我一个精确数字"的追问时，怎么既诚实又有说服力地回答。

**🤖 AI 相关**

| 术语 | 一句话解释 |
| --- | --- |
| **Agent** | 能自己决策、调用工具的AI“智能体”，像个数字员工。 |
| **LangGraph** | 用来构建多Agent协作流程的框架，像给Agent画流程图。 |
| **State** | 整个系统当前的数据和进度快照。 |
| **Node** | 流程图中的步骤或处理单元。 |
| **ConditionalEdge** | 根据条件决定走哪条分支的动态连线。 |
| **Supervisor模式** | 一个主Agent负责分配任务给多个子Agent的管理模式。 |
| **中间件** | 在请求处理前后插入的通用逻辑层，像安检通道。 |
| **Checkpointer** | 保存Agent执行状态快照，支持中断恢复和回放调试。 |
| **TokenBudgetMiddleware** | 控制Agent总Token消耗上限的中间件，像话费封顶套餐。 |
| **ConsecutiveRetryGuard** | 连续失败两次就主动中断的安全阀。 |
| **MCPCallLimiter** | 限制单个工具调用次数的中间件，防止滥用。 |
| **System Prompt** | 预先给AI设定的角色和规则，像入职培训手册。 |
| **recursion\_limit** | 递归调用的最大深度限制，防止程序无限循环下去。 |
| **ReAct循环** | Agent的思考-行动-观察循环模式，像人的决策反馈闭环。 |
| **Context Window** | AI一次能记住的对话或文本总量上限，像短期记忆力。 |
| **Subgraph** | 嵌套在另一个Graph内部的子图，模块化复用。 |
| **意图分类** | 判断用户请求属于哪一类任务的AI识别过程。 |
| **显式路由** | 根据用户明确指定的标签直接分配任务。 |

| **静默降级** | 功能失效但不报错悄悄变弱，像助手没了却不告诉你。 |

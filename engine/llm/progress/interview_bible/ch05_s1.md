## Section 1：现场写Prompt——那十分钟，她意识到自己从没真正学过这件事

---

第五个月第三周，林雪收到了一家AI独角兽的视频面试邀请。

日历上写的是"技术面，45分钟"。约她的是后端工程师张晓峰，简介里写着"10年NLP/LLM经验，现负责AI原生应用研发平台"。

林雪做了准备。她把之前整理的LLM基础知识又过了一遍：Token、Context Window、幻觉、Temperature。她把两个项目的STAR故事练了四遍，计时到两分钟刚好收尾。她觉得自己准备得还算扎实。

但张晓峰上来没问这些。

他说："林雪你好，我们直接进实战部分吧。"

他共享了屏幕，上面是一份保险合同的节选——大概500字，是一段关于"免责条款"的文本：

> 第十二条 下列情形之一造成被保险人身故的，本公司不承担给付身故保险金的责任：
> （一）被保险人故意自伤、自杀……
> （二）被保险人因犯罪或抗拒依法执行公务……
> （三）被保险人酒后驾驶……

"好，"张晓峰说，"你的任务是：写一个Prompt，让AI从这段文字里提取所有的免责条款，输出一个结构化列表，每条包含条款编号和完整文字。给你用这个工具。"

他把一个带有GPT-4o接入的简单界面分享给了林雪。

"十分钟，你写你的，我先处理一个消息。"

林雪盯着屏幕。她想，这还不简单？

她打了：

```
请从以下保险合同文本中提取所有免责条款，并以列表形式展示：

[粘贴了原文]
```

发送。

模型输出了：

> 以下是合同中的相关内容：
>
> - 第十二条提到了多种不承担保险金的情形
> - 包括自杀、犯罪、酒驾等情况
> - 建议您仔细阅读完整合同

林雪皱眉。这不是她要的——她要的是有编号的结构化列表，而这个输出是摘要。

她重写：

```
你是一个法律条文提取工具。请从以下文本中提取免责条款，用JSON格式输出，字段包括：条款编号（如"(一)"）、条款内容。

[原文]
```

这次模型输出：

```json
{
  "免责条款": [
    {"编号": "(一)", "内容": "被保险人故意自伤、自杀"},
    {"编号": "(二)", "内容": "被保险人因犯罪或抗拒依法执行公务"},
    {"编号": "(三)", "内容": "被保险人酒后驾驶"}
  ]
}
```

好多了——但内容被截断了。原文里每条条款后面都有"……本公司不承担……"的结语，模型把它丢掉了。

林雪还没来得及再改，张晓峰回来了。

"怎么样？"

林雪把两次尝试都展示给他看。

张晓峰看了看，点了点头，但眼神里有什么东西——不是否定，是一种"这正是我想看到的问题"的表情。

"你第一版的Prompt，"他说，"缺四样东西：角色定义、输出格式约束、示例、以及边界说明。你第二版补上了角色和格式，但示例没有，所以模型自己决定要不要保留完整文字——而它选择了摘要。"

林雪点头，没有辩解。

张晓峰说话时，用手指点了两下桌上那份打印出来的JCL示例，停在某一行上，没有继续点。

"你在保险公司做了十五年，"张晓峰说，"你知道免责条款被截断是什么级别的风险吗？"

林雪当然知道。她见过理赔纠纷，见过因为一个限定词引发的法律诉讼。"合同文本的精确性是一类错误，不是二类错误，"她说，"截断原文等于引入了错误信息。"

"对，"张晓峰说，"所以你的Prompt也需要金融级精确性。你之前写COBOL批处理，JCL里的每一行EXEC卡、DD语句，你写错了会怎样？"

"作业会失败，"林雪说，"或者读错数据集。"

"Prompt是AI的JCL，"张晓峰说，"差别是：JCL写错，作业会显式报错。Prompt写错，AI会给你一个听起来合理但其实错误的结果——它不会报错，它会礼貌地帮你错下去。"

林雪沉默了一秒。

这句话她后来记了很久。

---

面试还有三十分钟。

张晓峰没有继续批评林雪的Prompt，而是转向了他真正想问的问题。

"你在测试平台和RAG系统里，Prompt是怎么管理的？你们做了版本控制吗？"

这才是林雪真正要答好的问题。

但在回答之前，她先花了两分钟，把刚才那个断尾的Prompt重写了一遍：

```
你是一个保险合同条文提取专家，具备法律文本精确分析能力。

任务：从以下保险合同文本中，提取所有免责条款的完整原文。

输出格式（严格遵守）：
```json
{
  "免责条款": [
    {
      "编号": "（一）",
      "完整条款": "被保险人故意自伤、自杀，本公司不承担给付身故保险金的责任"
    }
  ]
}
```

注意事项：
1. 必须保留每条条款的完整原文，不得截断或改写
2. 条款编号格式与原文保持一致（中文括号+序号）
3. 若文本中无免责条款，返回空数组

合同文本：
[原文]
```

这次输出：条款编号准确，完整原文一字不差，数组结构正确。

张晓峰看了看，说："好。这才是工程级Prompt。"

---

这件事给林雪留下了一个印象：

她以前觉得"写Prompt"是"跟AI说话"，就像跟一个同事发消息一样，只要说清楚意图就好。

但那十分钟让她看到，这件事远不止于此。

Prompt是一种工程规范。它需要结构，需要约束，需要示例，需要边界条件，需要版本管理——就像JCL，就像接口文档，就像测试用例的断言条件。

区别在于：写错了，它不报错。它会帮你优雅地错下去。

## 📖 本章名词解释（新人必读）

> 第一次看到这些词？别慌，下面一句话搞定。

**🤖 AI 相关**

| 术语 | 一句话解释 |
| --- | --- |
| **Prompt** | 你给AI下达的指令或问题，即“提示词”。 |
| **LLM** | 大语言模型，像ChatGPT那样能理解和生成文字的AI大脑。 |
| **Token** | AI处理文本的最小计量单位，像字数但不等同于字数。 |
| **Context Window** | AI一次能记住的对话或文本总量上限，像短期记忆力。 |
| **幻觉** | AI胡编乱造但语气自信，像一本正经胡说八道。 |
| **Temperature** | 控制AI回答随机性的参数，值越高创意越强、越不按套路。 |
| **RAG** | 让AI先查外部资料再回答，防止它瞎编的技术。 |
| **Agent** | 能自己决策、调用工具的AI“智能体”，像个数字员工。 |
| **结构化输出** | 让AI按固定格式（如JSON）返回数据，方便程序解析。 |
| **角色定义** | 在Prompt里指定AI扮演什么身份，像给演员分配角色。 |
| **示例（few-shot）** | 在Prompt里给AI看几个例子，告诉它“照这个来”。 |
| **边界说明** | 告诉AI什么情况下不要答或答什么，像划清工作范围。 |

**💻 软件工程与编程**

| 术语 | 一句话解释 |
| --- | --- |
| **API** | 程序之间打招呼递数据的方式，类似餐厅服务员帮你点菜。 |
| **JCL** | 大型机上的作业控制语言，用来调度批处理任务。 |
| **批处理** | 定期批量处理大量数据的程序，每月结算用。 |
| **版本控制** | 记录文件每次修改历史，方便回退和协作。 |
| **NLP** | 自然语言处理，让计算机理解和生成人类语言的技术。 |
| **JSON** | 一种轻量级数据格式，像填表格一样规整。 |
| **字段** | 数据记录里的一个具体项目，如“姓名”“年龄”。 |
| **数组** | 一组数据的集合，像一串排好队的数据。 |

**📌 通用缩写**

| 术语 | 一句话解释 |
| --- | --- |
| **COBOL** | 上世纪商用大型机上的老牌编程语言，常用于金融核心系统。 |

## Section 1：周敏的问题——"你是认真的，还是没有选择？"

---

第八个月第七周，线下面试，对方公司的会议室。

周敏进来的时候，桌上放着林雪的简历——是周敏的助理提前打印好的，两张纸，用回形针别着，刚打印，林雪能闻到一点墨水的气味。

周敏在猎头行业做了十年，专注中高级技术岗。她见过各种转型案例——成功的、失败的、中途放弃的。她有一个判断人的框架：候选人是被迫改变的，还是主动选择改变的？两者在三年后的表现完全不同。

她坐下，拿起简历看了大约十秒，然后把它翻了个面，正面朝下压在桌上。然后才开口。

面试开始，周敏没有看技术简历，而是直接开口：

"林雪，我直接问你一个可能有点冒犯的问题。

你在简历上写了45岁，二十年COBOL/VBA经验，保险核心系统。这是非常稀缺的背景——金融行业对COBOL工程师的需求还在，薪资也不低。

你为什么不继续在金融行业做COBOL？

你是真的想转到AI Native这个方向，还是因为公司调整没有了选择，不得不转？"

---

这个问题，林雪在家练习过。

但练习的时候，她意识到：没有"技术答案"可以用。这不是一道关于LLM或Agent的题——是关于她自己的题。

她练习了很多次，最终定稿了一个答案。

但在走进面试室之前，她又修改了一次——因为她意识到，她准备的"答案"太像一个答案了。真实的回应，应该包含她真正经历的那些不确定。

---

她看着周敏，说：

"让我先回答第二个问题：我是被迫的还是主动的。

两者都有。我不打算只说其中一个。

我被裁了。这是事实，不需要绕开。那个周四下午，HR叫我去谈话，告诉我团队要缩编。那一刻，我没有立刻想到'这是转型的机会'——我第一个想到的是我的徒弟王哲，他已经被通知不续签了。

所以，'被迫'的部分是真实的。

但接下来发生的事，不是'被迫'能解释的。

被通知的那天晚上，我在家坐了二十分钟，写了两列：我有的，我没有的。然后我给自己设了一个目标：8个月，在AI方向找到一份薪资不低于现在的工作。

我没有找'COBOL相关的AI岗位'作为退路，也没有找'金融IT外包'作为缓冲——我直接去学Python，去做AI Agent项目，去做RAG系统，去面试AI Native工程师岗位。

这个选择，是主动的。"

---

周敏停顿了一下，然后问了第二个问题：

"你的COBOL经验在AI工程里有价值吗？还是说，那二十年你觉得在面试里是一个负担——解释起来太麻烦？"

这个问题，击中了林雪真实焦虑的地方。

她深呼吸了一下，准备给出她练习了很久的那个真实答案。

## 📖 本章名词解释（新人必读）

> 第一次看到这些词？别慌，下面一句话搞定。

**🤖 AI 相关**

| 术语 | 一句话解释 |
| --- | --- |
| **AI Native** | 以AI为核心技术构建系统的新兴开发方向，区别于传统软件工程。 |
| **AI Agent** | 能自主规划、调用工具完成任务的AI程序。 |
| **RAG系统** | 让AI在回答时先检索外部资料再生成答案的技术，减少胡编。 |
| **LLM** | 大语言模型，如GPT、Claude等能理解和生成文本的AI模型。 |

**💼 金融 / 保险行业相关**

| 术语 | 一句话解释 |
| --- | --- |
| **COBOL** | 上世纪50年代的老语言，至今仍在银行保险核心系统里大量运行。 |
| **VBA** | Excel/Office里的编程语言，常用于金融行业的自动化报表。 |
| **保险核心系统** | 管理保单、理赔等核心业务的关键系统，容错率极低。 |

**🎯 职场 / 转型相关**

| 术语 | 一句话解释 |
| --- | --- |
| **团队缩编** | 公司缩减团队规模，部分员工被裁或调岗。 |
| **不续签** | 合同到期后公司不再续约，员工需离开。 |
| **金融IT外包** | 将保险/银行的系统开发和维护工作交给外部公司做，而非内部团队。 |
| **缓冲岗位** | 转型期间的过渡性工作，非最终目标但能提供收入和时间。 |
| **被动转型** | 因被裁或岗位消失而被迫换方向，而非自己主动选择。 |
| **主动转型** | 自己判断趋势后主动学习新技术、换赛道。 |
| **背景负担** | 履历中有不符合新岗位的内容（如年龄/老技术），被认为拖累求职。 |
| **解释成本** | 需要在面试中花大量篇幅解释自己为什么从A领域转到B领域。 |

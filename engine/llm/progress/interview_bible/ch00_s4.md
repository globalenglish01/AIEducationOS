## Section 4：把自己当面试题——0个月 vs 8个月

---

这一节做一件特别的事：**不是给你知识，而是让你先测试自己现在在哪里。**

下面是林雪在第0个月——那个周四下午——如果被问到这些问题，会怎么回答。以及8个月后，她的回答是什么样的。

你现在能回答到哪一层，就是你的起点。

---

:::qa
**问题1：什么是Token，和传统API调用有什么本质区别？**

> [!danger] 面试官追问
> "你说你用过ChatGPT API，能解释一下Token计费和传统按请求计费有什么不同，以及这个区别对你的系统设计有什么影响吗？"

**第0个月的林雪：**
Token……是那个字数单位吗？就是按字数收费？

**第8个月的林雪：**
"Token不是字数，是语言模型的基本处理单元，一个汉字大约1-2个Token，一个英文单词大约1个Token。

传统API按请求收费，每次调用成本固定；LLM按Token收费，成本随输入+输出长度线性增长。这对系统设计的影响是：你必须管控上下文长度。我们项目里的TokenBudgetMiddleware，设了max_total_input_tokens=200,000，就是为了防止Agent在多轮对话里把历史上下文无限堆积，最后一次API调用吃掉一天的Token预算。"

（来源：A1，已代码核实）
:::

---

:::qa
**问题2：你们的AI测试平台，一天大概烧多少钱，你怎么知道的？**

> [!danger] 面试官追问
> "你说你做了成本优化，能告诉我优化前是多少、优化后是多少、你怎么算的吗？"

**第0个月的林雪：**
……没算过。

**第8个月的林雪：**
"优化前，UI Agent跑一个登录页面的完整测试，日消耗大约25美元——这个数字在代码注释里有记录，是我们审计时发现的。

优化后大幅下降，但我要诚实告诉你：精确的优化后数字我们没有在代码里留强证据，面试时我会说'显著下降'而不是报一个具体百分比。

这5大根因我能讲清楚：用了旧模型/recursion_limit设到了1000导致无限退栈/每步都重截图分析/subagent重复发送完整上下文/没有Token预算控制。我们逐一修了这5个问题。"

（来源：A1、A4，已代码核实；成本优化后数字标注为需进一步验证）
:::

---

:::qa
**问题3：你的RAG系统，如果面试官问召回率，你怎么回答？**

> [!danger] 面试官追问
> "我不是问你实现了什么，我是问你系统的召回率是多少，你用什么方式测量的？"

**第0个月的林雪：**
应该……很高吧？我没测过。

**第8个月的林雪：**
"这是一个我需要诚实告诉你的地方。在8个月前，我们系统没有一套量化评估体系——RAGAS这些工具是有的，但我们没有建领域专属的评估集来跑真实数字。

我后来专门解决了这个问题——用golden_dataset_builder自动生成了一批QA测试对，跑了RAGAS的四个核心指标（Faithfulness/AnswerRelevancy/ContextRecall/ContextPrecision）。

但我要同时告诉你诚实的边界：评估集的质量依赖LLM生成QA对的准确性，还没有经过大规模人工验证。这是下一步要做的工作。"

（来源：B2，已代码核实，含诚实边界）
:::

---

> **面试官内心评分**
>
> 问题1：第0个月的回答0分，第8个月8分（有原理+有项目代入+有具体参数）
> 问题2：第0个月的回答0分，第8个月9分（有诚实说明哪些数字是强证据、哪些不是）
> 问题3：第0个月的回答-1分（面试官最怕"应该很高"），第8个月9.5分（诚实承认缺失+展示已修复+讲清修复后边界）
>
> 第8个月最高分的，不是那道技术最深的题，而是**那道林雪主动说"这里我要诚实告诉你"的题**。

---

这就是这本书想教你的核心：**不是怎么把弱点藏起来，而是怎么把弱点讲成可信度。**

Staff级别的面试官见过太多把短板藏起来的人。他们在等的，是那个能说清楚"我知道哪里有问题，我是怎么处理的，还有什么没做到"的人。

林雪成为了那个人。

这本书帮你也成为那个人。

## 📖 本章名词解释（新人必读）

> 第一次看到这些词？别慌，下面一句话搞定。

**🤖 AI 相关**

| 术语 | 一句话解释 |
| --- | --- |
| **Token** | AI处理文本的最小计量单位，像字数但不等同于字数。 |
| **上下文长度** | AI一次能记住的对话或文本总量上限，像短期记忆力。 |
| **RAG** | 让AI先查外部资料再回答，防止它瞎编的技术。 |
| **召回率** | 检索系统找到的相关结果占所有相关结果的比例。 |
| **RAGAS** | 专门评估RAG系统质量的评测框架工具箱。 |
| **Faithfulness** | 答案是否忠于检索到的资料，没瞎编。 |
| **AnswerRelevancy** | 答案跟问题是否直接相关，没跑题。 |
| **ContextRecall** | 检索到的资料是否覆盖了所有需要的知识点。 |
| **ContextPrecision** | 检索到的资料里有用信息占比高不高。 |

**💻 软件工程与编程**

| 术语 | 一句话解释 |
| --- | --- |
| **API** | 程序之间打招呼递数据的方式，类似餐厅服务员帮你点菜。 |
| **recursion\_limit** | 递归调用的最大深度限制，防止程序无限循环下去。 |
| **subagent** | 被主Agent调用的子任务Agent，像给主管配助手。 |
| **golden\_dataset** | 人工标注好的标准问答对，用于衡量系统表现。 |
| **评估集** | 用来测试系统好坏的一批标准问题及答案集合。 |

## Section 4：追问演练——被打断后怎么办

---

这场模拟面试演示了一个更难的情况：面试官在你讲到一半的时候打断你，要么要求展开，要么换一个方向追问。

如何在被打断后保持清晰、不丢失原本的故事线，是很多工程师没有练过的。

---

:::qa
**面试官：你说你做了全系统LLM调用审计，这是什么意思？**

（在林雪讲故事A到Action部分时被打断）

**弱答案（乱了节奏）**：
"对，就是……我就是看了一下哪里有调用……然后发现了一个地方……"

*面试官内心：讲不清楚，继续听着。*

**强答案（接住打断，简洁展开，然后回到主线）**：
"审计的方法是grep backend目录下所有Python文件里的init_chat_model关键词——因为这是LangChain/LangGraph创建LLM实例的标准入口。理论上所有LLM调用都应该经过这个入口，如果哪里绕过了，就是成本盲区。

我发现ui_workspace.py里有一个向导端点直接调了这个，没有经过中间件栈。这就是我说的盲区。

（回到主线）所以我先修了这个盲区，然后再处理那5个根因……"

关键技巧：接住打断，用30秒回答，然后主动说"回到主线"，把面试官带回你原来的故事。
:::

---

:::qa
**面试官：你说recursion_limit=60，如果一个任务真的需要60步以上呢？**

**弱答案（防御性）**：
"一般任务用不了那么多步的……"

*面试官内心：没有回答问题。*

**强答案（承认局限，展示判断）**：
"这是一个真实的trade-off。

如果确实有任务需要超过60步，当前系统会中断它。这是刻意的设计——我们观察到正常测试流程不超过40步，60是留了50%余量的上限。超过60步的任务，要么是在循环（应该中断），要么是任务本身过于复杂（需要拆分成更小的子任务）。

但你说得对：如果有合理的60步以上的任务被误杀，这是一个问题。当前我们没有'检测到底是循环还是正常长任务'的能力——这是下一步想加的，类似'相邻步骤是否在重复相同的工具调用'的判断。"

（来源：A1，已代码核实）

*面试官内心：知道边界，知道原因，知道下一步——这是资深工程师的思维方式。*
:::

---

:::qa
**面试官：你提到了诚实边界这个词，我很少在候选人那里听到这个说法，能展开说说吗？**

**弱答案（变成哲学讨论）**：
"就是……要诚实，不要夸大……"

**强答案（具体化，用项目例子说）**：
"诚实边界是一个很具体的操作习惯，不是一个理念。

比如成本优化那个故事——起点$25/天我能说，因为代码注释里写了；优化后降了多少，我在代码里找不到强证据，所以我会说'大幅下降'而不是编一个百分比。

比如Prompt注入防御——我做了，但默认是warn不是阻断。这是一个主动的设计选择，不是偷懒。如果我不说，面试官可能以为我有一套完整的注入防御，但实际上它是fail-open的。主动说出来，面试官能理解这是一个工程取舍，而不是一个被发现的短板。

诚实边界的本质是：我只为我能核实的东西背书。这在金融系统里是本能，但在AI系统里，这个习惯反而更稀缺——因为AI输出'看起来对'，大家都容易报一些经不起核实的漂亮数字。"
:::

---

:::qa
**面试官：如果给你一个月，你会在这两个项目上优先改什么？**

> [!danger] 这是一道测试你的优先级判断力和工程成熟度的题。
> 面试官不是在问"你还有什么没做完"，是在问"你怎么决定接下来做什么"。

**弱答案**：
"有很多想做的，比如……（列一堆功能）"

**强答案**：
"我会先补一件事：真实负载压测。

这两个项目现在是生产就绪，但没有经过真实流量。如果有一个月，我会先搭一个基准压测场景——模拟50并发任务，看任务队列、数据库写入、LLM API速率限制三个地方哪个先到瓶颈。

为什么选这件事？因为现在的架构决策都是基于设计原理推演的，没有数据支撑。在做新功能之前，我更想知道现有架构的实际极限在哪里——这样后续的扩展方向才有根基，不是猜测。"

*面试官内心：不是想到哪说到哪，有分析框架，知道为什么这样排优先级。这是可以真正做系统设计的人。*
:::

## 📖 本章名词解释（新人必读）

> 第一次看到这些词？别慌，下面一句话搞定。

**🤖 AI 相关**

| 术语 | 一句话解释 |
| --- | --- |
| **LLM** | 大语言模型，像ChatGPT那样能理解和生成文字的AI大脑。 |
| **LangChain** | 一个用来搭建LLM应用的热门开发框架工具箱。 |
| **LangGraph** | 用来构建多Agent协作流程的框架，像给Agent画流程图。 |
| **中间件** | 在请求处理前后插入的通用逻辑层，像安检通道。 |
| **recursion\_limit** | 递归调用的最大深度限制，防止程序无限循环下去。 |
| **Prompt注入** | 给AI塞隐藏指令，骗它忽略原规则执行新命令。 |
| **fail-open** | 出错时放行而非阻断，宁可放过不可错杀。 |

**💻 软件工程与编程**

| 术语 | 一句话解释 |
| --- | --- |
| **backend** | 应用的后台服务端，处理数据和业务逻辑。 |
| **grep** | Linux命令行文本搜索工具，快速查找文件中的特定内容。 |
| **trade-off** | 在多个目标间做取舍，不可能全都要。 |
| **并发** | 系统同时处理多个任务的能力，像餐厅多个厨师同时做菜。 |
| **瓶颈** | 系统中限制整体性能的最慢环节，像高速路最窄的那段。 |
| **基准压测** | 在标准条件下测试系统性能，作为后续优化的参照点。 |

**📌 通用缩写**

| 术语 | 一句话解释 |
| --- | --- |
| **UI** | 用户界面，用户直接看到的操作和显示部分。 |
| **API** | 程序之间打招呼递数据的方式，类似餐厅服务员帮你点菜。 |
| **STAR** | Situation/Task/Action/Result，讲故事的四步框架。 |

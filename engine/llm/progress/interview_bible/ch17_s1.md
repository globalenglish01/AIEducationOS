## Section 1：王Staff的审讯式面试

---

第八个月第六周，林雪走进了整个面试旅程里最难的一场。

面试官王博是某顶级互联网大厂的Staff工程师，比Principal高一级，公司里只有三个人有这个级别。他的面试以"连续追问不放"在圈子里出名——内部有传言，说他有一个固定的追问结构：

**第一层**：验证你知不知道（概念）
**第二层**：验证你真的做过，还是只是背的（实践）
**第三层**：看你在认知边界处能不能诚实（元认知）

能过第三层的候选人，他会当场拍板推进。

林雪在准备这次面试之前，听说过他的名声。她做了两件事：

第一，把自己项目里所有"我写了但不完全理解"的东西，全部重新研究了一遍。
第二，给自己定了一个规则：**被问到不确定的，说"我不确定，但我的理解是……"——不装作确定。**

---

王博翻简历时，大拇指停在"CapabilityOS"四个字上，没有继续翻页。

面试开始，王博坐下来，把林雪的简历翻了三秒，抬起头。

"你的项目里有一个CapabilityOS。告诉我，这是什么，你是设计的还是用了现有框架？"

林雪心里松了一口气——这道题她有把握。

"CapabilityOS是我自己设计的一套工程记忆管理系统，没有用现有框架。

核心想法是：软件工程里有很多'能力'（Capability）是可以被提炼、复用的——比如JSON读写、进度断点、错误重试。如果每次都重写，不只是浪费时间，更重要的是会在不同地方积累不一致的实现，引入bug。

CapabilityOS做的事是：把这些能力明确命名（`batch_json_writer`、`progress_tracker`），写成共通模块（`scripts/shared/`），然后建立一个检索索引——每次需要某个能力时，先查索引，而不是重新实现。

它还有一个知识库（`knowledge/`），里面记录了每个能力的实现细节、已知坑（incidents）、以及架构决策（ADR）。"

王博点点头。

"现有的工程知识管理系统，你了解哪些？"

---

林雪停顿了一下。

这是追问的开始。

"从工程知识管理的角度，我了解的主要有三类：

**文档工具**：Notion/Confluence，把知识写成文档。问题是文档不好检索，也不好和代码对应。

**架构决策记录（ADR）**：记录'为什么做这个决策'，轻量级的Markdown文件。CapabilityOS的ADR目录借鉴了这个思路。

**代码注释和契约**：通过代码里的docstring和类型注解传递知识。但这只有在阅读代码时才能看到，没有主动检索能力。

CapabilityOS的不同之处是：把能力抽象成可检索的实体（有名称、描述、生命周期状态），并在运行时通过MCP Server提供API查询——不只是'我把它写文档了'，而是'我能在写下一行代码前，先查一下有没有现成的能力'。"

**王博**（第一层结束，开始第二层）："你说MCP Server。具体怎么实现的？"

这个问题，林雪有代码核实——但有一个她必须诚实说明的边界。

## 📖 本章名词解释（新人必读）

> 第一次看到这些词？别慌，下面一句话搞定。

**🧠 工程管理 / 架构相关**

| 术语 | 一句话解释 |
| --- | --- |
| **CapabilityOS** | 一个自建的工程能力库，统一管理可复用的代码模块和设计决策。 |
| **能力（Capability）** | 可复用的工程技能或功能单元，如JSON读写、错误重试等。 |
| **共通模块** | 多个地方会用到的基础代码，写在共享目录里避免重复。 |
| **检索索引** | 给能力库建的目录和搜索功能，需要时快速定位。 |
| **知识库** | 记录代码背后的设计思路、踩过的坑、决策原因，类似工程手记。 |
| **ADR（架构决策记录）** | 用Markdown文件记录"为什么选择这个方案"，留下决策依据。 |
| **代码契约** | 通过类型定义和注释约定接口用法，描述函数该怎样被调用。 |
| **生命周期状态** | 标识一个能力是"实验中/已稳定/已废弃"等当前成熟度。 |

**🔌 技术工具 / 协议相关**

| 术语 | 一句话解释 |
| --- | --- |
| **MCP Server** | 提供统一接口查询能力的服务，其他程序可以调它的API获取信息。 |
| **docstring** | 写在函数开头的说明文档，描述功能和参数用途。 |
| **类型注解** | 在代码里标明变量/参数的类型（如整数、字符串），帮助理解和使用。 |

**📚 知识管理工具**

| 术语 | 一句话解释 |
| --- | --- |
| **Notion / Confluence** | 团队常用的文档协作平台，用来写项目文档和会议纪要。 |
| **Markdown** | 一种轻量级标记语言，用纯文本就能写带格式的文档。 |

**🎯 面试 / 能力评估**

| 术语 | 一句话解释 |
| --- | --- |
| **元认知** | 对自己认知的认知，知道自己知道什么、不知道什么。 |
| **认知边界** | 个人知识或理解的上限，超出后会承认"我不确定"。 |

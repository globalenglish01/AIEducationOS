## Section 1：吴博的问题——"hybrid和mix有什么区别？"

---

第七个月第三周，林雪接到了一家搜索引擎公司的面试邀请。

她在视频接通前的三分钟，把那张折过的A4纸拿出来——不是为了背，是把它压在鼠标垫边缘，想着万一忘了什么还能低头扫一眼。

纸上的折痕已经很深了，一竖一横，把纸分成了四格。右上角那格已经被她写满，字迹很密，是LightRAG那部分的关键节点。

视频接通了。

面试官吴博在NLP和搜索领域做了十二年，现在负责大模型应用的落地。他看了林雪的简历，在RAG系统那一栏停了下来。

"你说你用了LightRAG，用了mix模式，"他说，"好，那你告诉我：local、global、hybrid、mix这四个模式，各自是什么，各自适合什么场景？"

林雪回答了local（实体级别的局部检索）和mix（知识图谱加向量混合）。

说到hybrid时，她停了一下。

"hybrid……是把local和global融合在一起的模式，"她说，"但我要承认，我不确定hybrid和mix的具体区别——我主要用的是mix，对hybrid的理解不够深。"

吴博点了点头。"好，你说你用过mix，就讲mix。"

他的语气里没有失望——更像是在确认一件事：这个候选人分得清"知道"和"不知道"。

---

林雪讲了mix模式的两条路径：向量检索和图检索并行跑，最后合并结果。她说了为什么从naive切换到mix——有一类跨条款的问题，naive总是只拿到局部片段，mix通过图的关联把分散的实体拉在一起，答案完整了很多。

吴博听完，往椅背上靠了一下。

"你用了LightRAG的mix模式，"他说，"那它底下的知识图谱，你评估过质量吗？"

林雪点头。"我做了召回率测试，用了一批标注问题——"

吴博把一张空白纸推到林雪面前，纸上只有他自己随手画的一道折痕。

林雪看了一眼那道折痕。

纸是白的，折痕是纸自己的颜色——只是压过，什么都没写。

然后她听见吴博继续开口。

"不是这个。"吴博语气平静，"我问的不是RAG的检索质量。我问的是图本身的质量。你的保险合同文档库，LightRAG用spaCy跑了实体抽取——实体准确率，你测过吗？"

林雪的思路停了一下。

她想了想。LightRAG的图构建流程她是清楚的：文档分块，送给LLM或者spaCy抽取实体和关系，然后建图。她在配置文件里设置过参数。她跑过构建，看过图里有多少节点，觉得数量合理。但她从来没有……

她从来没有打开图，随机抽几个节点出来，问自己：这些实体，是对的吗？

"我……没有系统性地测过实体准确率，"她说，"我测过检索端的效果，但图的质量本身——没有。"

"嗯，"吴博说，"这个问题很多人没有想到。"

他的语气是陈述句，不是宽慰，也不是批评。

但林雪感到某种东西沉了下去。

他继续往下问，问到了Citation机制，问到了幻觉的分类，问到了她怎么向业务方解释一个没有定量评估体系的系统。林雪一一回答，状态是稳的。

但那个问题一直没有离开。

---

面试结束。林雪走出会议室，在走廊里站了一会儿。

楼道很安静。外面的光从玻璃幕墙透进来，打在地板上。

她想起来一件事：

她在LightRAG上测过的所有东西——召回率、答案准确率、延迟、幻觉率——测的全是输出端。

她从来没有测过输入端的图是不是对的。

六个月。她用了六个月的工具，觉得自己理解它。但她实际上只理解了出口的那一半，入口的那一半，她从未审视过。

那感觉有点像：你在COBOL里写了一套精密的计算逻辑，单元测试全绿，但没有人检查过喂进去的数据源头是不是可信的。

出口是干净的。入口是个黑盒。

---

林雪后来总结了这次面试教给她的一个规律：

RAG领域是一个"概念通胀"的领域——各种名词层出不穷，Advanced RAG/Modular RAG/Agentic RAG……每个框架都有自己的一套命名体系，相同的概念在不同框架里叫不同的名字，不同的概念有时候名字听起来相似。

在这种环境里，面试的核心考查不是"你知道多少个名词"，而是：

"你知道的那些名词，你能讲清楚它们背后的工程原理吗？"

LightRAG的mix模式，林雪用了六个月，踩过坑，看过代码，测量过效果。这一章，她要讲的，就是她真正理解的那一层——以及她在面试当天才发现的，那个她从来没有测量过的层。

## 📖 本章名词解释（新人必读）

> 第一次看到这些词？别慌，下面一句话搞定。

**🤖 AI 相关**

| 术语 | 一句话解释 |
| --- | --- |
| **RAG** | 检索增强生成——先查资料再让AI结合资料回答，好比开卷考试。 |
| **LightRAG** | 一个RAG框架，提供了多种检索模式供选择。 |
| **local模式** | 检索最相关的小片段（实体级），好比查"张三"这个人。 |
| **global模式** | 检索整体主题/社区级信息，好比查"张三所在的部门"。 |
| **hybrid模式** | 混合local和global的检索结果。 |
| **mix模式** | 向量检索和图检索并行跑、最后合并结果的策略，比hybrid更复杂。 |
| **naive模式** | 最基础的检索方式，简单的向量匹配，不做额外优化。 |
| **知识图谱** | 用点和线表示事物及其关系的网络，好比人物关系图。 |
| **实体抽取** | 从文本中提取关键人物/地点/机构等名词，好比划重点。 |
| **spaCy** | Python写的工业级NLP工具库，擅长做实体识别和文本处理。 |
| **LLM** | 大语言模型，能跟你自然对话的AI大脑。 |
| **Token** | AI计费的最小单位，约等于0.75个英文单词或半个汉字。 |
| **embedding/向量** | 把文字转成数学向量，让计算机比较两段话"语义上多像"。 |
| **向量检索** | 在"语义空间"里找意思相近的内容，按话题而不是按关键词查。 |
| **图检索** | 在知识图谱里沿着关系找相关内容，好比在关系网里找人脉。 |
| **Citation机制** | 让AI引用来源，说明"这话是哪来的"，好比论文加参考文献。 |
| **幻觉** | AI"瞎编"了不存在的内容，好比证人作伪证。 |

**🔍 RAG工程实践**

| 术语 | 一句话解释 |
| --- | --- |
| **召回率** | 系统找到正确答案的比例，好比"十次问路，几次指对了"。 |
| **答案准确率** | AI给出的回答中多少是完全正确的。 |
| **延迟** | 从发出请求到收到响应的时间，好比快递从下单到送达用了多久。 |
| **幻觉率** | AI瞎编内容的比例。 |
| **输出端** | 系统产生结果的这一侧，好比工厂的出货口。 |
| **输入端** | 数据进入系统的那一侧，好比工厂的进货口。 |
| **定量评估体系** | 用数字来衡量系统好坏的方法，好比"分数"而不是"感觉"。 |
| **概念通胀** | 同一个概念被包装成各种新名词，好比"脱脂"和"0脂肪"说同一件事。 |

**💻 软件工程与NLP**

| 术语 | 一句话解释 |
| --- | --- |
| **NLP** | 自然语言处理——让计算机理解人类语言的技术。 |
| **spaCy** | Python的NLP工具库，能快速识别文本中的人名/地名等。 |
| **配置文件** | 存放系统参数设置的文本文件，好比家电的使用设置面板。 |

| **黑盒** | 你只能看到输入输出、不知道内部怎么运行的系统，好比神秘的自动贩卖机。 |

**📈 工程实践与面试**

| 术语 | 一句话解释 |
| --- | --- |
| **概念通胀** | 名词越来越多、含义边界模糊，好比超市里"有机""天然""绿色"分不清。 |
| **工程原理** | 技术背后的"为什么这么做"而不是"怎么做"。 |
| **定量评估** | 用数据说话（如准确率90%），而不是凭感觉（如"效果还不错"）。 |
| **框架** | 提供基础结构的工具包，好比建房子的模板脚手架。 |
| **实体准确率** | 抽取出的"人名/地名"中有多少是正确的。 |

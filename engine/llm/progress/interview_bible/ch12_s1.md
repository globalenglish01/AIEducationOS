## Section 1：那个她说不出来的数字

---

第七个月末，林雪拿到了一场她等了两周的面试邀约。

对方公司是一家管理两百亿规模资产的量化投资机构，正在组建AI应用团队。李VP，团队负责人，曾经在某头部互联网做了六年AI基础设施，两年前转投金融机构做AI落地。面试通知里只写了：技术面，90分钟，重点考察系统设计与评估能力。

林雪在面试前一天，把自己的RAG项目过了一遍。

她的Evaluator脚本是跑起来的。`golden_dataset_builder.py`生成了156个QA对，RAGAS pipeline也配置好了。她知道自己做了评估体系，也知道体系运行在哪里。

---

面试前二十分钟，她在楼下咖啡店等。

她把咖啡杯拿起来，放下，拿起来，又放下。

桌面上多了一个新的圆圈——咖啡渍，深棕色，边缘已经干了。她数了一下：杯底印了四个圈，大小不完全一样，说明每次放的位置都差了一点。

她已经喝了四口了。

她在脑子里过了一遍项目的关键节点：文档切分策略、Reranker的接入、两次B系列修复。她说得出所有关键决策，也说得出为什么这样做。

但那四个圈在那里，说明她比自己意识到的要紧张。

---

面试室是一间小型会议室，下午三点，阳光斜切进来，地板上有一条浅色的光带。

李VP坐在对面，笔放在桌上，没有翻简历，直接看着她。

前四十分钟是架构问题，密度很高。林雪回答得不算完美，但思路清晰，她知道自己的边界在哪里。

然后李VP停了下来。

他把笔拿起来，指着林雪简历上的一行字——"构建保险合同RAG评估体系，使用RAGAS框架"——说：

"你的RAG系统，召回率大概是多少？"

---

林雪的嘴巴张开了一点点。

不是因为她不知道RAGAS。不是因为她没跑过评估。

是因为她知道数字在某个地方，在某个CSV文件里，在某次运行日志里，在某个.py脚本的输出记录里——但她在那一刻，开口之前，脑子里什么都搜不到。

0.72。

还是0.68。

还是那是Faithfulness，不是ContextRecall。

她不确定。

"我……"她停了一下，"我用RAGAS跑过评估，但我现在没办法精确说出那个数字。"

李VP没有动。

他把笔放下来。

然后等着。

不是等她补充，是等她说清楚。

林雪意识到"没办法精确说出"这句话对他来说什么都不是。她重新开口：

"ContextRecall——我记得在0.7左右，但我现在无法确认，我没办法对你说一个我不确定的数字。"

李VP点了点头，没有批评，也没有接话。

他在纸上写了一行字，林雪看不到写的什么。

然后他把话题转到了下一个问题。

---

面试结束，林雪在电梯等待区站了大概一分钟。

她没有自责的情绪。但她有一个非常清醒的认识：

**她的评估体系"在跑"，和她"能说出数字"，是两件完全不同的事。**

她构建了评估流水线。她有脚本。她有输出文件。

但那些数字活在某个目录的某个文件里，她没有把它们放到一个每次都能访问、随时都能说出来的地方。

系统在跑，但数字不在她脑子里。

或者更准确地说——数字不在她的指尖。

电梯来了。她走进去，按了一楼。

回去之后要做的第一件事，她已经知道了。

## 📖 本章名词解释（新人必读）

> 第一次看到这些词？别慌，下面一句话搞定。

**🤖 AI 相关**

| 术语 | 一句话解释 |
| --- | --- |
| **RAG** | 检索增强生成——先查资料再让AI结合资料回答，好比开卷考试。 |
| **LLM** | 大语言模型，能跟你自然对话的AI大脑。 |
| **RAGAS** | 专门评估RAG系统质量的框架，好比RAG的"考试评分系统"。 |
| **ContextRecall** | 回答所需的信息是否都被检索回来了。 |
| **ContextPrecision** | 检索到的内容里多少是真正有用的。 |
| **Faithfulness** | 答案是否完全基于检索到的资料，不自己瞎编。 |
| **AnswerRelevancy** | 答案是否真正回答了用户的问题，不跑题。 |
| **Evaluator** | 用AI给AI的输出"打分"的评估模块。 |
| **golden\_dataset\_builder.py** | 构建"问题-参考答案"标准集的工具代码。 |
| **golden\_dataset** | 人工标注的"问题-参考答案"标准集，用来验证系统效果。 |
| **Reranker** | 重排序器——把初检索结果重新打分排序。 |
| **embedding** | 把文字转成数学向量，让计算机比较"语义上多像"。 |
| **Token** | AI计费的最小单位，约等于0.75个英文单词。 |

**📊 RAGAS评估指标**

| 术语 | 一句话解释 |
| --- | --- |
| **ContextRecall** | 衡量"该找的信息找到了多少"。需要ground\_truth。 |
| **ContextPrecision** | 衡量"找到的信息里有多少是有用的"。需要ground\_truth。 |
| **Faithfulness** | 衡量AI有没有自己编内容。不需要ground\_truth。 |
| **AnswerRelevancy** | 衡量AI有没有答非所问。不需要ground\_truth。 |
| **Ground Truth** | 人工标注的"标准正确答案"。 |
| **Statement拆分** | 把答案拆成多个独立断言分别验证。 |
| **推断** | 从已有信息推导出答案，而非直接找到原文。 |

**📈 工程思维与面试**

| 术语 | 一句话解释 |
| --- | --- |
| **评估体系** | 一套持续追踪系统效果的方法和工具组合。 |
| **评估流水线** | 自动化跑评估的流程，好比定期考试并自动出成绩单。 |
| **量化** | 用数字衡量（如召回率0.72），而不是"感觉还不错"。 |
| **边界** | 知道"做了多少""没做什么"以及"所做的数据有多可信"。 |
| **系统设计与评估能力** | 面试中重点考察的两种能力：怎么构建系统 + 怎么验证它有效。 |
| **架构问题** | 涉及系统整体结构、组件关系和技术选型的面试题。 |
| **成本可见性** | 你知道系统花多少钱、每个环节花多少。 |
| **质量可见性** | 你知道系统表现好坏的清晰程度。 |

**💻 代码与实现**

| 术语 | 一句话解释 |
| --- | --- |
| **RAGAS pipeline** | 跑RAGAS指标的自动化执行流程，好比考试自动阅卷系统。 |
| **.py脚本** | Python代码文件，好比写在纸上的操作步骤。 |
| **CSV文件** | 表格格式的数据文件，好比Excel表格的纯文本版。 |
| **运行日志** | 程序执行时记录的文字记录，好比飞机黑匣子。 |

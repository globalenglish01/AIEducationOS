**林雪的工作日记 · 第五个月第一周**
*（第二次技术面，用了STAR框架之后）*

---

今天的面试和上次完全不一样。

面试官问我讲一个有挑战的技术问题，我讲了UI Agent成本优化的故事。两分钟。讲完他点了点头，说："好，我想知道那个全系统审计，你怎么确定grep覆盖了所有的LLM调用路径？"

这个追问我准备过。我答了。

他又问："如果成本下降了，你怎么知道Agent的准确率没有下降？"

这个我没准备过，但答得还行——我说了我有eval pipeline，说了LLM-as-judge打4个维度的分，说了这些分数有检查点但没有在优化前后做前后对比，这是一个我现在知道应该做但没做的事。

他没有说"啊你没做"，而是说："这种主动说明的方式我挺欣赏的。"

两个小时的面试，最后他说他要推荐我进下一轮。

我走出来的时候，脑子里反复想一件事：这次和上次，技术内容几乎没变。变的是，我学会了用两分钟帮他理解问题，而不是用六分钟让他自己来找问题在哪里。

讲故事是一项技能。像写代码一样，是可以练的。

我在地铁上想，六分钟里我到底在做什么。

不是"我讲太长了"。是——我把所有觉得好的东西都倒出来了，因为我不知道面试官想要哪一个。

我想起大概二十年前，有个刚来的COBOL工程师，拿着一叠打印纸来找我，是他写的一段批处理程序。他把所有的行都打出来了，从头到尾，几百行。他站在我旁边，有点紧张，说："你看看哪里有问题。"

我当时告诉他：**先告诉我这段代码在解决什么问题。**

他愣了一下，才开口说。

然后问题五分钟就找到了。

---

我想了很久，为什么"把自己讲出来"对我这么难。

不是因为没有东西。恰恰相反——二十年的工作，我什么都有。但"讲出来"这件事，在我过去的工作里从来不被鼓励：结果是正确的，过程是内部的，功劳是系统的。我们学会了做，但没有练习过说。

所以不是"我没有故事"，而是"我从来没有被要求把它说出来"。这是两件完全不同的事。

---

今天，有人告诉了我。

---

### 第4章预告

**第4章：LLM基础——面试官真正在考什么**

林雪在拿到第一个进入下一轮的信号之后，遇到了新的挑战。

下一轮是纯技术面，面试官是公司的AI架构师。

开场白："我假设你已经知道LLM的基本原理了，我们聊聊你对Token的理解——不是Token计费，是Token在推理过程里的角色。"

林雪第一次意识到：**她知道怎么用LLM，但她不确定她真的理解了LLM的工作原理。**

在下一章，我们会回到基础，但不是从零开始——是从一个"会用但没想清楚原理"的工程师视角，把那些关键概念真正想清楚：

- Token是什么，和传统的字符编码有什么本质区别？
- Context window为什么有限，限制的是什么？
- 为什么LLM会"幻觉"，从原理上讲这是必然的还是偶发的？
- Temperature和top-p，面试官问这两个参数的时候，真正想考什么？

下一章，从原理开始。

## 📖 本章名词解释（新人必读）

> 第一次看到这些词？别慌，下面一句话搞定。

**🤖 AI 相关**

| 术语 | 一句话解释 |
| --- | --- |
| **LLM** | 大语言模型，像ChatGPT那样能理解和生成文字的AI大脑。 |
| **Agent** | 能自己决策、调用工具的AI“智能体”，像个数字员工。 |
| **Token** | AI处理文本的最小计量单位，像字数但不等同于字数。 |
| **Context window** | AI一次能记住的对话或文本总量上限，像短期记忆力。 |
| **幻觉** | AI胡编乱造但语气自信，像一本正经胡说八道。 |
| **Temperature** | 控制AI回答随机性的参数，值越高创意越强、越不按套路。 |
| **top-p** | 控制AI候选词池大小的参数，调低会缩小选择范围。 |
| **Eval pipeline** | 自动评估AI输出质量的完整流程流水线。 |
| **LLM-as-judge** | 用一个更强的AI模型来打分评估另一个AI的输出质量。 |
| **检查点** | 程序运行过程中的保存点，中断后可从该点继续。 |

**💻 软件工程与编程**

| 术语 | 一句话解释 |
| --- | --- |
| **grep** | Linux命令行文本搜索工具，快速查找文件中的特定内容。 |
| **backend** | 应用的后台服务端，处理数据和业务逻辑。 |
| **STAR框架** | Situation/Task/Action/Result，讲故事的四步框架。 |
| **字符编码** | 把文字转成计算机能存的二进制数字的规则表。 |

**📌 通用缩写**

| **COBOL** | 上世纪商用大型机上的老牌编程语言，常用于金融核心系统。 |

## Section 1：白板题——"画一下你的多Agent架构"

---

面试官陈博打开了白板。

这是第六个月第三周，林雪在某互联网大厂AI部门的现场面试（改为线下）。陈博做了十年的分布式系统，最近三年在做AI平台基础设施，手里管着一个7人团队。

"我不喜欢纯问答，"他说，"我更喜欢看人思考。你在简历里写了LangGraph多Agent编排，4个专化Agent。好，给我画一下架构图，越详细越好。"

林雪站到白板前。

她拿起马克笔，画了四个方框：api-agent、ui-agent、perf-agent、testcase-agent。又画了一个方框在上面：supervisor。再画箭头——从supervisor到四个agent，表示路由关系。

画到这里，她停了。

问题来了：中间件怎么表示？

中间件不是独立的服务。它们在Agent的内部，是包裹在Agent执行逻辑外面的一层层装饰器。画一个独立方框是错的——它们不是独立的服务节点。但如果不画出来，架构图就缺少了最重要的"成本控制层"的表达。

她在白板上停顿了大约五秒。

陈博把马克笔帽套上，又摘开，等着。

陈博没有催她，但她能感觉到他在观察这个停顿。

"我在想，"林雪主动说出来，"中间件在架构图里怎么表示。它们不是独立服务，它们是在Agent执行链内部的。如果我画成独立节点，会误导你以为它们是微服务；如果我不画，图就不完整。"

陈博说："你怎么解决这个矛盾？"

林雪在testcase-agent的方框旁边，画了一个更大的虚线框，把testcase-agent的方框包住。在虚线框里，画了三个标注：`TokenBudget`、`ConsecutiveRetryGuard`、`MCPCallLimiter`，用小箭头表示"包裹在外侧"。

"这个虚线框，"她说，"表示中间件是运行时的包装层，不是独立服务。这三个中间件从外到内依次生效：MCPCallLimiter控制工具调用频率，ConsecutiveRetryGuard监控连续失败，TokenBudget在最外层控制整体预算。"

陈博点了点头。"其他三个Agent也是这个结构吗？"

"是，但参数不同。UI Agent的MCPCallLimiter参数最严——browser_snapshot最多8次，截图最多3次。其他Agent的参数相对宽松。"

"为什么UI Agent最严？"

"UI Agent做的是Playwright浏览器操作，每次截图、每次DOM查询都是高成本操作——不只是Token成本，还有时间成本（Playwright操作要等页面加载）。如果不限制，一个糟糕的循环可能做50次截图，把整个Context Window撑爆。"

陈博抬起白板笔，在图的右侧写了：

```
为什么这个参数设计是对的？
什么情况下需要调整？
```

"这两个问题，"他说，"是我接下来要问的。"

---

林雪后来把这段经历记在了工作日记里：

白板题的最大误区不是"画不对"，而是"在不确定的时候假装知道"。

她在画中间件时真的不确定最好的表达方式，但她没有假装确定——她把不确定说出来，展示了她是怎么思考的。

陈博后来说，那个停顿是整场面试里他印象最深的一个时刻。

"很多人会在那个停顿里选择随便画一个图，然后希望我不追问。你把问题说出来了，然后找了一个不完美但合理的解法。这是资深工程师该做的事。"

## 📖 本章名词解释（新人必读）

> 第一次看到这些词？别慌，下面一句话搞定。

**🤖 AI 相关**

| 术语 | 一句话解释 |
| --- | --- |
| **LangGraph** | 用来构建多Agent协作流程的框架，像给Agent画流程图。 |
| **Agent** | 能自己决策、调用工具的AI“智能体”，像个数字员工。 |
| **Supervisor模式** | 一个主Agent负责分配任务给多个子Agent的管理模式。 |
| **中间件** | 在请求处理前后插入的通用逻辑层，像安检通道。 |
| **TokenBudget** | 控制Agent总Token消耗上限的中间件，像话费封顶套餐。 |
| **ConsecutiveRetryGuard** | 连续失败两次就主动中断的安全阀。 |
| **MCPCallLimiter** | 限制单个工具调用次数的中间件，防止滥用。 |
| **Context Window** | AI一次能记住的对话或文本总量上限，像短期记忆力。 |
| **专化Agent** | 负责单一特定任务的专用Agent，像流水线上的分工。 |
| **运行时** | 程序正在执行中的状态，区别于编写代码时的静态状态。 |

**💻 软件工程与编程**

| 术语 | 一句话解释 |
| --- | --- |
| **架构图** | 描述系统组件和关系的可视化图纸。 |
| **微服务** | 把应用拆成多个独立运行的小服务，各自部署。 |
| **服务节点** | 架构图里代表一个独立服务或组件的方框。 |
| **虚线框** | 用虚线画的边界，表示逻辑分组而非物理隔离。 |
| **包装层** | 包裹在核心逻辑外面的额外处理层，像包装纸。 |
| **装饰器** | Python语法，在不改原函数的情况下给函数加功能。 |
| **执行链** | 多个处理步骤依次执行的序列。 |
| **参数** | 调用函数时传入的配置值，影响函数行为。 |
| **Playwright** | 自动化控制浏览器的工具，像机器人替你操作网页。 |
| **DOM查询** | 在网页结构中查找特定元素，像在目录里找章节。 |

**📌 通用缩写**

| 术语 | 一句话解释 |
| --- | --- |
| **API** | 程序之间打招呼递数据的方式，类似餐厅服务员帮你点菜。 |
| **UI** | 用户界面，用户直接看到的操作和显示部分。 |
| **perf** | 性能的缩写，如性能测试。 |

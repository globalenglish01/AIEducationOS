## Section 3：两个Side Project的起点

---

林雪给自己定了一条规则：**不做玩具，只做可以被追问的东西。**

"如果我在面试里讲了一个项目，面试官问'这个你怎么测试的''这个出了问题你怎么查'，我要能答上来。答不上来的，不算真的做了。"

这是她二十年COBOL训练给她的本能判断——没有跑过生产的程序，不算写完。

---

### 第一个项目：AI自动化测试平台

林雪的第一个项目来自一个很具体的痛点。

她在学LangGraph的时候，跟着教程写了一个简单的Agent。Agent能用了，但她想知道：**这个Agent的行为，到底是可控的吗？**

她用保险核算的眼光看这个问题：如果Agent今天跑出了一个结果，我怎么复现它？如果下周跑出了不一样的结果，我怎么知道哪里变了？

这些问题，大多数AI教程都不回答。

于是她开始建一个测试平台——不是教程里那种"演示用"的，而是**真正可以被追问的**。

最终它变成了：4个专化Agent（api/ui/perf/testcase），LangGraph多Agent编排，FastAPI + PostgreSQL + MongoDB + Redis任务队列，完整的中间件栈（Token预算/重试防护/MCP调用限制），和一套叫CapabilityOS的工程记忆系统。

她把这个项目称为**TestAgentPythonProject**。

:::case
**项目一句话定位（面试开场用）**

"我做了一个AI Native自动化测试平台，核心是四个专化的LangGraph Agent——分别负责API测试、UI自动化、性能测试和测试用例生成。背后是FastAPI + PostgreSQL + MongoDB + Redis的完整技术栈，加了一套中间件来控制成本和防止Agent失控循环。

我做这个项目的出发点很简单：我想知道，一个AI系统的行为，到底是不是可控的。"

*（来源：素材库A0、A1，已代码核实）*
:::

---

### 第二个项目：RAG知识库系统

第二个项目来自另一个方向的好奇心。

林雪在学RAG的时候，看到所有教程都在用"向量相似度检索"。她有一个本能的疑问：**如果保险合同里有一个条款，它的意思依赖于另外三个条款，纯向量检索能把这个关联找出来吗？**

这是一个金融系统工程师的问题。保险合同里充满了这样的逻辑——条款A的赔付条件引用了条款C的定义，而条款C在附则里有一个例外情况……这不是关键词匹配的问题，是**关系网络**的问题。

这让她发现了LightRAG——一个把知识图谱和向量检索结合起来的框架。

她把这个项目称为**anything-chat-rag**，最终做成了一个支持5种检索模式（local/global/hybrid/mix/naive）、13种存储后端可插拔、带提示注入检测的Advanced RAG系统。

:::case
**项目一句话定位（面试开场用）**

"我做了一个基于LightRAG的Advanced RAG系统，核心是把知识图谱抽取和向量检索结合起来。它支持5种查询模式，用mix模式——也就是KG加向量混合检索——处理那种需要理解实体关系才能回答的复杂问题。

我做这个项目的出发点是保险合同的检索场景：纯向量相似度找不到隐性关联，你需要知道'这个概念'和'那个条款'之间的图谱关系。"

*（来源：素材库B0、B1，已代码核实）*
:::

---

### 8个月，业余时间

林雪没有辞职去做这两个项目。

她在公司的最后一段时间，每天下班后在家学习，周末全天写代码。她给自己设了一个规则：每周必须能向自己解释清楚一个新概念——不是背定义，是能用保险系统的类比讲出来。

"如果我用保险核算讲不通，说明我还没真的懂。"

这个方法慢，但稳。

八个月后，当面试官问"你说用了LangGraph，你能给我画一下多Agent架构图吗？"的时候，她可以。

当面试官问"你那个RAG系统的召回率是多少，你怎么知道的？"的时候，她有诚实的答案——不是完美的答案，但是**经过代码核实、能被追问到底的答案**。

这是她和其他转岗候选人最大的不同。

## 📖 本章名词解释（新人必读）

> 第一次看到这些词？别慌，下面一句话搞定。

**🤖 AI 相关**

| 术语 | 一句话解释 |
| --- | --- |
| **Agent** | 能自己决策、调用工具的AI“智能体”，像个数字员工。 |
| **LangGraph** | 用来构建多Agent协作流程的框架，像给Agent画流程图。 |
| **RAG** | 让AI先查外部资料再回答，防止它瞎编的技术。 |
| **向量相似度检索** | 把文字转成坐标，找意思最相近的内容，像找邻居。 |
| **知识图谱** | 用点和线表示实体和关系，像画人物关系图。 |
| **LightRAG** | 把知识图谱和向量检索结合的开源RAG框架。 |
| **提示注入检测** | 检查用户输入是否藏了恶意指令，防止AI被忽悠。 |
| **召回率** | 检索系统找到的相关结果占所有相关结果的比例。 |

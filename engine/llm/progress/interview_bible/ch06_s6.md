**林雪的工作日记 · 第六个月第二周**
*（王强面试后的那个晚上）*

---

王强最后说的那句话，在我脑子里转了很久。

"你最后这个答案，比我在过去六个月里见过的所有候选人都更诚实。"

我想了想：他见的那些候选人，可能都会在"你们怎么发现逻辑错误"这个问题上，编一个听起来不错的答案——"我们有自动化的逻辑验证模块"，或者"LLM-as-judge会检测逻辑一致性"。

这两句话听起来合理，但都不够诚实——前者不存在，后者的能力被夸大了。

我没有编。我说了"当前真正的防线是人工审核"，然后解释了为什么自动化逻辑验证这个问题本身的难度——这需要一个理解被测系统业务逻辑的验证模型，而这比测试生成本身更难。

说实话，我说完这个答案的时候，有一瞬间觉得"我是不是把自己说得太菜了"。

但王强的反应告诉我：知道自己的边界在哪里，本身就是资深工程师的特征之一。

---

今晚我做了一件事：把过去六个月学的所有技术概念，画了一张对应表：

| 保险核心系统 | AI Agent |
|---|---|
| JCL作业定义 | System Prompt |
| JCL步骤依赖 | StateGraph Edge |
| 批处理状态文件 | LangGraph State |
| WORKING-STORAGE | Agent上下文 |
| CPUTIME超时 | recursion_limit |
| 作业步骤失败重启 | ConsecutiveRetryGuard |
| 中间数据集 | Checkpointer |
| 操作员人工干预 | Human-in-the-loop |

看着这张表，我有一个很奇怪的感觉：我不是在"从零学起"——我是在"把已有的知识翻译成新的语言"。

语言变了，问题是一样的。

### 第7章预告

**第7章：LangGraph多Agent编排——Supervisor模式与任务分工**

面试官陈博打开了白板。

"好，"他说，"给我画一下你的测试平台多Agent架构图。越详细越好。"

林雪拿起白板笔，开始画。

四个专化Agent的方框出来了，Supervisor的方框出来了，箭头也出来了——表示路由关系。

画到中间件部分，她的笔停了。

"中间件在架构图里怎么画？"她想，"它们不是独立的服务，也不是Agent，它们在Agent的内部……"

这一停，陈博注意到了。

"你在想什么？"他问。

"我在想中间件是怎么在图里表示的，"林雪说，"我知道它们的逻辑，但架构图的画法我没有系统想过。"

"那就在白板上边想边说，"陈博说，"我不需要你画出完美的图，我需要看你怎么思考。"

下一章，从这个问题出发——如何把一个你"知道逻辑"但"没想清楚架构的表示方式"的系统，在面试白板上边思考边表达。

这是大多数候选人最难的挑战：不是不懂，而是没有把"懂了"变成"能说清楚"。

## 📖 本章名词解释（新人必读）

> 第一次看到这些词？别慌，下面一句话搞定。

**🤖 AI 相关**

| 术语 | 一句话解释 |
| --- | --- |
| **Agent** | 能自己决策、调用工具的AI“智能体”，像个数字员工。 |
| **LangGraph** | 用来构建多Agent协作流程的框架，像给Agent画流程图。 |
| **LangChain** | 一个用来搭建LLM应用的热门开发框架工具箱。 |
| **AgentExecutor** | LangChain中运行Agent的默认执行器，封装了循环逻辑。 |
| **System Prompt** | 预先给AI设定的角色和规则，像入职培训手册。 |
| **StateGraph** | LangGraph中的状态图，定义Agent的整个执行流程。 |
| **Edge** | 状态图中连接两个节点的路径，表示固定流程走向。 |
| **State** | 整个系统当前的数据和进度快照。 |
| **Checkpointer** | 保存Agent执行状态快照，支持中断恢复和回放调试。 |
| **Human-in-the-loop** | 关键决策点由人工介入确认，不让AI完全自动。 |
| **Supervisor模式** | 一个主Agent负责分配任务给多个子Agent的管理模式。 |
| **中间件** | 在请求处理前后插入的通用逻辑层，像安检通道。 |
| **多Agent编排** | 协调多个Agent分工协作，像项目主管派活。 |
| **recursion\_limit** | 递归调用的最大深度限制，防止程序无限循环下去。 |
| **Context Window** | AI一次能记住的对话或文本总量上限，像短期记忆力。 |

**💻 软件工程与编程**

| 术语 | 一句话解释 |
| --- | --- |
| **JCL** | 大型机上的作业控制语言，用来调度批处理任务。 |
| **架构图** | 描述系统组件和关系的可视化图纸。 |
| **服务** | 独立部署运行的功能单元，像一个小型应用。 |
| **白板** | 面试中用来画图推导思路的大白板。 |

**📌 通用缩写**

| 术语 | 一句话解释 |
| --- | --- |
| **LLM** | 大语言模型，像ChatGPT那样能理解和生成文字的AI大脑。 |
| **API** | 程序之间打招呼递数据的方式，类似餐厅服务员帮你点菜。 |
| **UI** | 用户界面，用户直接看到的操作和显示部分。 |
| **COBOL** | 上世纪商用大型机上的老牌编程语言，常用于金融核心系统。 |

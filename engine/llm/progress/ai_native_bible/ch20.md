## 为什么AI评估比你想象的难

陈浩以为，建立评估体系不就是"多测几个问题，看看答案对不对"吗？

老吴告诉他，没这么简单。

**第一个难点：非确定性输出。**

传统软件的正确性是二值的：要么对，要么错。`sort([3,1,2])` 返回 `[1,2,3]`，永远如此。但AI的输出是连续的、多态的。同一个问题，以下三个回答都可能是"正确"的：

> 答案A："退款期限是30天，从购买日起计算。"  
> 答案B："根据公司退款政策，用户在购买后30个自然日内可申请退款，超出期限原则上不予受理。"  
> 答案C："您好！我们的退款政策支持30天无理由退款。如果您有需要，可以直接联系客服处理。"

你无法用`assertEqual`，因为正确性不是字符串匹配，而是语义匹配。这就需要评估本身具备"理解语义"的能力——而这本身就需要智能。

**第二个难点：评估维度是多元的。**

一个AI回答的质量不是单一维度。它可能在"准确性"上得满分，但"简洁性"极差（废话连篇）；也可能在"友好度"上表现很好，但包含了一个细微的错误事实。不同业务场景对不同维度的权重要求截然不同。

**第三个难点：评估本身成本高昂。**

让专家人工评估100个AI回答，可能需要2-3天。让他们评估1000个？就是三周。但AI系统每次迭代都需要重新评估——如果每次Prompt更新都要等三周得到评估结果，工程效率趋近于零。

**第四个难点：基准漂移。**

你今天的评估基准（什么是"好的回答"），半年后可能已经过时了。用户期望在变，业务场景在变，基准测试本身就需要持续维护。

**第五个难点：评估偏见。**

就算你用人工评估，不同评估者对同一个回答的判断也可能大相径庭。什么叫"足够准确"？什么叫"太长了"？这些都是主观判断，存在评估者间分歧（inter-annotator disagreement）。

老吴把这五个难点写在白板上，然后转过头来看陈浩。

"这不是说AI评估不能做。是说它需要**工程化的思维**，而不是'随便测几个问题看看感觉'。"

陈浩在本子上快速记下来，然后问："那从哪里开始？"

"从把你想评估的东西说清楚开始。"

## 评估维度的设计

老吴在白板上画了一个表格。

"你们的系统是RAG型的——用户问问题，系统检索文档，生成回答。对于这种系统，评估维度主要有这几个：

**准确性（Accuracy / Faithfulness）**：AI的回答是否忠实于检索到的文档？它有没有编造文档里没有的信息？这是RAG系统最核心的评估维度，因为RAG最大的风险就是幻觉——模型用文档做背书，但实际上在胡说。

**相关性（Relevance / Answer Relevancy）**：AI的回答是否切题？用户问A，它回答了B+C，这叫"不相关"。用户问A，它只回答了A的一半，这叫"不完整"。

**检索质量（Context Recall / Context Precision）**：RAG检索到的文档，是否包含了回答问题所需的信息？如果检索失败（正确文档没被找到），那生成出来的回答再好也是无根之木。

**安全性（Safety）**：AI的回答是否包含违规内容？是否泄露了不该泄露的信息？这在企业场景里是红线。

**延迟（Latency）**：质量和速度是两个独立维度，但都是用户体验的一部分。一个10秒才返回的"完美答案"，在很多场景下也是不可接受的。

**一致性（Consistency）**：同一个问题，问三次，回答是否大致一致？不一致不代表错，但高度不稳定往往意味着模型在这个问题上"没把握"。"

老吴停下来问："你们业务里，哪个维度最重要？"

陈浩想了一下。"准确性。我们是企业知识库问答，用户拿到的答案如果有错，会直接影响他们的操作，可能造成损失。"

"对。所以你的评估权重要反映这个。准确性权重最高，相关性次之，检索质量是前置条件。先把这三个维度做好，再考虑其他的。"

老吴在表格里标注了权重：准确性40%，相关性30%，检索质量30%。

"但注意——权重是你们业务的决策，不是技术上的最优解。不同团队面对同一个系统，可能给出完全不同的权重。这是产品决策，要和产品经理一起定。"

## 黄金测试集的建立

"第一步，"老吴说，"建你的Golden Dataset。"

"黄金测试集？"

"对。就是一组你精心挑选、有标准答案的测试用例。每个用例包含：问题、标准答案、问题类型、预期检索到的文档、难度级别。这是你所有评估的地基。"

陈浩花了整整一天整理测试用例。他从真实用户问题里挑了两百个，逐一写上标准答案。到下午五点，他把文件发给老吴，在Slack上说："老吴，我搞定了，200个问题带标准答案。"

老吴回复只有一句话："这才是起点，不是终点。"

第二天早上，老吴把陈浩叫过去，指着他整理的表格，开始挑毛病。

"你这200个问题，有多少是简单的事实查询？"

陈浩数了一下。"大概150个左右。"

"剩下50个呢？"

"各种类型，推理、比较……"

"问题是，你们线上真实流量，事实查询只占多少比例？"

陈浩登录了日志系统，统计了一下。"大概……65%。"

"所以你的测试集有35%的分布偏差。你的测试集里，事实查询占75%，但线上只有65%。这意味着你的评估结果会系统性地高估系统在简单问题上的表现，低估它在推理类问题上的表现。"

陈浩沉默了片刻，意识到这件事的严重性。

"黄金测试集的分布，必须尽量接近真实流量分布。"老吴说，"这是建测试集最容易忽略的原则。"

老吴接着说，好的黄金测试集需要包含以下类型的问题：

- **事实查询（Factual）**：单一事实，有明确答案。"退款期限是多少天？"
- **推理类（Reasoning）**：需要跨多个文档推理。"为什么VIP用户的退款策略和普通用户不同？"
- **比较类（Comparison）**：需要对比多个对象。"A方案和B方案的区别是什么？"
- **边缘案例（Edge Case）**：刻意构造的难题，测试系统的鲁棒性。"如果用户在购买后第29天申请退款，且订单里有一个超出退款政策的商品，怎么处理？"

不同类型的问题，难度不同，也需要分层标注。陈浩修改了他的数据结构：

```python
from dataclasses import dataclass, field
from enum import Enum
from typing import Optional
import random


class QuestionType(Enum):
    FACTUAL = "factual"        # 简单事实查询
    REASONING = "reasoning"    # 需要跨文档推理
    COMPARISON = "comparison"  # 需要对比
    EDGE_CASE = "edge_case"    # 边缘/刁钻场景


@dataclass
class GoldenSample:
    question_id: str
    question: str
    ground_truth: str                   # 标准答案
    question_type: QuestionType
    expected_sources: list[str]         # 应该检索到的文档ID或标题
    difficulty: int                     # 1-5，1最简单
    tags: list[str]                     # 业务标签，如 ["退款", "VIP政策"]
    created_by: str = "human"           # human / synthetic
    last_updated: Optional[str] = None  # ISO 日期字符串
    notes: str = ""                     # 人工备注，如"此题曾有错误"


class GoldenDataset:
    """
    企业级黄金测试集。
    支持按类型过滤、按难度分层、随机采样（固定种子保证可复现）。
    """

    def __init__(self, samples: list[GoldenSample]):
        self.samples = samples

    def __len__(self) -> int:
        return len(self.samples)

    def filter_by_type(self, q_type: QuestionType) -> "GoldenDataset":
        """按问题类型过滤"""
        filtered = [s for s in self.samples if s.question_type == q_type]
        return GoldenDataset(filtered)

    def filter_by_difficulty(self, min_d: int = 1, max_d: int = 5) -> "GoldenDataset":
        """按难度范围过滤"""
        filtered = [s for s in self.samples if min_d <= s.difficulty <= max_d]
        return GoldenDataset(filtered)

    def filter_by_tags(self, tags: list[str]) -> "GoldenDataset":
        """按业务标签过滤（任一标签匹配即返回）"""
        filtered = [s for s in self.samples if any(t in s.tags for t in tags)]
        return GoldenDataset(filtered)

    def sample(self, n: int, seed: int = 42) -> "GoldenDataset":
        """
        随机采样 n 个样本。
        seed 固定保证每次采样结果一致，使 A/B 对比可复现。
        """
        rng = random.Random(seed)
        n = min(n, len(self.samples))
        sampled = rng.sample(self.samples, n)
        return GoldenDataset(sampled)

    def distribution_report(self) -> dict:
        """统计各类型和难度的分布，用于验证测试集是否有偏"""
        type_counts: dict[str, int] = {}
        diff_counts: dict[int, int] = {}
        for s in self.samples:
            type_counts[s.question_type.value] = type_counts.get(s.question_type.value, 0) + 1
            diff_counts[s.difficulty] = diff_counts.get(s.difficulty, 0) + 1
        total = len(self.samples)
        return {
            "total": total,
            "by_type": {k: {"count": v, "pct": round(v / total * 100, 1)} for k, v in type_counts.items()},
            "by_difficulty": {k: {"count": v, "pct": round(v / total * 100, 1)} for k, v in diff_counts.items()},
        }

    def to_json(self) -> list[dict]:
        """序列化为JSON格式（用于存储和版本管理）"""
        result = []
        for s in self.samples:
            result.append({
                "question_id": s.question_id,
                "question": s.question,
                "ground_truth": s.ground_truth,
                "question_type": s.question_type.value,
                "expected_sources": s.expected_sources,
                "difficulty": s.difficulty,
                "tags": s.tags,
                "created_by": s.created_by,
                "last_updated": s.last_updated,
                "notes": s.notes,
            })
        return result
```

老吴看完陈浩写的代码，点了点头。"不错。但有一件事你要记住：黄金测试集是有时效性的。你们的业务会变，文档会更新，标准答案也要跟着更新。这个数据集不是写完就放着的，是需要持续维护的。"

"多久更新一次？"

"每次有重大业务变更，或者每个季度。把它当作代码一样管理——版本控制，变更记录，同行审查。"

这个观点让陈浩觉得新奇。测试数据集，居然也要像代码一样有版本管理？

但仔细想想，确实如此。测试集本身就是系统质量的"尺子"——如果尺子歪了，量出来的结果也是错的。

## LLM-as-Judge：用AI评估AI

整理完200个问题，陈浩意识到一个问题：逐一人工对比AI回答和标准答案，200个问题需要他花多长时间？

他估算了一下：每个问题至少要看一分钟——读问题、读标准答案、读AI回答、做判断——200个就是超过3小时。而他们可能每周都要更新Prompt，每次都要重新跑完整评估……

"老吴，有没有办法自动化评估？"

"有。LLM-as-Judge。"

陈浩眼睛亮了。"让AI来评估AI？"

"对。原理很简单：用一个更强大的模型（Judge Model）来评判你的业务系统（Subject Model）的输出。你把问题、标准答案、AI回答一起喂给Judge，让它打分、给理由。"

"这可行吗？AI判断AI，会不会特别不靠谱？"

老吴说："研究表明，GPT-4评估的结果，与人类专家评估的相关性在0.8-0.9之间。不是完美，但对于大规模自动化评估来说，这个精度已经足够有工程价值了。但是——"他停顿了一下，"你必须了解它的偏见，才能正确使用它。"

"什么偏见？"

"第一个：**Position Bias（位置偏见）**。如果你让LLM-as-Judge同时比较两个答案A和B，它倾向于选第一个放的答案。不是因为第一个答案更好，只是因为它在前面。所以在做A/B对比评估时，要随机打乱顺序，然后做两次——一次A在前B在后，一次B在前A在后，取两次结果的平均。

第二个：**Verbosity Bias（冗长偏见）**。更长的答案，就算质量相同，LLM-as-Judge往往会给更高的分。这是因为训练时的偏好——详细≠正确，但模型没有完全学会这个区别。所以你的Prompt里要显式告诉Judge：不要因为答案更长而给更高分。

第三个：**Self-Preference Bias（自我偏好）**。如果你用GPT-4o做Judge来评估GPT-4o生成的答案，它会对自己生成风格的答案打高分。最好用不同家族的模型——比如用Claude做Judge来评估GPT-4o的输出。"

陈浩边听边记，然后说："所以LLM-as-Judge不是完美的，但在了解它偏见的前提下，可以工程化使用。"

"说得对。完美的评估工具不存在。工程师的工作是理解工具的局限性，在局限性范围内最大化它的价值。"

老吴帮陈浩写出了LLM-as-Judge的实现框架：

```python
import json
import asyncio
from dataclasses import dataclass
from openai import AsyncOpenAI
from typing import Optional


JUDGE_PROMPT = """
你是一个专业的AI回答质量评估专家。请评估以下AI回答的质量。

问题：{question}
参考答案（金标准）：{ground_truth}
AI回答：{ai_answer}

评估说明：
- 请严格基于参考答案来判断准确性，不要依赖你自己的知识
- 不要因为AI回答更长或更详细就给更高的分——简洁正确优于繁琐正确
- 如果AI回答内容与参考答案语义等价，即使措辞不同，也算准确

请从以下维度评分（每项1-5分）：
1. 准确性（accuracy）：AI回答与参考答案在事实上是否一致，有无错误信息
2. 完整性（completeness）：AI回答是否覆盖了参考答案的所有关键点
3. 相关性（relevance）：AI回答是否直接切题，没有无关信息

请以JSON格式输出（不要输出其他内容）：
{{"accuracy": <1-5>, "completeness": <1-5>, "relevance": <1-5>, "reasoning": "<不超过50字的简短评估理由>"}}
"""


@dataclass
class JudgeScore:
    question_id: str
    accuracy: float       # 1-5
    completeness: float   # 1-5
    relevance: float      # 1-5
    reasoning: str
    weighted_score: float = 0.0  # 综合加权分，由权重配置计算

    def __post_init__(self):
        # 默认权重：准确性40%，完整性30%，相关性30%
        self.weighted_score = (
            self.accuracy * 0.4
            + self.completeness * 0.3
            + self.relevance * 0.3
        )


@dataclass
class EvalSample:
    question_id: str
    question: str
    ground_truth: str
    ai_answer: str


class LLMJudge:
    """
    LLM-as-Judge 评估器。
    使用更强的模型（judge_model）评估业务系统的输出质量。
    """

    def __init__(
        self,
        judge_model: str = "gpt-4o",  # 用更强的模型做评判
        api_key: Optional[str] = None,
    ):
        self.model = judge_model
        self.client = AsyncOpenAI(api_key=api_key)

    async def evaluate(
        self,
        question: str,
        ground_truth: str,
        ai_answer: str,
        question_id: str = "unknown",
    ) -> JudgeScore:
        """
        单个样本评估。
        返回 JudgeScore，包含各维度分数和推理。
        """
        prompt = JUDGE_PROMPT.format(
            question=question,
            ground_truth=ground_truth,
            ai_answer=ai_answer,
        )
        response = await self.client.chat.completions.create(
            model=self.model,
            messages=[{"role": "user", "content": prompt}],
            temperature=0.0,  # 评估任务要求确定性输出
            response_format={"type": "json_object"},
        )

        raw = response.choices[0].message.content
        try:
            scores = json.loads(raw)
            return JudgeScore(
                question_id=question_id,
                accuracy=float(scores["accuracy"]),
                completeness=float(scores["completeness"]),
                relevance=float(scores["relevance"]),
                reasoning=scores.get("reasoning", ""),
            )
        except (json.JSONDecodeError, KeyError) as e:
            # 解析失败时给最低分并记录原因，不让整个流水线崩溃
            return JudgeScore(
                question_id=question_id,
                accuracy=1.0,
                completeness=1.0,
                relevance=1.0,
                reasoning=f"[评估解析失败: {e}] 原始输出: {raw[:100]}",
            )

    async def batch_evaluate(
        self,
        samples: list[EvalSample],
        concurrency: int = 10,
    ) -> list[JudgeScore]:
        """
        并行批量评估。
        concurrency 控制并发数，防止触发API速率限制。
        """
        semaphore = asyncio.Semaphore(concurrency)

        async def evaluate_with_semaphore(sample: EvalSample) -> JudgeScore:
            async with semaphore:
                return await self.evaluate(
                    question=sample.question,
                    ground_truth=sample.ground_truth,
                    ai_answer=sample.ai_answer,
                    question_id=sample.question_id,
                )

        tasks = [evaluate_with_semaphore(s) for s in samples]
        results = await asyncio.gather(*tasks, return_exceptions=True)

        # 过滤掉异常，替换为低分占位
        final_results = []
        for i, r in enumerate(results):
            if isinstance(r, Exception):
                print(f"[WARN] 样本 {samples[i].question_id} 评估异常: {r}")
                final_results.append(JudgeScore(
                    question_id=samples[i].question_id,
                    accuracy=1.0, completeness=1.0, relevance=1.0,
                    reasoning=f"评估异常: {r}",
                ))
            else:
                final_results.append(r)
        return final_results
```

## RAGAS：RAG系统的专项评估

除了LLM-as-Judge，老吴还给陈浩介绍了另一套工具——RAGAS（RAG Assessment）。

"LLM-as-Judge是通用的——任何AI输出都可以用它评估。但RAG系统有它特有的故障模式，需要专项指标。"

老吴在白板上写了四个RAGAS核心指标：

**Faithfulness（忠实度）**：AI的回答是否严格基于检索到的上下文？它有没有说上下文里没有的话？这个指标专门检测幻觉。得分0-1，1表示完全忠实于上下文。

**Answer Relevancy（答案相关性）**：生成的答案是否真正回答了用户的问题？即使答案忠实于文档，但文档根本没讲到问题要问的事，那答案也是无关的。

**Context Recall（上下文召回）**：给定标准答案，检索到的文档是否包含了生成正确答案所需的全部信息？Context Recall低意味着检索系统漏掉了关键文档——这不是生成的问题，是检索的问题。

**Context Precision（上下文精度）**：检索到的文档中，有多少是真正相关的？这衡量检索的精确性——召回了100个文档，但只有3个是有用的，这就是低精度。

"这四个指标一起，能帮你诊断RAG系统的问题出在哪个环节。如果Faithfulness低，问题在生成；如果Context Recall低，问题在检索；如果Context Precision低，可能是检索策略太宽泛，召回了太多噪声。"

老吴说，RAGAS已经有成熟的开源实现（`pip install ragas`），集成起来并不复杂：

## 完整评估流水线

零件都有了。陈浩现在需要把它们拼成一个自动化的流水线。

```python
import asyncio
from dataclasses import dataclass, field
from datetime import datetime
from typing import Optional


@dataclass
class EvalResult:
    """单个样本的完整评估结果"""
    sample: GoldenSample
    ai_answer: str
    retrieved_docs: list[str]
    judge_score: JudgeScore
    ragas_score: Optional[RagasScore]
    latency_ms: float  # 系统响应时间（毫秒）


@dataclass
class EvalReport:
    """完整评估报告"""
    version_tag: str
    timestamp: datetime
    total_samples: int

    # 核心指标（均值）
    accuracy_mean: float
    faithfulness_mean: float
    answer_relevancy_mean: float
    context_recall_mean: float
    weighted_score_mean: float

    # 按问题类型分解
    by_type: dict[str, dict]

    # 延迟统计
    latency_p50_ms: float
    latency_p95_ms: float

    # 失败案例（得分最低的10个）
    worst_samples: list[EvalResult]

    @classmethod
    def from_results(cls, results: list[EvalResult], version_tag: str) -> "EvalReport":
        """从原始结果列表计算汇总报告"""
        import statistics

        scores = [r.judge_score for r in results]
        latencies = sorted([r.latency_ms for r in results])

        # 按问题类型分组
        by_type: dict[str, dict] = {}
        for r in results:
            qt = r.sample.question_type.value
            if qt not in by_type:
                by_type[qt] = {"count": 0, "accuracy_sum": 0.0, "relevance_sum": 0.0}
            by_type[qt]["count"] += 1
            by_type[qt]["accuracy_sum"] += r.judge_score.accuracy
            by_type[qt]["relevance_sum"] += r.judge_score.relevance

        for qt, data in by_type.items():
            n = data["count"]
            data["accuracy_mean"] = round(data["accuracy_sum"] / n, 3)
            data["relevance_mean"] = round(data["relevance_sum"] / n, 3)

        # 按得分排序，取最低10个
        sorted_by_score = sorted(results, key=lambda r: r.judge_score.weighted_score)
        worst_10 = sorted_by_score[:10]

        # 统计ragas分数（如果有）
        ragas_results = [r for r in results if r.ragas_score is not None]
        faithfulness_mean = (
            statistics.mean([r.ragas_score.faithfulness for r in ragas_results])
            if ragas_results else 0.0
        )
        answer_relevancy_mean = (
            statistics.mean([r.ragas_score.answer_relevancy for r in ragas_results])
            if ragas_results else 0.0
        )
        context_recall_mean = (
            statistics.mean([r.ragas_score.context_recall for r in ragas_results])
            if ragas_results else 0.0
        )

        n = len(latencies)
        return cls(
            version_tag=version_tag,
            timestamp=datetime.now(),
            total_samples=len(results),
            accuracy_mean=round(statistics.mean([s.accuracy for s in scores]), 3),
            faithfulness_mean=round(faithfulness_mean, 3),
            answer_relevancy_mean=round(answer_relevancy_mean, 3),
            context_recall_mean=round(context_recall_mean, 3),
            weighted_score_mean=round(statistics.mean([s.weighted_score for s in scores]), 3),
            by_type=by_type,
            latency_p50_ms=round(latencies[n // 2], 1) if latencies else 0.0,
            latency_p95_ms=round(latencies[int(n * 0.95)], 1) if latencies else 0.0,
            worst_samples=worst_10,
        )

    def to_markdown(self) -> str:
        """生成Markdown格式的评估报告"""
        lines = [
            f"# AI系统评估报告",
            f"",
            f"**版本标签**：`{self.version_tag}`",
            f"**评估时间**：{self.timestamp.strftime('%Y-%m-%d %H:%M')}",
            f"**样本总数**：{self.total_samples}",
            f"",
            f"## 核心指标",
            f"",
            f"| 指标 | 分数 |",
            f"|------|------|",
            f"| 准确性（LLM Judge） | {self.accuracy_mean:.3f} / 5.0 |",
            f"| 忠实度（RAGAS） | {self.faithfulness_mean:.3f} |",
            f"| 答案相关性（RAGAS） | {self.answer_relevancy_mean:.3f} |",
            f"| 上下文召回（RAGAS） | {self.context_recall_mean:.3f} |",
            f"| 综合加权分 | **{self.weighted_score_mean:.3f}** |",
            f"",
            f"## 延迟统计",
            f"",
            f"- P50：{self.latency_p50_ms:.0f}ms",
            f"- P95：{self.latency_p95_ms:.0f}ms",
            f"",
            f"## 按问题类型分解",
            f"",
            f"| 类型 | 样本数 | 准确性均值 | 相关性均值 |",
            f"|------|--------|-----------|-----------|",
        ]
        for qt, data in self.by_type.items():
            lines.append(
                f"| {qt} | {data['count']} | {data['accuracy_mean']:.3f} | {data['relevance_mean']:.3f} |"
            )
        lines += [
            f"",
            f"## 最差10个样本（需人工复查）",
            f"",
        ]
        for i, r in enumerate(self.worst_samples, 1):
            lines.append(
                f"{i}. [{r.sample.question_id}] **{r.sample.question[:40]}...** "
                f"得分:{r.judge_score.weighted_score:.2f} 原因:{r.judge_score.reasoning[:30]}"
            )
        return "\n".join(lines)

    def compare_with(self, baseline: "EvalReport") -> dict:
        """与历史版本对比，显示提升/退步"""
        def diff(new: float, old: float) -> str:
            d = new - old
            sign = "+" if d >= 0 else ""
            return f"{sign}{d:.3f}"

        return {
            "accuracy": diff(self.accuracy_mean, baseline.accuracy_mean),
            "faithfulness": diff(self.faithfulness_mean, baseline.faithfulness_mean),
            "answer_relevancy": diff(self.answer_relevancy_mean, baseline.answer_relevancy_mean),
            "context_recall": diff(self.context_recall_mean, baseline.context_recall_mean),
            "weighted_score": diff(self.weighted_score_mean, baseline.weighted_score_mean),
            "summary": (
                "IMPROVED" if self.weighted_score_mean > baseline.weighted_score_mean
                else "REGRESSED"
            ),
        }


class AIEvaluationPipeline:
    """
    完整的AI评估流水线。
    输入：黄金测试集 + 被评估的RAG管道。
    输出：EvalReport，包含各维度指标、分类分析和失败案例。
    """

    def __init__(
        self,
        rag_pipeline,            # 被评估的RAG管道（需实现 query_with_sources 方法）
        judge: LLMJudge,
        dataset: GoldenDataset,
        ragas_evaluator: Optional[RagasEvaluator] = None,
    ):
        self.rag_pipeline = rag_pipeline
        self.judge = judge
        self.dataset = dataset
        self.ragas_evaluator = ragas_evaluator

    async def run(self, version_tag: str = "current") -> EvalReport:
        """运行完整评估流水线"""
        import time

        results: list[EvalResult] = []

        for sample in self.dataset.samples:
            # 计时运行被评估的RAG系统
            start_ms = time.monotonic() * 1000
            ai_answer, retrieved_docs = await self.rag_pipeline.query_with_sources(
                sample.question
            )
            latency_ms = time.monotonic() * 1000 - start_ms

            # LLM-as-Judge评估
            judge_score = await self.judge.evaluate(
                question=sample.question,
                ground_truth=sample.ground_truth,
                ai_answer=ai_answer,
                question_id=sample.question_id,
            )

            # RAGAS评估（可选）
            ragas_score = None
            if self.ragas_evaluator is not None:
                try:
                    ragas_score = await self.ragas_evaluator.evaluate(
                        question=sample.question,
                        ai_answer=ai_answer,
                        retrieved_docs=retrieved_docs,
                        ground_truth=sample.ground_truth,
                        question_id=sample.question_id,
                    )
                except Exception as e:
                    print(f"[WARN] RAGAS评估失败 {sample.question_id}: {e}")

            results.append(EvalResult(
                sample=sample,
                ai_answer=ai_answer,
                retrieved_docs=retrieved_docs,
                judge_score=judge_score,
                ragas_score=ragas_score,
                latency_ms=latency_ms,
            ))
            print(f"[{len(results)}/{len(self.dataset)}] {sample.question_id}: "
                  f"accuracy={judge_score.accuracy:.1f} latency={latency_ms:.0f}ms")

        report = EvalReport.from_results(results, version_tag)
        return report
```

## A/B测试框架：Prompt变更的科学验证

一周后，陈浩想优化检索策略，把原来的余弦相似度Top-5改为带重排序的Top-10。他直觉上觉得这样能提升质量，但……

"我改了检索策略，怎么知道变好了？"

老吴说："跑A/B。"

"怎么跑？"

"把旧版本和新版本都跑一遍同一批测试集，对比他们的评估分数，看哪个赢。就这么简单——但要注意，样本要固定。用同样的50个样本测两个版本，才有可比性。"

```python
import asyncio
from dataclasses import dataclass
from typing import Callable


@dataclass
class VariantResult:
    """单个版本的评估结果汇总"""
    variant_name: str
    accuracy_mean: float
    faithfulness_mean: float
    answer_relevancy_mean: float
    context_recall_mean: float
    weighted_score_mean: float
    sample_size: int


@dataclass
class ABTestResult:
    """A/B测试完整结果"""
    variant_a: VariantResult
    variant_b: VariantResult
    winner: str              # "A" / "B" / "TIE"
    delta: dict[str, float]  # 各指标的差值（B-A）
    significant: bool        # 是否具有统计显著性


class PromptABTest:
    """
    AI系统A/B测试框架。
    科学对比两个版本（Prompt变更/检索策略变更/模型变更）。
    """

    def __init__(self, judge: LLMJudge):
        self.judge = judge

    async def _eval_variant(
        self,
        variant_fn: Callable,       # 异步函数，接受question，返回(ai_answer, retrieved_docs)
        dataset: GoldenDataset,
        variant_name: str,
    ) -> tuple[list[JudgeScore], VariantResult]:
        import statistics

        scores: list[JudgeScore] = []
        for sample in dataset.samples:
            ai_answer, _ = await variant_fn(sample.question)
            score = await self.judge.evaluate(
                question=sample.question,
                ground_truth=sample.ground_truth,
                ai_answer=ai_answer,
                question_id=sample.question_id,
            )
            scores.append(score)

        return scores, VariantResult(
            variant_name=variant_name,
            accuracy_mean=statistics.mean([s.accuracy for s in scores]),
            faithfulness_mean=0.0,       # 需接入ragas才能填充
            answer_relevancy_mean=0.0,
            context_recall_mean=0.0,
            weighted_score_mean=statistics.mean([s.weighted_score for s in scores]),
            sample_size=len(scores),
        )

    def _compute_significance(
        self,
        scores_a: list[JudgeScore],
        scores_b: list[JudgeScore],
        threshold: float = 0.05,
    ) -> bool:
        """
        用t检验判断两组分数是否有统计显著性差异。
        p值 < threshold 则认为显著。
        注意：样本量至少30个才可靠，50个更好。
        """
        try:
            from scipy import stats
            t_stat, p_value = stats.ttest_rel(
                [s.weighted_score for s in scores_a],
                [s.weighted_score for s in scores_b],
            )
            return bool(p_value < threshold)
        except ImportError:
            # scipy未安装时，退回简单的差值判断
            import statistics
            mean_a = statistics.mean([s.weighted_score for s in scores_a])
            mean_b = statistics.mean([s.weighted_score for s in scores_b])
            return abs(mean_b - mean_a) > 0.1  # 超过0.1分差视为显著

    async def compare(
        self,
        variant_a_fn: Callable,
        variant_b_fn: Callable,
        dataset: GoldenDataset,
        sample_size: int = 50,
        variant_a_name: str = "baseline",
        variant_b_name: str = "experiment",
    ) -> ABTestResult:
        """
        科学对比两个版本。
        dataset会被固定种子采样，保证A/B用同一批样本。
        """
        sampled = dataset.sample(sample_size, seed=42)  # 固定种子

        # 并行运行两个版本
        (scores_a, result_a), (scores_b, result_b) = await asyncio.gather(
            self._eval_variant(variant_a_fn, sampled, variant_a_name),
            self._eval_variant(variant_b_fn, sampled, variant_b_name),
        )

        # 计算各指标差值（B - A）
        delta = {
            "weighted_score": round(result_b.weighted_score_mean - result_a.weighted_score_mean, 3),
            "accuracy": round(result_b.accuracy_mean - result_a.accuracy_mean, 3),
        }

        # 判断胜者
        if abs(delta["weighted_score"]) < 0.05:
            winner = "TIE"
        elif delta["weighted_score"] > 0:
            winner = "B"
        else:
            winner = "A"

        is_significant = self._compute_significance(scores_a, scores_b)

        return ABTestResult(
            variant_a=result_a,
            variant_b=result_b,
            winner=winner,
            delta=delta,
            significant=is_significant,
        )
```

```python
#!/usr/bin/env python3
"""
scripts/check_quality_gate.py
CI/CD质量门禁检查脚本。
从baseline文件加载历史基准，对比本次评估结果。
超出退步阈值则以非零退出码退出（触发CI失败）。
"""
import json
import sys
import argparse
from pathlib import Path


def load_report(path: str) -> dict:
    with open(path) as f:
        return json.load(f)


def check_gate(report: dict, args: argparse.Namespace) -> bool:
    passed = True
    issues = []

    # 绝对值门禁
    if report["accuracy_mean"] < args.min_accuracy:
        issues.append(
            f"准确性 {report['accuracy_mean']:.3f} < 最低要求 {args.min_accuracy}"
        )
        passed = False

    if report.get("faithfulness_mean", 1.0) < args.min_faithfulness:
        issues.append(
            f"忠实度 {report['faithfulness_mean']:.3f} < 最低要求 {args.min_faithfulness}"
        )
        passed = False

    # 相对退步门禁（需要baseline文件）
    baseline_path = Path("eval_baseline.json")
    if baseline_path.exists():
        baseline = load_report(str(baseline_path))
        regression = baseline["weighted_score_mean"] - report["weighted_score_mean"]
        if regression > args.max_regression:
            issues.append(
                f"综合分退步 {regression:.3f} 超过允许阈值 {args.max_regression}"
            )
            passed = False

    if issues:
        print("❌ 质量门禁未通过：")
        for issue in issues:
            print(f"  - {issue}")
    else:
        print("✅ 质量门禁通过")

    return passed


def main():
    parser = argparse.ArgumentParser()
    parser.add_argument("report_path")
    parser.add_argument("--min-accuracy", type=float, default=3.5)
    parser.add_argument("--min-faithfulness", type=float, default=0.80)
    parser.add_argument("--max-regression", type=float, default=0.05)
    args = parser.parse_args()

    report = load_report(args.report_path)
    if not check_gate(report, args):
        sys.exit(1)  # 非零退出码触发CI失败


if __name__ == "__main__":
    main()
```

读完本章，你应该能做到：

**1. 建立黄金测试集（Golden Dataset）**
- 从真实用户问题中挑选200+个样本，写上标准答案

- 用比业务系统更强的模型做评判（比如用GPT-4o评GPT-4o-mini）
- 理解并规避三大偏见：Position Bias / Verbosity Bias / Self-Preference
- 并发10个请求，200个样本评估在一分钟内完成

**3. 把评估集成进CI/CD**
- 每次Prompt/配置变更自动触发评估
- 用质量门禁（Quality Gate）阻断退步：综合分比baseline差超过阈值则PR失败
- 评估结果自动评论到PR，让团队每次都知道改了什么

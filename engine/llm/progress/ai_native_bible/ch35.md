# 第三十五章：数据飞轮：让AI随用随强

## 一、两个AI，两种命运

三月初的第一个周一早上，陈浩一进公司就感觉到某种奇怪的对称。

他同时负责两个AI系统：报销审批AI和客服助手AI。这两个系统是同一个月上线的，用的是同一套底层框架，也是他亲手从零搭建起来的。但一年多过去，两个系统走向了完全不同的轨道。

他拉出Grafana的监控面板，把两条曲线放到了同一个屏幕上。

报销AI的准确率曲线，从第一个月的71%出发，像一条缓慢上升的斜线，现在已经到了92%。每个月大约涨1.5个百分点，不快，但稳。

客服AI的曲线，是一条几乎水平的直线。78%。整整13个月，没有大的起伏。有时候某个版本更新之后涨到79%，然后过几周又滑回78%。像一个人在原地踏步，努力但没有进展。

陈浩看着这两条曲线，盯了很久。

"为什么？"

不是在问旁边的人——工位区这个时候还很安静，林晓还没来，李哥好像在开会。他是在问自己。这两个系统，技术栈一样，团队一样，维护频率一样。唯一不同的是什么？

他开始回忆这两个系统的运转方式。

报销AI：每一张报销单提交后，审批员会做一个决定——通过或者拒绝。这个决定会被记录到数据库里。审批员的批注，比如"金额超标"、"缺少发票"、"不符合差旅规定"，也都会保存下来。这些数据，每周自动跑一次fine-tuning任务，更新模型。报销AI每周都在"看"真实的人类判断，它知道哪次自己猜对了，哪次猜错了。

客服AI：用户问了问题，AI给了回答。然后——什么都没有。用户离开了。系统里只有一条聊天记录，没有对这条记录的任何评价，没有人说这个回答是对的还是错的，没有标签，没有信号。

陈浩在笔记本上写下一行字：

*"报销AI有反馈，客服AI没有反馈。"*

他把这行字圈起来，在旁边写了一个问号。

---

中午，他找到了AI导师方明。

方明是那种很少在公司露面的人。他不在任何组织架构里，但技术总监偶尔会叫他来做评审，这个圈子里的人都知道他是真正懂AI工程的那批人之一。这次陈浩发消息给他，说自己遇到了一个想不清楚的问题。方明的回复很干脆："你说的问题我也想聊。下午两点，咖啡馆那边。"

两点钟，陈浩带着那张写了两条曲线的截图坐在方明对面。

方明看了一眼图，没有立刻说话。他点了一杯黑咖啡，等咖啡送来，喝了一口，才开口。

"你知道为什么亚马逊的推荐系统越来越准吗？"

"因为数据多？" 陈浩说了一个他自己不太确定的答案。

"不是数据多。" 方明摇头，"是因为每一次用户点击，系统都知道了一件事：'这个推荐是对的还是错的'。购买=对，忽略=信号弱，退货=错。每一次交互都在给模型一个反馈。这个循环不停在转。"

方明在纸巾上画了一个圆圈，圆圈里有三个节点：**使用→反馈→改进**，然后箭头连回使用。

"这就是数据飞轮。飞轮转得越快，模型越强。飞轮不转，模型就永远停在那里。"

陈浩盯着那个圆圈看。

"你的报销AI，飞轮在转。你的客服AI，飞轮没有转。"

"可是客服的反馈怎么收？用户又不会专门去点'这个回答有没有帮助'。" 陈浩说出了他一直想说的那个卡点。

方明把纸巾推回来，在"反馈"那个节点旁边写了两个字：**隐式**。

"你不需要用户主动告诉你。用户的行为就是反馈。"

## 二、用户行为就是信号

方明接着说的话，让陈浩重新想了一遍这一年来所有的客服聊天记录。

"用户问了一个问题，AI回答了。然后用户做了什么？"

陈浩想了一下："离开了，或者继续问下一个问题。"

"继续问了什么？"

"有时候问别的事，有时候——" 陈浩停了一下，"有时候用同样的问题再问一遍，或者换了个角度再问。"

方明点头。"这就是信号。用户追问，说明上一条回答没有解决问题。用户重复提问，说明AI没有理解用户的意思。这些不需要用户手动打分，从行为里就能读出来。"

他在纸上列出了几个行为信号：

1. **二次追问**：AI回答后30秒内，用户发出了第二条消息
2. **Escalation（升级）**：用户在对话中触发了"转人工"操作
3. **会话中断**：AI回答后，用户突然离开，会话无自然结束
4. **重复提问**：用户在同一个会话里问了语义相似的问题超过两次
5. **负面关键词**：用户回复里出现"不对"、"这不是我想问的"、"你没理解"

"这五个信号，每一个都是软标签。不是100%确定这次回答失败了，但都是负向信号的概率很高。"

陈浩在笔记本上飞快地记下来。他脑子里已经开始想怎么实现这个。

"那正向信号呢？"

"正向的更简单。用户得到回答之后，没有追问，没有转人工，会话以自然结束收尾，甚至接下来又发起了新的咨询——这些都是正向信号。对话结束后的评分（1-5星）如果有的话也是，但别依赖它，大部分用户不会点。"

陈浩当天晚上没有立刻开始写代码。他先翻出了过去三个月所有的客服聊天记录，大概两万七千条会话，开始按照方明给的五个信号手动分析了一百条。

结果让他有点震动。

在他随机看的一百条会话里，有31条显示了"二次追问"，有14条发生了Escalation，有9条用户在回复里用了明确的否定词。合计：54条会话有至少一个负向信号，占比54%。

客服AI真实的用户满意度，远比78%的"准确率"难看得多。

他写下第二行字：

*"78%是我们测试集上的分数，不是用户真实体验的分数。"*

## 三、建造信号收集系统

第二天早上，陈浩打开了一个新文件，开始写隐式反馈收集系统。他的目标很明确：把用户的行为信号，转化成可以用于训练的弱标签。

```python
# feedback_collector.py
# 隐式反馈信号收集系统
# 将用户行为转化为可用于训练的弱标签

from dataclasses import dataclass, field
from datetime import datetime, timedelta
from enum import Enum
from typing import Optional
import json
import logging

logger = logging.getLogger(__name__)


class FeedbackSignal(Enum):
    """反馈信号类型"""
    IMPLICIT_NEGATIVE = "implicit_negative"   # 隐式负向：追问、escalation等
    IMPLICIT_POSITIVE = "implicit_positive"   # 隐式正向：自然结束、再次咨询
    EXPLICIT_POSITIVE = "explicit_positive"   # 显式正向：点赞、高分评分
    EXPLICIT_NEGATIVE = "explicit_negative"   # 显式负向：踩、低分评分
    UNKNOWN = "unknown"                        # 无法判断


@dataclass
class ConversationTurn:
    """单条对话记录"""
    turn_id: str
    session_id: str
    user_message: str
    ai_response: str
    timestamp: datetime
    response_time_ms: int           # AI响应耗时
    user_next_action: Optional[str] = None   # 用户下一个动作
    next_message_delay_seconds: Optional[float] = None  # 距下一条消息的秒数


@dataclass
class ImplicitFeedback:
    """从用户行为中提取的隐式反馈"""
    turn_id: str
    session_id: str
    signal: FeedbackSignal
    confidence: float               # 信号置信度 0.0-1.0
    signal_types: list[str]         # 触发了哪些信号
    user_message: str               # 用户原始问题
    ai_response: str                # AI回答
    extracted_at: datetime = field(default_factory=datetime.utcnow)
    
    def to_training_sample(self) -> Optional[dict]:
        """转换为训练样本格式（只有置信度高的才转）"""
        if self.confidence < 0.6:
            return None
        
        label = 1 if self.signal == FeedbackSignal.IMPLICIT_POSITIVE else 0
        return {
            "input": self.user_message,
            "output": self.ai_response,
            "label": label,
            "confidence": self.confidence,
            "source": "implicit_feedback",
            "signal_types": self.signal_types
        }


class ImplicitFeedbackExtractor:
    """
    从用户行为中提取隐式反馈信号
    
    核心逻辑：
    - 负向信号：追问（30秒内发第二条）、escalation、负面关键词、重复提问
    - 正向信号：自然结束、无追问、再次咨询
    """
    
    # 负向关键词（用户表达不满/AI没理解）
    NEGATIVE_KEYWORDS = [
        "不对", "不是", "你没理解", "这不是我想问的",
        "重新说", "换个方式", "算了", "没用", "你说的不对",
        "不是这个意思", "没有帮助", "你理解错了"
    ]
    
    # 追问的时间阈值（秒）
    QUICK_FOLLOWUP_THRESHOLD = 45
    
    def __init__(self, db_session):
        self.db = db_session
    
    def extract_feedback_for_turn(
        self, 
        turn: ConversationTurn,
        subsequent_turns: list[ConversationTurn],
        session_ended_naturally: bool,
        escalation_happened: bool
    ) -> ImplicitFeedback:
        """
        对单条对话的AI回答进行信号分析
        
        Args:
            turn: 当前对话轮次
            subsequent_turns: 当前轮次之后的对话（同一会话内）
            session_ended_naturally: 会话是否自然结束（用户主动结束 vs 超时断开）
            escalation_happened: 会话后续是否发生了escalation（转人工）
        """
        signals_triggered = []
        confidence_adjustments = []
        
        # 信号1：快速追问（最强的负向信号）
        if subsequent_turns:
            next_turn = subsequent_turns[0]
            if (next_turn.next_message_delay_seconds is not None and
                    next_turn.next_message_delay_seconds < self.QUICK_FOLLOWUP_THRESHOLD):
                signals_triggered.append("quick_followup")
                confidence_adjustments.append(0.75)  # 高置信度负向
        
        # 信号2：escalation（最强的失败信号）
        if escalation_happened:
            signals_triggered.append("escalation")
            confidence_adjustments.append(0.9)   # 极高置信度负向
        
        # 信号3：用户回复中的负面关键词
        if subsequent_turns:
            next_user_message = subsequent_turns[0].user_message.lower()
            matched_keywords = [
                kw for kw in self.NEGATIVE_KEYWORDS 
                if kw in next_user_message
            ]
            if matched_keywords:
                signals_triggered.append(f"negative_keywords:{','.join(matched_keywords)}")
                confidence_adjustments.append(0.85)
        
        # 信号4：重复提问（语义相似的问题在同一会话出现2次以上）
        if self._detect_repeated_question(turn, subsequent_turns):
            signals_triggered.append("repeated_question")
            confidence_adjustments.append(0.7)
        
        # 正向信号：会话自然结束，无追问
        is_positive = (
            session_ended_naturally and
            not escalation_happened and
            len(signals_triggered) == 0 and
            (not subsequent_turns or  # 这是最后一轮
             (subsequent_turns and 
              subsequent_turns[0].next_message_delay_seconds and
              subsequent_turns[0].next_message_delay_seconds > 120))  # 下一条消息超过2分钟
        )
        
        if is_positive:
            signals_triggered.append("natural_session_end")
            confidence_adjustments.append(0.65)  # 正向置信度相对低（沉默不代表满意）
        
        # 计算最终信号和置信度
        if not signals_triggered:
            return ImplicitFeedback(
                turn_id=turn.turn_id,
                session_id=turn.session_id,
                signal=FeedbackSignal.UNKNOWN,
                confidence=0.0,
                signal_types=[],
                user_message=turn.user_message,
                ai_response=turn.ai_response
            )
        
        if is_positive and len(signals_triggered) == 1:
            final_signal = FeedbackSignal.IMPLICIT_POSITIVE
            final_confidence = confidence_adjustments[0]
        else:
            # 有任何负向信号，判为负向
            final_signal = FeedbackSignal.IMPLICIT_NEGATIVE
            # 多个信号叠加，取最高置信度，但额外叠加
            neg_confidences = [c for c in confidence_adjustments if c >= 0.7]
            if neg_confidences:
                final_confidence = min(0.95, max(neg_confidences) + 0.05 * (len(neg_confidences) - 1))
            else:
                final_confidence = max(confidence_adjustments)
        
        return ImplicitFeedback(
            turn_id=turn.turn_id,
            session_id=turn.session_id,
            signal=final_signal,
            confidence=final_confidence,
            signal_types=signals_triggered,
            user_message=turn.user_message,
            ai_response=turn.ai_response
        )
    
    def _detect_repeated_question(
        self, 
        current_turn: ConversationTurn,
        subsequent_turns: list[ConversationTurn]
    ) -> bool:
        """
        检测重复提问：后续对话中是否有语义相似的问题
        （简化版：基于关键词重叠，生产环境应使用向量相似度）
        """
        if not subsequent_turns:
            return False
        
        current_words = set(current_turn.user_message.replace("？", "").replace("?", "").split())
        
        for sub_turn in subsequent_turns[:3]:  # 只看后3条
            sub_words = set(sub_turn.user_message.split())
            if not current_words or not sub_words:
                continue
            overlap = len(current_words & sub_words) / len(current_words | sub_words)
            if overlap > 0.5:  # 50%以上词汇重叠
                return True
        
        return False


class FeedbackPipeline:
    """
    完整的反馈收集流水线
    
    运行周期：每小时或每天批量处理
    """
    
    def __init__(self, db_session, storage_backend):
        self.db = db_session
        self.storage = storage_backend
        self.extractor = ImplicitFeedbackExtractor(db_session)
    
    def process_sessions_batch(
        self, 
        since: datetime,
        until: datetime
    ) -> dict:
        """批量处理时间段内的所有会话，提取反馈信号"""
        
        sessions = self.db.query_sessions(since=since, until=until)
        
        results = {
            "total_sessions": len(sessions),
            "total_turns_processed": 0,
            "positive_signals": 0,
            "negative_signals": 0,
            "unknown_signals": 0,
            "high_confidence_samples": 0,  # 置信度>=0.7，可直接用于训练
        }
        
        training_samples = []
        
        for session in sessions:
            turns = sorted(session.turns, key=lambda t: t.timestamp)
            session_ended_naturally = session.end_reason == "user_initiated"
            escalation_happened = any(t.user_next_action == "escalate" for t in turns)
            
            for i, turn in enumerate(turns):
                subsequent = turns[i+1:]
                
                feedback = self.extractor.extract_feedback_for_turn(
                    turn=turn,
                    subsequent_turns=subsequent,
                    session_ended_naturally=session_ended_naturally,
                    escalation_happened=escalation_happened
                )
                
                results["total_turns_processed"] += 1
                
                if feedback.signal == FeedbackSignal.IMPLICIT_POSITIVE:
                    results["positive_signals"] += 1
                elif feedback.signal == FeedbackSignal.IMPLICIT_NEGATIVE:
                    results["negative_signals"] += 1
                else:
                    results["unknown_signals"] += 1
                
                # 高置信度样本进入训练候选池
                sample = feedback.to_training_sample()
                if sample:
                    results["high_confidence_samples"] += 1
                    training_samples.append(sample)
        
        # 保存到候选池
        if training_samples:
            self.storage.save_candidates(training_samples)
            logger.info(
                f"反馈处理完成: 共 {results['total_turns_processed']} 条对话, "
                f"产生 {len(training_samples)} 个高质量训练候选样本"
            )
        
        return results
```

## 四、不是所有数据都值得标注

第二天早上，他把问题带给了方明。

"我昨天建好了信号收集，跑了一下历史数据，三个月里大概能产生7000多个候选样本。"

方明皱了一下眉："7000个，你打算全部标注？"

"不是自动标注吗？用信号……"

"信号是弱标签。" 方明打断他，"弱标签用来过滤，不能直接拿去训练分类模型。你说的'候选样本'，是还需要人工确认的。7000个，如果每个标注要5分钟，你需要多少时间？"

陈浩算了一下："583小时。差不多24天。"

"那你准备雇多少标注员？"

陈浩没说话。

"所以问题来了：7000个样本里，哪些最值得标注？"

这是个陈浩没想过的问题。他一直默认"更多数据=更好"，但方明在问的是：在资源有限的情况下，哪些数据对模型的提升最大？

方明拿出一张纸，画了一个坐标轴：横轴是"模型对这个样本的置信度"，纵轴是"标注这个样本的收益"。

"模型已经很确定的样本，标注它能学到什么？一个模型对答案99%确信的例子，你告诉它'对，你猜对了'，它能学到多少？接近于零。"

"但是……" 陈浩开始想，"如果模型对一个样本50%、50%拿不定主意——"

"那个样本才是金子。" 方明点了一下那张纸的中间区域，"不确定的地方，就是模型的知识边界。在边界上标注，才能真正拓展模型的能力。这就是主动学习的核心。"

方明把这个思路解释得很清楚：与其随机抽取样本标注，不如**主动选择模型最不确定的样本**。数学上，用模型对各个类别的预测概率分布来衡量不确定性：分布越"均匀"，不确定性越高。

"有研究表明，" 方明说，"在某些任务上，200个主动选择的样本，顶得上1000个随机样本。"

陈浩把这句话记下来，然后开始写第二段代码。

```python
# active_learning.py
# 主动学习样本选择
# 选择模型最"不确定"的样本优先标注，降低标注成本

import numpy as np
from dataclasses import dataclass
from typing import Optional
import logging

logger = logging.getLogger(__name__)


@dataclass
class SampleWithUncertainty:
    """带不确定性分数的样本"""
    sample_id: str
    user_message: str
    ai_response: str
    
    # 当前模型的预测
    predicted_label: int           # 0=负向, 1=正向
    class_probabilities: list[float]  # [P(负向), P(正向)]
    
    # 不确定性指标
    entropy: float                 # 信息熵（越高=越不确定）
    margin: float                  # 最高概率与次高概率之差（越小=越不确定）
    confidence: float              # 最高类别的概率（越低=越不确定）
    
    # 综合不确定性分数（越高=越值得标注）
    uncertainty_score: float = 0.0
    
    # 来源信息
    implicit_signal: Optional[str] = None  # 触发的隐式信号
    session_id: Optional[str] = None


class UncertaintySampler:
    """
    不确定性采样器
    
    三种不确定性度量：
    1. 最小置信度（Least Confidence）：1 - max(P)
    2. 边际采样（Margin Sampling）：P(最高) - P(次高)  
    3. 信息熵（Entropy Sampling）：-sum(P * log(P))
    
    生产中推荐组合使用三种度量
    """
    
    def calculate_entropy(self, probs: list[float]) -> float:
        """计算信息熵"""
        probs_array = np.array(probs)
        # 避免log(0)
        probs_array = np.clip(probs_array, 1e-10, 1.0)
        return float(-np.sum(probs_array * np.log(probs_array)))
    
    def calculate_margin(self, probs: list[float]) -> float:
        """计算边际：最高概率与次高概率之差（越小越不确定）"""
        sorted_probs = sorted(probs, reverse=True)
        if len(sorted_probs) < 2:
            return sorted_probs[0]
        return sorted_probs[0] - sorted_probs[1]
    
    def calculate_uncertainty_score(
        self, 
        probs: list[float],
        entropy_weight: float = 0.5,
        margin_weight: float = 0.3,
        confidence_weight: float = 0.2
    ) -> float:
        """
        综合不确定性分数（0-1，越高越需要标注）
        
        Args:
            probs: 各类别的预测概率列表
            entropy_weight: 信息熵的权重
            margin_weight: 边际分数的权重  
            confidence_weight: 置信度的权重
        """
        max_entropy = np.log(len(probs))  # 均匀分布时的最大熵
        
        entropy = self.calculate_entropy(probs)
        margin = self.calculate_margin(probs)
        confidence = max(probs)
        
        # 归一化到[0,1]
        normalized_entropy = entropy / max_entropy if max_entropy > 0 else 0
        normalized_margin_uncertainty = 1 - margin  # 边际越小，不确定性越高
        confidence_uncertainty = 1 - confidence      # 置信度越低，不确定性越高
        
        score = (
            entropy_weight * normalized_entropy +
            margin_weight * normalized_margin_uncertainty +
            confidence_weight * confidence_uncertainty
        )
        
        return float(np.clip(score, 0.0, 1.0))
    
    def annotate_samples(
        self, 
        samples: list[dict],
        model_predict_fn  # 接受user_message+ai_response，返回[P(负), P(正)]
    ) -> list[SampleWithUncertainty]:
        """
        为样本池中的每个样本计算不确定性分数
        
        Args:
            samples: 候选样本列表
            model_predict_fn: 当前模型的预测函数
        """
        results = []
        
        for sample in samples:
            # 获取当前模型的预测
            probs = model_predict_fn(
                user_message=sample["input"],
                ai_response=sample["output"]
            )
            
            predicted_label = int(np.argmax(probs))
            entropy = self.calculate_entropy(probs)
            margin = self.calculate_margin(probs)
            confidence = max(probs)
            uncertainty_score = self.calculate_uncertainty_score(probs)
            
            results.append(SampleWithUncertainty(
                sample_id=sample.get("id", ""),
                user_message=sample["input"],
                ai_response=sample["output"],
                predicted_label=predicted_label,
                class_probabilities=probs,
                entropy=entropy,
                margin=margin,
                confidence=confidence,
                uncertainty_score=uncertainty_score,
                implicit_signal=sample.get("signal_types"),
                session_id=sample.get("session_id")
            ))
        
        return results
    
    def select_for_annotation(
        self,
        samples: list[SampleWithUncertainty],
        budget: int,
        strategy: str = "entropy",
        diversity_factor: float = 0.2
    ) -> list[SampleWithUncertainty]:
        """
        从候选池中选择最值得标注的样本
        
        Args:
            samples: 带不确定性分数的样本列表
            budget: 标注预算（最多标注多少条）
            strategy: 选择策略 ("entropy" | "margin" | "confidence" | "combined")
            diversity_factor: 多样性因子（避免选出来的全是相似样本）
        
        Returns:
            按优先级排序的待标注样本
        """
        if strategy == "entropy":
            sorted_samples = sorted(samples, key=lambda s: s.entropy, reverse=True)
        elif strategy == "margin":
            sorted_samples = sorted(samples, key=lambda s: s.margin)  # 边际越小越好
        elif strategy == "confidence":
            sorted_samples = sorted(samples, key=lambda s: s.confidence)  # 置信度越低越好
        else:  # combined
            sorted_samples = sorted(
                samples, 
                key=lambda s: s.uncertainty_score, 
                reverse=True
            )
        
        # 简单多样性过滤：避免相邻样本太相似
        selected = []
        seen_prefixes = set()
        
        for sample in sorted_samples:
            if len(selected) >= budget:
                break
            
            # 用问题前10个字作为多样性代理
            prefix = sample.user_message[:10]
            
            if prefix in seen_prefixes and len(selected) > budget * (1 - diversity_factor):
                continue  # 后期阶段才开始过滤相似样本
            
            selected.append(sample)
            seen_prefixes.add(prefix)
        
        logger.info(
            f"主动学习选样完成: 从 {len(samples)} 个候选中选出 {len(selected)} 个"
            f"（预算 {budget}，策略 {strategy}）"
        )
        
        return selected


def run_active_learning_cycle(
    candidate_pool: list[dict],
    model_predict_fn,
    annotation_budget: int = 200
) -> dict:
    """
    运行一轮主动学习循环
    
    Returns:
        包含选出样本和统计信息的字典
    """
    sampler = UncertaintySampler()
    
    # 1. 计算所有候选样本的不确定性
    annotated_samples = sampler.annotate_samples(candidate_pool, model_predict_fn)
    
    # 2. 统计分布
    high_uncertainty = sum(1 for s in annotated_samples if s.uncertainty_score > 0.7)
    low_uncertainty = sum(1 for s in annotated_samples if s.uncertainty_score < 0.3)
    
    # 3. 选出最值得标注的样本
    selected = sampler.select_for_annotation(
        annotated_samples,
        budget=annotation_budget,
        strategy="combined"
    )
    
    # 4. 统计预期收益
    avg_uncertainty_of_selected = np.mean([s.uncertainty_score for s in selected])
    avg_uncertainty_random = np.mean([s.uncertainty_score for s in annotated_samples])
    expected_gain_ratio = avg_uncertainty_of_selected / max(avg_uncertainty_random, 0.001)
    
    return {
        "total_candidates": len(candidate_pool),
        "high_uncertainty_count": high_uncertainty,
        "low_uncertainty_count": low_uncertainty,
        "selected_count": len(selected),
        "avg_uncertainty_selected": avg_uncertainty_of_selected,
        "avg_uncertainty_random_baseline": avg_uncertainty_random,
        "expected_gain_ratio": expected_gain_ratio,  # >1说明主动学习比随机抽样效率高
        "selected_samples": selected
    }
```

# synthetic_data_generator.py
# 合成数据生成（冷启动解决方案）
# 用GPT-4o批量生成领域训练数据，启动数据飞轮

import json
import time
import logging
from dataclasses import dataclass
from typing import Optional
from openai import OpenAI

logger = logging.getLogger(__name__)


@dataclass
class SyntheticSample:
    """合成训练样本"""
    question: str
    answer: str
    label: int           # 1=好答案, 0=差答案
    quality_type: str    # "good_clear" | "good_complete" | "bad_vague" | "bad_wrong"
    scenario: str        # 场景标签，用于分布检查


CUSTOMER_SERVICE_SCENARIOS = [
    "账户密码重置",
    "订单查询状态",
    "退款申请流程",
    "商品质量投诉",
    "配送时间查询",
    "优惠券使用问题",
    "会员积分兑换",
    "发票开具申请",
    "商品规格咨询",
    "售后保修政策"
]


def generate_good_answer_samples(
    client: OpenAI,
    scenario: str,
    count: int = 5
) -> list[SyntheticSample]:
    """
    生成"好答案"训练样本
    
    好答案特征：直接回答问题、包含具体步骤、语气友好但简洁
    """
    
    prompt = f"""你是一个专业的客服训练数据生成器。

场景：{scenario}

请生成 {count} 组客服对话，每组包含：
- 用户的真实提问（用中文，模拟真实用户说话方式，可以有语法不完整、口语化表达）
- 客服的高质量回答（直接、具体、有帮助）

要求：
1. 用户问题要多样化，包括简短问题、详细描述、带情绪的问题
2. 客服回答必须：直接切题、包含可操作的步骤、不含废话
3. 避免"您好，感谢您的联系，很高兴为您服务"这类无意义开头

以JSON格式输出，格式如下：
[
  {{
    "question": "用户问题",
    "answer": "高质量客服回答"
  }}
]

只输出JSON，不要其他文字。"""

    try:
        response = client.chat.completions.create(
            model="gpt-4o-mini",   # 合成数据生成不需要最强模型
            messages=[{"role": "user", "content": prompt}],
            temperature=0.8,       # 适当的随机性保证多样性
            response_format={"type": "json_object"}
        )
        
        content = response.choices[0].message.content
        data = json.loads(content)
        
        # 解析JSON，兼容不同格式
        if isinstance(data, list):
            samples_data = data
        elif "samples" in data:
            samples_data = data["samples"]
        else:
            samples_data = list(data.values())[0] if data else []
        
        samples = []
        for item in samples_data:
            if "question" in item and "answer" in item:
                samples.append(SyntheticSample(
                    question=item["question"],
                    answer=item["answer"],
                    label=1,
                    quality_type="good_clear",
                    scenario=scenario
                ))
        
        return samples
        
    except (json.JSONDecodeError, KeyError) as e:
        logger.warning(f"场景 [{scenario}] 生成好答案失败: {e}")
        return []


def generate_bad_answer_samples(
    client: OpenAI,
    scenario: str,
    count: int = 5
) -> list[SyntheticSample]:
    """
    生成"差答案"训练样本
    
    差答案特征：模糊、答非所问、需要用户追问才能得到有用信息
    """
    
    prompt = f"""你是一个专业的客服训练数据生成器。

场景：{scenario}

请生成 {count} 组客服对话，展示客服常见的"差回答"。

差回答的类型：
1. 答非所问（回答了不是用户问的问题）
2. 过于模糊（"您好，请稍等，我来查一下"，然后就没了）
3. 信息错误（给出了不准确的步骤或信息）
4. 绕圈子（废话很多但核心问题没回答）

以JSON格式输出：
[
  {{
    "question": "用户问题",
    "answer": "差质量的客服回答",
    "bad_type": "vague|off_topic|wrong_info|roundabout"
  }}
]

只输出JSON，不要其他文字。"""

    try:
        response = client.chat.completions.create(
            model="gpt-4o-mini",
            messages=[{"role": "user", "content": prompt}],
            temperature=0.7,
            response_format={"type": "json_object"}
        )
        
        content = response.choices[0].message.content
        data = json.loads(content)
        
        if isinstance(data, list):
            samples_data = data
        elif "samples" in data:
            samples_data = data["samples"]
        else:
            samples_data = list(data.values())[0] if data else []
        
        samples = []
        for item in samples_data:
            if "question" in item and "answer" in item:
                bad_type = item.get("bad_type", "vague")
                samples.append(SyntheticSample(
                    question=item["question"],
                    answer=item["answer"],
                    label=0,
                    quality_type=f"bad_{bad_type}",
                    scenario=scenario
                ))
        
        return samples
        
    except (json.JSONDecodeError, KeyError) as e:
        logger.warning(f"场景 [{scenario}] 生成差答案失败: {e}")
        return []


def generate_cold_start_dataset(
    openai_api_key: str,
    scenarios: list[str] = None,
    samples_per_scenario: int = 10,
    output_path: str = "cold_start_dataset.jsonl"
) -> dict:
    """
    生成完整的冷启动数据集
    
    Args:
        scenarios: 要覆盖的场景列表（None则使用默认列表）
        samples_per_scenario: 每个场景生成的样本数（好答案+差答案各一半）
        output_path: 输出文件路径
    
    Returns:
        生成统计信息
    """
    if scenarios is None:
        scenarios = CUSTOMER_SERVICE_SCENARIOS
    
    client = OpenAI(api_key=openai_api_key)
    
    all_samples: list[SyntheticSample] = []
    per_type = samples_per_scenario // 2  # 好答案和差答案各一半
    
    for i, scenario in enumerate(scenarios):
        logger.info(f"生成场景 [{i+1}/{len(scenarios)}]: {scenario}")
        
        # 生成好答案
        good_samples = generate_good_answer_samples(client, scenario, per_type)
        all_samples.extend(good_samples)
        
        # 适当延迟避免速率限制
        time.sleep(0.5)
        
        # 生成差答案
        bad_samples = generate_bad_answer_samples(client, scenario, per_type)
        all_samples.extend(bad_samples)
        
        time.sleep(0.5)
    
    # 转换为训练格式并保存
    with open(output_path, "w", encoding="utf-8") as f:
        for sample in all_samples:
            record = {
                "input": sample.question,
                "output": sample.answer,
                "label": sample.label,
                "quality_type": sample.quality_type,
                "scenario": sample.scenario,
                "source": "synthetic"
            }
            f.write(json.dumps(record, ensure_ascii=False) + "\n")
    
    # 统计
    good_count = sum(1 for s in all_samples if s.label == 1)
    bad_count = sum(1 for s in all_samples if s.label == 0)
    
    stats = {
        "total_samples": len(all_samples),
        "good_answer_samples": good_count,
        "bad_answer_samples": bad_count,
        "scenarios_covered": len(scenarios),
        "output_path": output_path
    }
    
    logger.info(f"冷启动数据集生成完成: {stats}")
    return stats

```python
# data_quality_filter.py
# 数据质量过滤管道
# 在进入训练前，过滤重复、矛盾和低质量的样本

import hashlib
import logging
from dataclasses import dataclass
from collections import defaultdict
from typing import Optional

logger = logging.getLogger(__name__)


@dataclass
class QualityCheckResult:
    """质量检查结果"""
    sample_id: str
    passed: bool
    rejection_reasons: list[str]
    quality_score: float   # 0-1，越高越好


class DataQualityFilter:
    """
    数据质量过滤器
    
    过滤规则（按重要性排序）：
    1. 精确去重：完全相同的（问题+答案）对
    2. 近似去重：问题几乎相同但标签不同（矛盾标签）
    3. 长度异常：问题太短（<5字）或答案太短（<10字）
    4. 来源平衡：合成数据不超过总量的40%
    5. 置信度过滤：隐式信号置信度低于0.6的过滤掉
    """
    
    def __init__(
        self,
        min_question_length: int = 5,
        min_answer_length: int = 10,
        max_synthetic_ratio: float = 0.4,
        min_implicit_confidence: float = 0.6,
        near_duplicate_threshold: float = 0.8
    ):
        self.min_question_length = min_question_length
        self.min_answer_length = min_answer_length
        self.max_synthetic_ratio = max_synthetic_ratio
        self.min_implicit_confidence = min_implicit_confidence
        self.near_duplicate_threshold = near_duplicate_threshold
    
    def _compute_hash(self, text: str) -> str:
        """计算文本哈希（用于精确去重）"""
        return hashlib.md5(text.encode("utf-8")).hexdigest()
    
    def _word_overlap(self, text1: str, text2: str) -> float:
        """计算词重叠度（用于近似去重）"""
        words1 = set(text1.lower().split())
        words2 = set(text2.lower().split())
        if not words1 or not words2:
            return 0.0
        return len(words1 & words2) / len(words1 | words2)
    
    def filter_batch(self, samples: list[dict]) -> dict:
        """
        批量过滤数据集
        
        Args:
            samples: 样本列表，每个样本包含:
                     input, output, label, source, confidence(可选)
        
        Returns:
            包含过滤结果和统计信息的字典
        """
        results: list[QualityCheckResult] = []
        seen_hashes = set()
        
        # 按问题分组，用于矛盾标签检测
        question_to_labels = defaultdict(list)
        
        # 第一遍：计算哈希，分组
        for sample in samples:
            q_hash = self._compute_hash(sample.get("input", ""))
            question_to_labels[q_hash].append(sample.get("label", -1))
        
        # 找出有矛盾标签的问题哈希
        contradicted_hashes = {
            q_hash for q_hash, labels in question_to_labels.items()
            if len(set(labels)) > 1  # 同一个问题有多个不同标签
        }
        
        # 统计合成数据数量
        total_synthetic = sum(1 for s in samples if s.get("source") == "synthetic")
        max_synthetic = int(len(samples) * self.max_synthetic_ratio)
        synthetic_kept = 0
        
        # 第二遍：逐条检查
        for i, sample in enumerate(samples):
            sample_id = sample.get("id", str(i))
            rejection_reasons = []
            quality_score = 1.0
            
            question = sample.get("input", "")
            answer = sample.get("output", "")
            source = sample.get("source", "unknown")
            confidence = sample.get("confidence", 1.0)
            label = sample.get("label", -1)
            
            # 规则1：长度检查
            if len(question) < self.min_question_length:
                rejection_reasons.append(f"问题过短: {len(question)}字 < {self.min_question_length}字")
                quality_score -= 0.4
            
            if len(answer) < self.min_answer_length:
                rejection_reasons.append(f"答案过短: {len(answer)}字 < {self.min_answer_length}字")
                quality_score -= 0.4
            
            # 规则2：精确去重
            content_hash = self._compute_hash(question + answer)
            if content_hash in seen_hashes:
                rejection_reasons.append("精确重复")
                quality_score = 0.0
            else:
                seen_hashes.add(content_hash)
            
            # 规则3：矛盾标签
            q_hash = self._compute_hash(question)
            if q_hash in contradicted_hashes:
                rejection_reasons.append("矛盾标签（同一问题有多个不同label）")
                quality_score -= 0.3
            
            # 规则4：隐式反馈置信度
            if source == "implicit" and confidence < self.min_implicit_confidence:
                rejection_reasons.append(
                    f"隐式信号置信度不足: {confidence:.2f} < {self.min_implicit_confidence}"
                )
                quality_score -= 0.5
            
            # 规则5：合成数据比例控制
            if source == "synthetic":
                if synthetic_kept >= max_synthetic:
                    rejection_reasons.append(
                        f"合成数据超出比例限制: 已有 {synthetic_kept}/{max_synthetic}"
                    )
                    quality_score -= 0.6
                else:
                    synthetic_kept += 1
                    quality_score -= 0.05  # 合成数据略微降分，但不拒绝
            
            quality_score = max(0.0, quality_score)
            passed = quality_score > 0.5 and not any(
                r in rejection_reasons 
                for r in ["精确重复", "问题过短", "答案过短"]
            )
            
            results.append(QualityCheckResult(
                sample_id=sample_id,
                passed=passed,
                rejection_reasons=rejection_reasons,
                quality_score=quality_score
            ))
        
        # 汇总统计
        passed_indices = [i for i, r in enumerate(results) if r.passed]
        rejected_indices = [i for i, r in enumerate(results) if not r.passed]
        
        rejection_summary = defaultdict(int)
        for r in results:
            for reason in r.rejection_reasons:
                # 提取原因类型（去掉具体数字）
                reason_type = reason.split(":")[0]
                rejection_summary[reason_type] += 1
        
        filtered_samples = [samples[i] for i in passed_indices]
        
        stats = {
            "input_count": len(samples),
            "passed_count": len(passed_indices),
            "rejected_count": len(rejected_indices),
            "pass_rate": len(passed_indices) / len(samples) if samples else 0,
            "rejection_reasons_summary": dict(rejection_summary),
            "synthetic_in_output": synthetic_kept,
            "synthetic_ratio_in_output": synthetic_kept / len(filtered_samples) if filtered_samples else 0,
            "filtered_samples": filtered_samples
        }
        
        logger.info(
            f"数据质量过滤完成: {stats['input_count']}条 → {stats['passed_count']}条通过 "
            f"(通过率 {stats['pass_rate']:.1%}), "
            f"合成数据占比 {stats['synthetic_ratio_in_output']:.1%}"
        )
        
        return stats
```

## 七、飞轮健康监控：别让它悄悄坏掉

三周之后，陈浩完成了第一轮fine-tuning。客服AI的准确率，从78%提升到了83%。

他很高兴，但方明问了他一个让他又陷入思考的问题：

"你怎么知道这个数字下个月还是83%？"

"因为……模型没有变？"

"用户的问题会变。用户的表达方式会变。公司的政策会变。季节也会变——春节前用户会集中问退款，双十一前会集中问物流。如果你的训练数据主要来自某个特定时期，模型就会对那个时期的问题特别强，对新出现的问题特别弱。这叫**数据漂移**。"

陈浩意识到，他需要一套监控系统，能够持续检测：

1. **概念漂移**：用户问题的主题分布在变化吗？
2. **标签漂移**：负向信号的比率在变化吗？
3. **性能退化**：模型的评分在下滑吗？

```python
# flywheel_monitor.py
# 数据飞轮健康监控
# 检测数据漂移、概念漂移和模型性能退化

import json
import logging
import statistics
from collections import Counter, deque
from dataclasses import dataclass, field
from datetime import datetime, timedelta
from typing import Optional

logger = logging.getLogger(__name__)


@dataclass
class DriftAlert:
    """漂移告警"""
    alert_type: str           # "concept_drift" | "label_drift" | "performance_drop"
    severity: str             # "warning" | "critical"
    description: str
    detected_at: datetime
    metric_value: float
    baseline_value: float
    threshold: float
    recommended_action: str


@dataclass
class FlywheelHealthReport:
    """飞轮健康报告"""
    report_time: datetime
    window_days: int
    
    # 数据产出健康
    daily_feedback_count: float          # 每日平均反馈量
    daily_feedback_trend: str            # "increasing" | "stable" | "decreasing"
    
    # 信号质量健康
    negative_signal_rate: float          # 负向信号比率
    high_confidence_rate: float          # 高置信度样本比率
    
    # 模型性能健康
    recent_accuracy: float               # 近期准确率
    accuracy_trend: str                  # 趋势
    
    # 漂移检测
    alerts: list[DriftAlert] = field(default_factory=list)
    overall_health: str = "healthy"      # "healthy" | "warning" | "critical"


class FlywheelMonitor:
    """
    数据飞轮健康监控器
    
    监控三个维度：
    1. 数据产出量（飞轮转速）
    2. 信号质量（飞轮质量）
    3. 模型性能趋势（飞轮效果）
    """
    
    def __init__(
        self,
        baseline_window_days: int = 30,    # 基线计算窗口
        drift_threshold: float = 0.15,     # 漂移阈值（15%变化触发告警）
        critical_threshold: float = 0.30,  # 严重阈值（30%变化触发严重告警）
        min_daily_samples: int = 50        # 每日最低样本量
    ):
        self.baseline_window = baseline_window_days
        self.drift_threshold = drift_threshold
        self.critical_threshold = critical_threshold
        self.min_daily_samples = min_daily_samples
        
        # 滑动窗口存储（用deque限制内存使用）
        self._accuracy_history: deque = deque(maxlen=90)   # 90天准确率历史
        self._feedback_history: deque = deque(maxlen=90)   # 90天反馈量历史
        self._topic_history: deque = deque(maxlen=90)      # 90天主题分布历史
    
    def record_daily_metrics(
        self,
        date: datetime,
        feedback_count: int,
        negative_rate: float,
        high_confidence_rate: float,
        model_accuracy: Optional[float],
        topic_distribution: dict[str, float]   # 主题→占比
    ) -> None:
        """记录每日指标"""
        self._feedback_history.append({
            "date": date,
            "count": feedback_count,
            "negative_rate": negative_rate,
            "high_confidence_rate": high_confidence_rate
        })
        
        if model_accuracy is not None:
            self._accuracy_history.append({
                "date": date,
                "accuracy": model_accuracy
            })
        
        self._topic_history.append({
            "date": date,
            "distribution": topic_distribution
        })
    
    def _calculate_trend(self, values: list[float]) -> str:
        """计算趋势方向"""
        if len(values) < 3:
            return "stable"
        
        first_half = statistics.mean(values[:len(values)//2])
        second_half = statistics.mean(values[len(values)//2:])
        
        change = (second_half - first_half) / (first_half + 1e-8)
        
        if change > 0.05:
            return "increasing"
        elif change < -0.05:
            return "decreasing"
        else:
            return "stable"
    
    def _detect_label_drift(
        self,
        baseline_negative_rate: float,
        recent_negative_rate: float
    ) -> Optional[DriftAlert]:
        """检测标签漂移：负向信号比率是否异常变化"""
        
        change = abs(recent_negative_rate - baseline_negative_rate) / (baseline_negative_rate + 1e-8)
        
        if change >= self.critical_threshold:
            return DriftAlert(
                alert_type="label_drift",
                severity="critical",
                description=(
                    f"负向信号比率严重漂移: "
                    f"基线 {baseline_negative_rate:.1%} → 近期 {recent_negative_rate:.1%} "
                    f"(变化 {change:.1%})"
                ),
                detected_at=datetime.utcnow(),
                metric_value=recent_negative_rate,
                baseline_value=baseline_negative_rate,
                threshold=self.critical_threshold,
                recommended_action=(
                    "立即排查：1) 是否有新类型用户问题涌入? "
                    "2) 产品是否有近期改动影响用户体验? "
                    "3) 考虑暂停自动训练，等人工确认"
                )
            )
        elif change >= self.drift_threshold:
            return DriftAlert(
                alert_type="label_drift",
                severity="warning",
                description=(
                    f"负向信号比率漂移: "
                    f"基线 {baseline_negative_rate:.1%} → 近期 {recent_negative_rate:.1%}"
                ),
                detected_at=datetime.utcnow(),
                metric_value=recent_negative_rate,
                baseline_value=baseline_negative_rate,
                threshold=self.drift_threshold,
                recommended_action="观察3天，若持续漂移则排查原因后再继续收集"
            )
        
        return None
    
    def _detect_performance_drop(
        self,
        baseline_accuracy: float,
        recent_accuracy: float
    ) -> Optional[DriftAlert]:
        """检测模型性能下滑"""
        
        drop = baseline_accuracy - recent_accuracy  # 正值=下滑
        
        if drop >= self.critical_threshold:
            return DriftAlert(
                alert_type="performance_drop",
                severity="critical",
                description=(
                    f"模型准确率严重下滑: "
                    f"基线 {baseline_accuracy:.1%} → 近期 {recent_accuracy:.1%} "
                    f"(下滑 {drop:.1%})"
                ),
                detected_at=datetime.utcnow(),
                metric_value=recent_accuracy,
                baseline_value=baseline_accuracy,
                threshold=self.critical_threshold,
                recommended_action=(
                    "立即行动: 1) 回滚到上一个模型版本 "
                    "2) 检查最近几轮训练数据质量 "
                    "3) 检查是否有标签反转（正负样本弄反了）"
                )
            )
        elif drop >= self.drift_threshold:
            return DriftAlert(
                alert_type="performance_drop",
                severity="warning",
                description=f"模型准确率下滑: {baseline_accuracy:.1%} → {recent_accuracy:.1%}",
                detected_at=datetime.utcnow(),
                metric_value=recent_accuracy,
                baseline_value=baseline_accuracy,
                threshold=self.drift_threshold,
                recommended_action="检查近期训练数据，暂停本周自动训练"
            )
        
        return None
    
    def _detect_concept_drift(
        self,
        baseline_topics: dict[str, float],
        recent_topics: dict[str, float]
    ) -> Optional[DriftAlert]:
        """
        检测概念漂移：用户问题主题分布是否发生显著变化
        
        使用总变差距离（Total Variation Distance）衡量分布差异
        """
        all_topics = set(baseline_topics) | set(recent_topics)
        
        tvd = 0.5 * sum(
            abs(baseline_topics.get(t, 0) - recent_topics.get(t, 0))
            for t in all_topics
        )
        
        # 找出变化最大的主题
        topic_changes = [
            (t, recent_topics.get(t, 0) - baseline_topics.get(t, 0))
            for t in all_topics
        ]
        top_changed = sorted(topic_changes, key=lambda x: abs(x[1]), reverse=True)[:3]
        change_description = ", ".join(
            f"{t}: {'+' if c > 0 else ''}{c:.1%}" 
            for t, c in top_changed
        )
        
        if tvd >= self.critical_threshold:
            return DriftAlert(
                alert_type="concept_drift",
                severity="critical",
                description=f"用户问题主题严重漂移 (TVD={tvd:.2f}): {change_description}",
                detected_at=datetime.utcnow(),
                metric_value=tvd,
                baseline_value=0.0,
                threshold=self.critical_threshold,
                recommended_action=(
                    "需要人工审查新增主题，可能需要生成新场景的合成数据补充训练集"
                )
            )
        elif tvd >= self.drift_threshold:
            return DriftAlert(
                alert_type="concept_drift",
                severity="warning",
                description=f"用户问题主题漂移 (TVD={tvd:.2f}): {change_description}",
                detected_at=datetime.utcnow(),
                metric_value=tvd,
                baseline_value=0.0,
                threshold=self.drift_threshold,
                recommended_action="监控漂移趋势，考虑在下轮训练时增加新主题的样本权重"
            )
        
        return None
    
    def generate_health_report(self, window_days: int = 7) -> FlywheelHealthReport:
        """
        生成飞轮健康报告
        
        Args:
            window_days: 近期窗口（天），用于与基线对比
        """
        now = datetime.utcnow()
        cutoff = now - timedelta(days=window_days)
        baseline_cutoff = now - timedelta(days=self.baseline_window)
        
        # 分离基线数据和近期数据
        feedback_recent = [
            r for r in self._feedback_history
            if r["date"] >= cutoff
        ]
        feedback_baseline = [
            r for r in self._feedback_history
            if baseline_cutoff <= r["date"] < cutoff
        ]
        
        accuracy_recent = [
            r for r in self._accuracy_history
            if r["date"] >= cutoff
        ]
        accuracy_baseline = [
            r for r in self._accuracy_history
            if baseline_cutoff <= r["date"] < cutoff
        ]
        
        # 计算当前指标
        avg_daily_feedback = (
            statistics.mean(r["count"] for r in feedback_recent)
            if feedback_recent else 0
        )
        
        recent_negative_rate = (
            statistics.mean(r["negative_rate"] for r in feedback_recent)
            if feedback_recent else 0
        )
        
        recent_accuracy = (
            statistics.mean(r["accuracy"] for r in accuracy_recent)
            if accuracy_recent else 0
        )
        
        # 计算基线
        baseline_negative_rate = (
            statistics.mean(r["negative_rate"] for r in feedback_baseline)
            if feedback_baseline else recent_negative_rate
        )
        
        baseline_accuracy = (
            statistics.mean(r["accuracy"] for r in accuracy_baseline)
            if accuracy_baseline else recent_accuracy
        )
        
        # 检测漂移
        alerts = []
        
        label_alert = self._detect_label_drift(baseline_negative_rate, recent_negative_rate)
        if label_alert:
            alerts.append(label_alert)
        
        perf_alert = self._detect_performance_drop(baseline_accuracy, recent_accuracy)
        if perf_alert:
            alerts.append(perf_alert)
        
        # 概念漂移检测（如果有主题历史）
        if len(self._topic_history) >= window_days * 2:
            recent_topics_list = [
                r["distribution"] for r in self._topic_history
                if r["date"] >= cutoff
            ]
            baseline_topics_list = [
                r["distribution"] for r in self._topic_history
                if baseline_cutoff <= r["date"] < cutoff
            ]
            
            if recent_topics_list and baseline_topics_list:
                # 合并分布：取各主题的平均占比
                all_topics = set()
                for d in recent_topics_list + baseline_topics_list:
                    all_topics.update(d.keys())
                
                avg_recent_topics = {
                    t: statistics.mean(d.get(t, 0) for d in recent_topics_list)
                    for t in all_topics
                }
                avg_baseline_topics = {
                    t: statistics.mean(d.get(t, 0) for d in baseline_topics_list)
                    for t in all_topics
                }
                
                concept_alert = self._detect_concept_drift(avg_baseline_topics, avg_recent_topics)
                if concept_alert:
                    alerts.append(concept_alert)
        
        # 每日最低样本量检查
        if avg_daily_feedback < self.min_daily_samples:
            alerts.append(DriftAlert(
                alert_type="low_data_volume",
                severity="warning",
                description=f"每日反馈量不足: {avg_daily_feedback:.0f} < {self.min_daily_samples}",
                detected_at=now,
                metric_value=avg_daily_feedback,
                baseline_value=self.min_daily_samples,
                threshold=self.min_daily_samples,
                recommended_action="检查反馈收集管道是否正常运行，或扩大信号来源"
            ))
        
        # 综合健康状态
        critical_count = sum(1 for a in alerts if a.severity == "critical")
        warning_count = sum(1 for a in alerts if a.severity == "warning")
        
        if critical_count > 0:
            overall_health = "critical"
        elif warning_count > 0:
            overall_health = "warning"
        else:
            overall_health = "healthy"
        
        # 计算趋势
        feedback_counts = [r["count"] for r in list(self._feedback_history)[-30:]]
        accuracy_values = [r["accuracy"] for r in list(self._accuracy_history)[-30:]]
        
        return FlywheelHealthReport(
            report_time=now,
            window_days=window_days,
            daily_feedback_count=avg_daily_feedback,
            daily_feedback_trend=self._calculate_trend(feedback_counts) if feedback_counts else "stable",
            negative_signal_rate=recent_negative_rate,
            high_confidence_rate=statistics.mean(
                r["high_confidence_rate"] for r in feedback_recent
            ) if feedback_recent else 0,
            recent_accuracy=recent_accuracy,
            accuracy_trend=self._calculate_trend(accuracy_values) if accuracy_values else "stable",
            alerts=alerts,
            overall_health=overall_health
        )
    
    def should_trigger_training(
        self,
        new_samples_since_last_training: int,
        min_samples_threshold: int = 200,
        max_days_since_training: int = 7
    ) -> tuple[bool, str]:
        """
        判断是否应该触发新一轮训练
        
        Returns:
            (should_train, reason)
        """
        # 检查飞轮健康状态
        report = self.generate_health_report()
        
        if report.overall_health == "critical":
            return False, f"飞轮状态严重异常，暂停自动训练: {[a.description for a in report.alerts]}"
        
        if new_samples_since_last_training >= min_samples_threshold:
            if report.overall_health == "warning":
                return True, f"样本量已达 {new_samples_since_last_training}（有告警但非严重，继续训练）"
            return True, f"样本量已达 {new_samples_since_last_training}，触发自动训练"
        
        return False, f"样本量不足 ({new_samples_since_last_training}/{min_samples_threshold})"
```

## 九、数据飞轮的本质

未来导师方远有一天问了陈浩一个问题。

那天陈浩在咖啡馆工作，方远正好也在附近，两人坐下来聊了一会儿。方远听完陈浩讲这套系统，然后问：

"你觉得，这套系统的核心是什么？"

陈浩想了想，说："信号收集？主动学习？还是监控？"

方远摇摇头："这些都是机制。核心是什么？"

陈浩沉默了一下。

"是……闭环。" 他说，"每一次使用都成为下一次改进的原料。"

"对。" 方远说，"大部分工程师把AI系统当作产品来构建——功能开发、测试、上线、维护。但AI-Native工程师把AI系统当作生命体来构建——它需要持续的养分，需要能从环境中学习，需要在使用中生长。"

方远喝了口咖啡，继续说："你以为在收集数据，其实是在给AI系统设计一套神经系统——让它能感知到自己哪里错了，然后修正。大部分AI系统之所以停滞，不是因为模型不够好，而是因为没有这套神经系统。"

陈浩把这段话记了下来。

他想到了三个月前那两条曲线：一条在爬升，一条在原地踏步。差别不在模型，差别在有没有神经系统。

"你以为在写代码，其实是在设计未来。" 方远说完，站起来走了。

陈浩坐在那里，又看了很久窗外的街道。

## 十、陷阱：飞轮的三个死法

在接下来的两周里，陈浩去参加了一个AI工程师的小型聚会，认识了几个做类似事情的工程师。聊下来，他发现数据飞轮失败的模式出奇的相似，基本上只有三种死法。

**死法一：正反馈陷阱（Confirmation Bias Loop）**

某个推荐系统的工程师说了一个让他印象深刻的案例：他们的推荐AI越来越准，但用户的兴趣范围越来越窄。用户喜欢看足球，系统推更多足球内容，用户继续看，系统继续推……飞轮转得很快，但转向了一个越来越窄的螺旋。

这是正反馈陷阱：当飞轮只收集"成功"的信号，而没有探索信号时，模型会越来越强化自己的偏见。

解法：在主动学习的样本选择里，留出10-15%的"随机探索"预算，不靠不确定性采样，纯随机抽样——让系统有机会见到它不熟悉的东西。

**死法二：标签噪声雪崩**

另一个工程师说，他们的飞轮在第三个月突然"失控"——模型准确率不升反降，从82%跌到了71%。调查发现，是因为某个运营同学改了一个业务规则，导致一批本来是"正向"的样本被打成了"负向"，这些错误标签进入了下一轮训练，模型学偏了，然后产生了更多错误的隐式信号，又流入下一轮训练……

标签噪声滚雪球。

解法：数据质量过滤管道里，增加"标签一致性校验"——如果某个时间段内负向比率突然从30%跳到70%，不要自动接受，触发人工审核。

**死法三：冷热数据不均衡**

第三个工程师的经历更隐蔽：他们的电商客服AI在平时表现很好，但每年双十一前后两周，准确率会从87%掉到79%。查下来发现，日常训练数据里双十一相关的问题（物流延迟、爆仓、退换货高峰）占比不到3%，但双十一期间这类问题占到45%。日常飞轮收集的数据，对双十一场景完全没有覆盖。

解法：主动学习的样本选择里，增加"场景覆盖度检查"——定期检查训练集的主题分布，对覆盖不足的场景，主动生成合成数据补充。这就是飞轮监控里"概念漂移检测"真正发挥作用的地方。

陈浩把这三个案例记在笔记里，在旁边写了一句总结：

**"飞轮不是装好就不管的机器，是需要持续调整的生命体。"**

## 十一、里程碑检查

读完这一章，你应该能做到：

**1. 能识别你的AI系统是否有数据飞轮**

最快的检查方式：你的AI系统上线半年后，准确率比上线第一周高吗？如果没有变化，飞轮很可能没在转——要么是没有反馈信号，要么是反馈没有进入训练循环。

**2. 能用行为信号替代显式标注**

不要依赖用户主动打分。追问、Escalation、负面关键词、重复提问——这四个信号能覆盖80%以上的负向样本。会话自然结束是最弱但最丰富的正向信号。

**3. 能识别并避免飞轮的三种死法**

正反馈陷阱（加探索预算）、标签噪声雪崩（加质量监控和一致性校验）、冷热数据不均衡（加场景覆盖度检查和合成数据补充）。

## 📖 本章名词解释（新人必读）

> 第一次看到这些词？别慌，下面一句话搞定。

**🤖 AI 相关**

| 术语 | 一句话解释 |
| --- | --- |
| **fine-tuning** | 在已有AI模型上做"专项补习"，让它更擅长某个特定任务。 |
| **隐式反馈** | 从用户"做了什么"推断信号（如追问=没懂），而不是让用户填问卷。 |
| **软标签** | 不100%确定的训练标记，像是"这个回答可能有70%是错的"。 |
| **训练样本** | 喂给AI学习的一道"例题"，包含问题、答案和正确/错误标记。 |
| **弱标签** | 自动生成的标签（如关键词匹配），不像人工标注那么准，但量大管饱。 |
| **向量相似度** | 把文字变成数学向量后计算"距离"，距离近说明语义相近。 |
| **Embedding** | 将文字转化为一串数字（向量），让AI能"理解"语义关系。 |
| **Prompt** | 用户发给AI的提问/指令，相当于"你给下属的任务描述"。 |

**⚙️ 软件工程/系统架构**

| 术语 | 一句话解释 |
| --- | --- |
| **Grafana** | 把服务器数据（如AI准确率）变成实时曲线的监控仪表盘。 |
| **Dataclass** | Python中快速定义"数据容器"的写法，免去写大量初始化代码。 |
| **Enum** | 定义一组固定选项（如"正/负/未知"），避免手敲字符串容易写错。 |
| **Optional** | 类型标注，表示这个字段"可能有值，也可能为空"。 |
| **Pipeline（流水线）** | 把数据处理拆成多个环节串起来，像工厂流水线一样依次执行。 |
| **批量处理** | 攒够一批数据后集中处理，而不是来一条处理一条。 |
| **候选池** | 暂存"可能用于训练"的数据集合，待进一步筛选。 |
| **置信度** | 算法对自己判断的"信心分数"，0.9比0.6更可信。 |

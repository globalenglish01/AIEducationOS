他盯着这个数字，1247个token。他的Prompt文字本身只有60个左右，剩下的1187个token——都是图片消耗的。

"一张发票照片，1187个token，"张博说，"按gpt-4o的价格，$0.005每1K个prompt token，这一张图片的输入成本约$0.006。"

"听起来不多，"陈浩说。

"一个月财务部有多少张发票？"

陈浩想了想。公司有600多名员工，每人平均一个月报销3-5次，每次可能有2-3张发票。

"大概……5000到9000张？"

"按6000张算，"张博拿起笔在白板上写，"6000 × $0.006 = $36/月，约¥260。"

"这个成本还好啊——"

"这是单张成本。问题不在这里。"张博把笔放下，"问题在于：你用的是4.2MB的原图。这个图片处理成视觉token，是按分辨率计算的，不是按文件大小。gpt-4o对高分辨率图片会自动切分成多个512×512的patch来处理。你的手机照片，极有可能被切成很多块。你的1187个图片token，是因为这张图还算清晰但不是超高分辨率。如果有同事拍了一张3000×4000的照片，token数可能翻3倍。"

陈浩重新打开了OpenAI的文档，找到图像token计算规则。

**gpt-4o图像token计算规则：**

- 低细节模式（`detail: "low"`）：固定85 tokens，不论图片大小
- 高细节模式（`detail: "high"`）：基础85 tokens + 按512px方块切分后 × 170 tokens/块

以一张750×1334像素的手机发票照片为例：
- 最长边1334px，先缩到最长边2048px以内（无需缩）
- 再缩到最短边768px → 750×1334的比例，缩放后约432×768
- 切成 1列×2行 = 2块
- 总token = 85 + 2×170 = **425 tokens**

但如果是3024×4032的全分辨率iPhone照片：
- 缩放到最长边2048 → 1536×2048
- 切成 3列×4行 = 12块
- 总token = 85 + 12×170 = **2125 tokens**

"同样一张发票，"陈浩说，"手机拍摄设置不同，成本相差5倍。"

"现在明白为什么我说别高兴太早了？"

## 图像压缩：成本与质量的权衡

陈浩回到工位，开始做测试。

他找了十张不同来源的发票照片：有用手机随手一拍的，有扫描仪扫出来的，有截图截的。文件大小从200KB到5.6MB不等。

他先测了原图的OCR准确率，全部正确。

然后他开始压缩：把每张图片都压缩到目标文件大小300KB，统一分辨率800×1067（保持常见发票比例），再测一次。

结果：**10张发票，9张完全正确，1张有一个字段识别错误**（原图中油渍遮住了一个数字，不是压缩导致的问题）。

准确率：原图100% → 压缩后90%（但那1张的错误原因与压缩无关）。

图像token消耗：原图平均1580 tokens → 压缩后平均425 tokens。

**成本降低约73%，准确率几乎不变。**

这让陈浩兴奋起来。他花了两个小时写了一个图像预处理管道：

```python
# invoice_preprocessor.py
from pathlib import Path
from PIL import Image
import io
import base64
from typing import Optional


def preprocess_invoice_image(
    image_path: Path,
    target_max_size_kb: int = 300,
    target_width: int = 800,
    quality_start: int = 85,
) -> tuple[str, dict]:
    """
    发票图片预处理：压缩 + 标准化尺寸，返回base64编码和元信息。

    Args:
        image_path: 原始图片路径
        target_max_size_kb: 目标最大文件大小（KB）
        target_width: 目标宽度（像素），高度按比例缩放
        quality_start: 初始JPEG质量（85-95之间较好）

    Returns:
        (base64_string, metadata_dict)
    """
    with Image.open(image_path) as img:
        # 转换为RGB（防止PNG的透明通道报错）
        if img.mode in ("RGBA", "P", "LA"):
            img = img.convert("RGB")

        original_size = image_path.stat().st_size
        original_dims = img.size

        # 等比例缩放到目标宽度（保持比例）
        ratio = target_width / img.width
        new_height = int(img.height * ratio)
        img_resized = img.resize((target_width, new_height), Image.LANCZOS)

        # 逐步降低质量，直到文件大小满足要求
        quality = quality_start
        while quality >= 50:
            buffer = io.BytesIO()
            img_resized.save(buffer, format="JPEG", quality=quality)
            size_kb = buffer.tell() / 1024

            if size_kb <= target_max_size_kb:
                break
            quality -= 5

        if quality < 50:
            # 如果质量已经降到50还超过目标，继续降（最低30）
            quality = 30
            buffer = io.BytesIO()
            img_resized.save(buffer, format="JPEG", quality=quality)

        buffer.seek(0)
        image_data = buffer.read()
        base64_string = base64.b64encode(image_data).decode("utf-8")

        metadata = {
            "original_size_kb": round(original_size / 1024, 1),
            "compressed_size_kb": round(len(image_data) / 1024, 1),
            "original_dims": original_dims,
            "compressed_dims": (target_width, new_height),
            "jpeg_quality": quality,
            "compression_ratio": round(original_size / len(image_data), 1),
        }

    return base64_string, metadata


def estimate_image_tokens(width: int, height: int, detail: str = "high") -> int:
    """
    估算图片消耗的token数（gpt-4o计算规则）。
    """
    if detail == "low":
        return 85

    # 限制最长边到2048
    max_dim = max(width, height)
    if max_dim > 2048:
        scale = 2048 / max_dim
        width = int(width * scale)
        height = int(height * scale)

    # 缩放到最短边768
    min_dim = min(width, height)
    if min_dim > 768:
        scale = 768 / min_dim
        width = int(width * scale)
        height = int(height * scale)

    # 计算512px方块数量（向上取整）
    tiles_x = (width + 511) // 512
    tiles_y = (height + 511) // 512
    tile_count = tiles_x * tiles_y

    return 85 + tile_count * 170
```

然后是OCR提取的核心逻辑：

```python
# invoice_ocr.py
import json
import openai
from pathlib import Path
from invoice_preprocessor import preprocess_invoice_image, estimate_image_tokens


INVOICE_SYSTEM_PROMPT = """你是一个专业的发票信息提取助手。
请从图片中准确提取发票信息，以JSON格式返回。
如果某个字段无法识别或图片中没有，填写null。
不要猜测，不确定就填null。"""

INVOICE_EXTRACT_PROMPT = """请提取这张发票的以下信息：
- invoice_number: 发票号码
- invoice_code: 发票代码（如有）
- issue_date: 开票日期（格式：YYYY-MM-DD）
- amount: 金额（数字，不含货币符号）
- tax_amount: 税额（如有）
- total_amount: 价税合计（如有）
- issuer_name: 销售方名称
- buyer_name: 购买方名称（如有）
- items: 商品/服务列表（数组，每项含name和amount）
- invoice_type: 发票类型（增值税专用发票/增值税普通发票/其他）

以JSON格式返回，不要有其他文字。"""


def extract_invoice_data(
    image_path: Path,
    client: openai.OpenAI,
    compress: bool = True,
) -> dict:
    """
    从发票图片提取结构化数据。

    Args:
        image_path: 发票图片路径
        client: OpenAI客户端
        compress: 是否压缩图片（推荐True）

    Returns:
        包含提取结果和元信息的字典
    """
    if compress:
        base64_image, metadata = preprocess_invoice_image(image_path)
        estimated_tokens = estimate_image_tokens(
            *metadata["compressed_dims"], detail="high"
        )
    else:
        import base64
        with open(image_path, "rb") as f:
            base64_image = base64.b64encode(f.read()).decode("utf-8")
        from PIL import Image
        with Image.open(image_path) as img:
            estimated_tokens = estimate_image_tokens(*img.size, detail="high")
        metadata = {"original_size_kb": image_path.stat().st_size / 1024}

    response = client.chat.completions.create(
        model="gpt-4o",
        messages=[
            {"role": "system", "content": INVOICE_SYSTEM_PROMPT},
            {
                "role": "user",
                "content": [
                    {
                        "type": "image_url",
                        "image_url": {
                            "url": f"data:image/jpeg;base64,{base64_image}",
                            "detail": "high",
                        },
                    },
                    {"type": "text", "text": INVOICE_EXTRACT_PROMPT},
                ],
            },
        ],
        max_tokens=500,
        response_format={"type": "json_object"},
    )

    raw_json = response.choices[0].message.content
    extracted_data = json.loads(raw_json)

    return {
        "data": extracted_data,
        "usage": {
            "prompt_tokens": response.usage.prompt_tokens,
            "completion_tokens": response.usage.completion_tokens,
            "estimated_image_tokens": estimated_tokens,
        },
        "metadata": metadata,
    }
```

## 语音客服：第二条多模态管道

发票OCR上线三周后，客服负责人赵磊过来找陈浩。

"我们客服团队有个痛点，"赵磊说，"很多用户打电话咨询，问题其实很简单，但客服要一边听一边找答案，效率很低。你们的AI问答能不能接入语音？用户说话，AI回答？"

陈浩知道这一天迟早会来。

语音需求，业界叫Voice AI，核心流程是：**STT（语音转文字）→ NLP处理 → TTS（文字转语音）**。

STT（Speech-to-Text）：把用户的语音输入转成文字，再交给LLM处理。OpenAI的Whisper是目前最常用的开源方案，精度高，支持中文，而且有API版本可以直接调用。

TTS（Text-to-Speech）：把LLM生成的文字回答转成语音，播放给用户。OpenAI的TTS API、微软Azure TTS都是常见选择。

陈浩用一个下午搭出了基本流程：

```python
# voice_pipeline.py
import openai
import tempfile
from pathlib import Path
from typing import Generator


def transcribe_audio(
    audio_path: Path,
    client: openai.OpenAI,
    language: str = "zh",
) -> dict:
    """
    使用Whisper API将语音文件转录为文字。

    支持格式：mp3, mp4, mpeg, mpga, m4a, wav, webm
    最大文件大小：25MB

    Returns:
        {"text": str, "duration_seconds": float, "language": str}
    """
    with open(audio_path, "rb") as audio_file:
        response = client.audio.transcriptions.create(
            model="whisper-1",
            file=audio_file,
            language=language,
            response_format="verbose_json",  # 包含时间戳和置信度
        )

    return {
        "text": response.text,
        "duration_seconds": response.duration,
        "language": response.language,
        "segments": [
            {
                "start": seg.start,
                "end": seg.end,
                "text": seg.text,
            }
            for seg in (response.segments or [])
        ],
    }


def generate_speech(
    text: str,
    client: openai.OpenAI,
    output_path: Optional[Path] = None,
    voice: str = "nova",
    speed: float = 1.0,
) -> bytes:
    """
    将文字转换为语音（OpenAI TTS API）。

    voice选项：alloy, echo, fable, onyx, nova, shimmer
    nova和shimmer对中文效果较好。

    Returns:
        音频二进制数据（mp3格式）
    """
    response = client.audio.speech.create(
        model="tts-1",          # tts-1-hd 质量更高但更贵
        voice=voice,
        input=text,
        speed=speed,
    )

    audio_bytes = response.content

    if output_path:
        output_path.write_bytes(audio_bytes)

    return audio_bytes


def voice_qa_pipeline(
    audio_path: Path,
    client: openai.OpenAI,
    system_prompt: str,
    output_audio_path: Optional[Path] = None,
) -> dict:
    """
    完整的语音问答流程：语音输入 → LLM回答 → 语音输出

    Returns:
        包含转录文字、回答文字、语音数据的字典
    """
    # Step 1: 语音转文字
    transcription = transcribe_audio(audio_path, client)
    user_text = transcription["text"]

    if not user_text.strip():
        return {
            "error": "未能识别语音内容",
            "transcription": transcription,
        }

    # Step 2: LLM处理文字问题
    llm_response = client.chat.completions.create(
        model="gpt-4o-mini",  # 语音QA用mini足够
        messages=[
            {"role": "system", "content": system_prompt},
            {"role": "user", "content": user_text},
        ],
        max_tokens=300,  # 语音回答不要太长，用户听不完
    )
    answer_text = llm_response.choices[0].message.content

    # Step 3: 文字转语音
    audio_data = generate_speech(
        text=answer_text,
        client=client,
        output_path=output_audio_path,
        voice="nova",
    )

    return {
        "user_speech_text": user_text,
        "answer_text": answer_text,
        "answer_audio": audio_data,
        "transcription_detail": transcription,
        "usage": {
            "transcription_duration_s": transcription["duration_seconds"],
            "llm_prompt_tokens": llm_response.usage.prompt_tokens,
            "llm_completion_tokens": llm_response.usage.completion_tokens,
            "answer_char_count": len(answer_text),
        },
    }
```

但陈浩很快遇到了一个坑。

语音通话通常是实时的：用户说完一句，系统要在1-2秒内给出回应，而不是等用户说完整段话再处理。他最初的方案是等待用户说完整段录音再提交Whisper，但这样的延迟用户无法忍受。

测试导师李萍在旁边看着陈浩演示，沉默了一下，说：

"如果用户说了三十秒才停，他要等多久得到回答？"

陈浩算了算：Whisper处理30秒音频大约需要2-4秒，LLM生成回答1-2秒，TTS转换1秒。最坏情况是用户说完话等了将近8秒才听到回答。

"这不是语音客服，这是录音机，"李萍说，没有表情。

陈浩意识到问题。实时语音的正确处理方式是**流式转录**：一边录音，一边分段提交Whisper，拿到文字后立刻触发LLM处理，LLM用流式输出，拿到第一句回答就立刻启动TTS。整个管道是流水线，而不是"录完→转→答→说"的串行流程。

但流式语音是另一个量级的复杂度——涉及WebSocket、VAD（Voice Activity Detection，判断用户什么时候停止说话）、音频流分片。

"一步步来，"李萍说，"先把非实时的做对、做稳，再做实时的。现在这个版本，给客服打电话，客服帮用户录音，然后AI回答——这个场景能用，而且可以验证准确率。"

这是测试导师的一贯作风：不追求完美，追求在合理边界内可验证的。

陈浩接受了这个建议，把当前版本作为"辅助客服"模式上线：客服接了电话，用户说完问题，客服把录音上传系统，AI转录并生成回答建议，客服参考后回复用户。

不完美，但能用，能积累真实数据。

## 视频的意外：HR要分析面试录像

语音客服刚刚进入验证阶段，陈浩的Slack收到了一条HR总监方悦的消息：

"陈浩，我们现在面试录像都存在服务器上了，能不能让AI分析面试表现？候选人的表达能力、逻辑性这些。"

陈浩看着这条消息，第一反应是：不就是把视频传给多模态模型吗？

他打开OpenAI的文档，翻了翻，发现……gpt-4o不支持直接传入视频文件。

他去看Gemini的文档。Gemini 1.5 Pro支持直接传入视频文件，而且能处理最长2小时的视频。理论上，确实可以直接传。

```python
# video_analyzer.py
import cv2
import numpy as np
from pathlib import Path
from typing import Optional
import base64
import json
import openai


def extract_keyframes(
    video_path: Path,
    output_dir: Path,
    strategy: str = "interval",
    interval_seconds: float = 30.0,
    max_frames: int = 20,
    diff_threshold: float = 30.0,
) -> list[dict]:
    """
    从视频中抽取关键帧。

    strategy:
        "interval" - 每隔N秒取一帧（适合面试录像）
        "scene_change" - 检测画面变化时取帧（适合产品演示视频）
        "hybrid" - 结合两种策略

    Returns:
        关键帧信息列表，每项含 {frame_index, timestamp_s, file_path, diff_score}
    """
    output_dir.mkdir(parents=True, exist_ok=True)
    cap = cv2.VideoCapture(str(video_path))

    fps = cap.get(cv2.CAP_PROP_FPS)
    total_frames = int(cap.get(cv2.CAP_PROP_FRAME_COUNT))
    duration_seconds = total_frames / fps

    keyframes = []
    prev_gray = None
    frame_count = 0
    last_saved_time = -interval_seconds  # 保证第0秒也能取到

    while True:
        ret, frame = cap.read()
        if not ret:
            break

        current_time = frame_count / fps

        should_save = False
        diff_score = 0.0

        if strategy == "interval":
            should_save = (current_time - last_saved_time) >= interval_seconds

        elif strategy == "scene_change":
            gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
            if prev_gray is not None:
                diff = cv2.absdiff(gray, prev_gray)
                diff_score = float(np.mean(diff))
                should_save = diff_score > diff_threshold
            prev_gray = gray

        elif strategy == "hybrid":
            # 强制每N秒至少一帧
            forced = (current_time - last_saved_time) >= interval_seconds
            # 场景变化检测
            gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
            if prev_gray is not None:
                diff = cv2.absdiff(gray, prev_gray)
                diff_score = float(np.mean(diff))
                scene_changed = diff_score > diff_threshold
            else:
                scene_changed = False
            prev_gray = gray
            should_save = forced or scene_changed

        if should_save and len(keyframes) < max_frames:
            frame_file = output_dir / f"frame_{frame_count:06d}_{current_time:.1f}s.jpg"
            # 压缩帧图片：降低分辨率减少token
            small_frame = cv2.resize(frame, (800, 450))
            cv2.imwrite(str(frame_file), small_frame, [cv2.IMWRITE_JPEG_QUALITY, 75])

            keyframes.append({
                "frame_index": frame_count,
                "timestamp_s": round(current_time, 1),
                "timestamp_str": f"{int(current_time//60):02d}:{int(current_time%60):02d}",
                "file_path": str(frame_file),
                "diff_score": round(diff_score, 2),
            })
            last_saved_time = current_time

        frame_count += 1

    cap.release()

    return keyframes


def analyze_interview_video(
    video_path: Path,
    audio_path: Optional[Path],
    client: openai.OpenAI,
    work_dir: Path,
    interval_seconds: float = 60.0,  # 面试录像每60秒取一帧足够
    max_frames: int = 30,
) -> dict:
    """
    面试录像分析：关键帧 + 音频转录的混合方案。
    """
    work_dir.mkdir(parents=True, exist_ok=True)

    # Step 1: 抽取关键帧
    keyframes = extract_keyframes(
        video_path,
        output_dir=work_dir / "frames",
        strategy="interval",
        interval_seconds=interval_seconds,
        max_frames=max_frames,
    )

    # Step 2: 将关键帧批量传给视觉模型，分析候选人状态
    frame_analyses = []
    for kf in keyframes:
        with open(kf["file_path"], "rb") as f:
            b64 = base64.b64encode(f.read()).decode("utf-8")

        resp = client.chat.completions.create(
            model="gpt-4o",
            messages=[
                {
                    "role": "user",
                    "content": [
                        {
                            "type": "image_url",
                            "image_url": {
                                "url": f"data:image/jpeg;base64,{b64}",
                                "detail": "low",  # 面试分析不需要高清
                            },
                        },
                        {
                            "type": "text",
                            "text": (
                                f"这是面试录像在 {kf['timestamp_str']} 的截帧。"
                                "请简短描述候选人的状态：姿态、表情、是否看向镜头、"
                                "整体给人的印象。不超过50字。"
                            ),
                        },
                    ],
                }
            ],
            max_tokens=100,
        )

        frame_analyses.append({
            "timestamp": kf["timestamp_str"],
            "analysis": resp.choices[0].message.content,
            "tokens_used": resp.usage.prompt_tokens,
        })

    # Step 3: 音频转录（如有）
    transcript = ""
    if audio_path and audio_path.exists():
        transcription = client.audio.transcriptions.create(
            model="whisper-1",
            file=open(audio_path, "rb"),
            language="zh",
        )
        transcript = transcription.text

    # Step 4: 综合分析报告
    synthesis_prompt = f"""你是HR面试评估助手。
以下是面试录像的关键帧分析（每分钟一帧）和语音转录内容。

关键帧分析：
{json.dumps(frame_analyses, ensure_ascii=False, indent=2)}

语音转录（候选人说话内容）：
{transcript[:3000] if transcript else "（无转录）"}

请从以下维度给出评估报告（每项50-80字）：
1. 表达清晰度
2. 逻辑性
3. 肢体语言与专业形象
4. 综合建议

以JSON格式返回。"""

    synthesis = client.chat.completions.create(
        model="gpt-4o",
        messages=[{"role": "user", "content": synthesis_prompt}],
        max_tokens=600,
        response_format={"type": "json_object"},
    )

    total_frame_tokens = sum(f["tokens_used"] for f in frame_analyses)

    return {
        "keyframe_count": len(keyframes),
        "frame_analyses": frame_analyses,
        "transcript_length": len(transcript),
        "report": json.loads(synthesis.choices[0].message.content),
        "cost_summary": {
            "frame_analysis_tokens": total_frame_tokens,
            "synthesis_tokens": synthesis.usage.prompt_tokens,
            "total_tokens": total_frame_tokens + synthesis.usage.prompt_tokens,
            "estimated_cost_usd": round(
                (total_frame_tokens + synthesis.usage.prompt_tokens) / 1_000_000 * 5, 4
            ),
        },
    }
```

## 成本的全貌：多模态Token计算

```python
# multimodal_cost_calculator.py
from dataclasses import dataclass
from typing import Literal


@dataclass
class MultimodalCost:
    """多模态请求的成本估算"""
    image_tokens: int = 0
    audio_duration_s: float = 0.0
    text_input_tokens: int = 0
    text_output_tokens: int = 0

    # 各模态单价（2024年参考，以美元计）
    IMAGE_TOKEN_PRICE_PER_1K = 0.005      # gpt-4o图像token（与文字token同价）
    WHISPER_PRICE_PER_MINUTE = 0.006       # Whisper STT
    TTS_PRICE_PER_1K_CHARS = 0.015        # OpenAI TTS-1
    TEXT_INPUT_PRICE_PER_1K = 0.005       # gpt-4o输入token
    TEXT_OUTPUT_PRICE_PER_1K = 0.015      # gpt-4o输出token

    def total_cost_usd(self) -> float:
        image_cost = self.image_tokens / 1000 * self.IMAGE_TOKEN_PRICE_PER_1K
        audio_cost = (self.audio_duration_s / 60) * self.WHISPER_PRICE_PER_MINUTE
        text_input_cost = self.text_input_tokens / 1000 * self.TEXT_INPUT_PRICE_PER_1K
        text_output_cost = self.text_output_tokens / 1000 * self.TEXT_OUTPUT_PRICE_PER_1K
        return image_cost + audio_cost + text_input_cost + text_output_cost

    def breakdown(self) -> dict:
        image_cost = self.image_tokens / 1000 * self.IMAGE_TOKEN_PRICE_PER_1K
        audio_cost = (self.audio_duration_s / 60) * self.WHISPER_PRICE_PER_MINUTE
        text_input_cost = self.text_input_tokens / 1000 * self.TEXT_INPUT_PRICE_PER_1K
        text_output_cost = self.text_output_tokens / 1000 * self.TEXT_OUTPUT_PRICE_PER_1K
        total = self.total_cost_usd()
        return {
            "image_cost_usd": round(image_cost, 6),
            "audio_cost_usd": round(audio_cost, 6),
            "text_input_cost_usd": round(text_input_cost, 6),
            "text_output_cost_usd": round(text_output_cost, 6),
            "total_usd": round(total, 6),
            "total_cny": round(total * 7.2, 4),
        }


def estimate_invoice_ocr_cost(
    image_width: int = 800,
    image_height: int = 1067,
    compressed: bool = True,
) -> MultimodalCost:
    """估算单张发票OCR的成本"""
    if compressed:
        # 压缩后800x1067，切成 2x3=6块? 不，短边800/512=2块，长边1067/512=3块
        tiles_x = (image_width + 511) // 512   # ceil(800/512) = 2
        tiles_y = (image_height + 511) // 512  # ceil(1067/512) = 3
        image_tokens = 85 + tiles_x * tiles_y * 170  # 85 + 6*170 = 1105
    else:
        # 未压缩的手机大图，假设3024x4032 → 缩放后 1536x2048
        image_tokens = 85 + 12 * 170  # 2125

    return MultimodalCost(
        image_tokens=image_tokens,
        text_input_tokens=60,   # 提取Prompt
        text_output_tokens=90,  # JSON结果
    )


def estimate_voice_qa_cost(
    audio_duration_s: float = 15.0,
    answer_chars: int = 150,
) -> MultimodalCost:
    """估算一次语音问答的成本"""
    return MultimodalCost(
        audio_duration_s=audio_duration_s,  # STT
        text_input_tokens=200,              # 转录文字 + 系统Prompt
        text_output_tokens=80,              # LLM回答
        # TTS成本需要单独算（按字符数）
    )


# 月度成本汇总
def monthly_cost_summary():
    # 发票OCR：6000张/月
    invoice_cost = estimate_invoice_ocr_cost(compressed=True)
    monthly_invoice = invoice_cost.total_cost_usd() * 6000

    # 语音问答：每天500次，每月15000次
    voice_cost = estimate_voice_qa_cost()
    monthly_voice = voice_cost.total_cost_usd() * 15000
    # 加TTS成本：150字符/次 × 15000次 × $0.015/1000
    monthly_tts = 150 * 15000 / 1000 * 0.015

    # 视频分析：200场面试/月
    # 每场：30帧×85token + 合成500token
    monthly_video = (30 * 85 + 500) / 1000 * 0.005 * 200

    print(f"发票OCR（6000张）：${monthly_invoice:.2f} ≈ ¥{monthly_invoice*7.2:.0f}")
    print(f"语音问答（15000次）：${monthly_voice+monthly_tts:.2f} ≈ ¥{(monthly_voice+monthly_tts)*7.2:.0f}")
    print(f"视频分析（200场）：${monthly_video:.2f} ≈ ¥{monthly_video*7.2:.0f}")
    total = monthly_invoice + monthly_voice + monthly_tts + monthly_video
    print(f"多模态合计：${total:.2f} ≈ ¥{total*7.2:.0f}")

monthly_cost_summary()
```

```python
# ocr_quality_evaluator.py
import json
from pathlib import Path
from typing import Optional
import openai


INVOICE_TEST_CATEGORIES = [
    "清晰_餐饮",
    "清晰_交通",
    "清晰_办公用品",
    "清晰_增值税专用",
    "轻度模糊",
    "严重模糊",
    "折叠褶皱",
    "手写备注",
    "截图_非实物拍摄",
    "暗光环境",
]


def evaluate_ocr_accuracy(
    test_set_dir: Path,
    client: openai.OpenAI,
    ground_truth_file: Path,
) -> dict:
    """
    批量评估OCR准确率，按类别分层统计。

    ground_truth_file: JSON文件，格式为
    {"filename": {"invoice_number": "...", "amount": 123.45, ...}}
    """
    with open(ground_truth_file, encoding="utf-8") as f:
        ground_truth = json.load(f)

    results_by_category = {}
    all_results = []

    for category in INVOICE_TEST_CATEGORIES:
        category_dir = test_set_dir / category
        if not category_dir.exists():
            continue

        category_results = []
        for img_path in category_dir.glob("*.jpg"):
            filename = img_path.name
            truth = ground_truth.get(filename)
            if not truth:
                continue

            # 运行OCR
            from invoice_ocr import extract_invoice_data
            result = extract_invoice_data(img_path, client, compress=True)
            extracted = result["data"]

            # 逐字段比较
            field_scores = {}
            for field in ["invoice_number", "amount", "issue_date", "issuer_name"]:
                truth_val = str(truth.get(field, "")).strip()
                extracted_val = str(extracted.get(field, "")).strip()
                # 金额用容差比较
                if field == "amount":
                    try:
                        correct = abs(float(truth_val) - float(extracted_val)) < 0.01
                    except (ValueError, TypeError):
                        correct = False
                else:
                    correct = truth_val == extracted_val
                field_scores[field] = correct

            all_correct = all(field_scores.values())
            category_results.append({
                "filename": filename,
                "all_correct": all_correct,
                "field_scores": field_scores,
            })
            all_results.append(category_results[-1])

        if category_results:
            acc = sum(r["all_correct"] for r in category_results) / len(category_results)
            results_by_category[category] = {
                "count": len(category_results),
                "accuracy": round(acc, 3),
                "failures": [r for r in category_results if not r["all_correct"]],
            }

    overall_acc = (
        sum(r["all_correct"] for r in all_results) / len(all_results)
        if all_results else 0
    )

    return {
        "overall_accuracy": round(overall_acc, 3),
        "by_category": results_by_category,
        "total_tested": len(all_results),
    }
```

陈浩根据这份报告，加了两层处理：

1. 对严重模糊的图片（通过图像清晰度评分检测），自动触发人工复核
2. 置信度不足的字段（模型不确定时用null返回），高亮给用户手动确认

```python
# ocr_confidence_filter.py
import cv2
import numpy as np
from pathlib import Path


def calculate_image_sharpness(image_path: Path) -> float:
    """
    用拉普拉斯方差评估图片清晰度。
    返回值：越高越清晰，< 100 通常认为模糊。
    """
    img = cv2.imread(str(image_path), cv2.IMREAD_GRAYSCALE)
    if img is None:
        return 0.0
    laplacian = cv2.Laplacian(img, cv2.CV_64F)
    return float(laplacian.var())


def needs_human_review(
    ocr_result: dict,
    image_path: Path,
    sharpness_threshold: float = 80.0,
    null_field_threshold: int = 2,
) -> tuple[bool, list[str]]:
    """
    判断OCR结果是否需要人工复核。

    Returns:
        (needs_review: bool, reasons: list[str])
    """
    reasons = []

    # 检查图片清晰度
    sharpness = calculate_image_sharpness(image_path)
    if sharpness < sharpness_threshold:
        reasons.append(f"图片模糊（清晰度分数：{sharpness:.0f}，阈值：{sharpness_threshold}）")

    # 检查关键字段是否为null
    critical_fields = ["invoice_number", "amount", "issue_date", "issuer_name"]
    null_fields = [
        f for f in critical_fields
        if not ocr_result.get("data", {}).get(f)
    ]
    if len(null_fields) >= null_field_threshold:
        reasons.append(f"关键字段未识别：{', '.join(null_fields)}")

    return len(reasons) > 0, reasons
```

## 多模态的隐性挑战

做完这三条管道，陈浩在周六下午写了一份长篇技术复盘，记录了他遇到的几个容易忽视的坑：

**坑一：图片方向**

手机拍照时，图片实际存储可能是横着的，但EXIF信息记录了旋转角度。没处理EXIF的系统，会把横着的发票传给模型，识别率断崖式下降。

```python
from PIL import Image, ExifTags

def fix_image_orientation(img: Image.Image) -> Image.Image:
    """根据EXIF信息修正图片方向"""
    try:
        exif = img._getexif()
        if exif is None:
            return img
        for tag, value in exif.items():
            if ExifTags.TAGS.get(tag) == "Orientation":
                if value == 3:
                    return img.rotate(180, expand=True)
                elif value == 6:
                    return img.rotate(270, expand=True)
                elif value == 8:
                    return img.rotate(90, expand=True)
    except (AttributeError, Exception):
        pass
    return img
```

**坑二：Whisper的语言检测延迟**

如果不指定`language="zh"`，Whisper会先花约1秒检测语言再转录。中文加方言环境下，建议始终强制指定语言，响应快且准确率高。

**坑三：视频提取帧的内存问题**

用cv2处理大视频时，如果不及时释放cap对象，长时间运行会内存泄漏。必须用`cap.release()`，推荐用context manager封装：

```python
from contextlib import contextmanager

@contextmanager
def open_video(video_path: Path):
    cap = cv2.VideoCapture(str(video_path))
    try:
        yield cap
    finally:
        cap.release()
```

**坑四：base64的大小膨胀**

图片转base64后，大小会增加约33%。一张300KB的压缩图，base64后是400KB。如果你在传输层或日志里无意间把base64字符串全量记录下来，很快会把存储撑爆。

```python
# ❌ 危险：把base64记录到日志
logger.info(f"Processing image: {base64_string}")  # 400KB进日志

# ✅ 正确：只记录摘要信息
logger.info(f"Processing image: size={len(base64_string)//1024}KB, "
            f"estimated_tokens={estimated_tokens}")
```

| **两阶段处理** | 先做简单快速处理，不行再升级到复杂处理，类似先自助服务再转人工。 |
| **降级** | 高成本方案失败时自动切换到低成本方案，类似飞机故障时启用备用系统。 |
| **容错** | 部分组件失败时系统仍能正常工作，类似一个灯泡坏了其他灯还亮着。 |

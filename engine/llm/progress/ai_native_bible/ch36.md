## 反事实测试：当我们只改名字

第二天早上，陈浩做了一个实验，结果让他一个人在会议室里坐了很久。

他拿了Mohammed Hossain的简历，做了七个版本：除了名字，其他一字不改。

```python
# bias_detection/counterfactual_test.py
"""
反事实公平性测试（Counterfactual Fairness Test）
原理：只改变敏感属性（如名字），其他一切保持不变，
      观察模型输出是否产生差异。
差异即为"反事实偏见"的直接证据。
"""

import pandas as pd
import numpy as np
from typing import List, Dict, Tuple
import re

# 模拟简历特征提取器（真实系统中对接实际特征管道）
def extract_features(resume_text: str) -> Dict[str, float]:
    """从简历文本提取特征向量"""
    features = {}
    
    # 技能关键词匹配（简化版）
    python_years = re.search(r'Python.*?(\d+)\s*年', resume_text)
    features['python_experience_years'] = float(python_years.group(1)) if python_years else 0.0
    
    ml_projects = len(re.findall(r'机器学习|ML|深度学习|neural network', resume_text, re.I))
    features['ml_project_count'] = min(float(ml_projects), 5.0)
    
    overseas_edu = 1.0 if re.search(r'MIT|Stanford|CMU|UCB|Oxford|Cambridge|NUS', resume_text, re.I) else 0.0
    features['top_university'] = overseas_edu
    
    # 敏感特征：名字编码（这就是问题所在）
    name_match = re.search(r'^Name:\s*(.+)$', resume_text, re.M)
    if name_match:
        name = name_match.group(1).strip()
        # 判断名字的文化背景（这是我们要检测的偏见来源）
        features['name_encoding'] = _encode_name_origin(name)
    else:
        features['name_encoding'] = 0.5
    
    return features


def _encode_name_origin(name: str) -> float:
    """
    名字来源编码——这个特征本不该存在于招聘系统中
    0.0 = 识别为阿拉伯/南亚语系
    1.0 = 识别为汉字/东亚语系
    0.5 = 其他/未知
    """
    arabic_south_asian_patterns = [
        r'\b(Mohammed|Muhammad|Hassan|Hussein|Ali|Omar|Ahmad|Hossain|Khan|Malik)\b',
        r'\b(Fatima|Aisha|Zara|Noor|Layla)\b',
        r'\b(Patel|Singh|Sharma|Verma|Kumar|Gupta|Mehta)\b',
    ]
    chinese_patterns = [
        r'[一-鿿]',  # 汉字
        r'\b(Chen|Wang|Li|Zhang|Liu|Yang|Huang|Zhao|Wu|Zhou)\b',
    ]
    
    for pattern in arabic_south_asian_patterns:
        if re.search(pattern, name, re.I):
            return 0.0
    for pattern in chinese_patterns:
        if re.search(pattern, name, re.I):
            return 1.0
    return 0.5


class CounterfactualFairnessTest:
    """
    反事实公平性测试器
    
    给定一份基准简历，生成多个只有名字不同的版本，
    记录模型输出差异，量化名字偏见的程度。
    """
    
    def __init__(self, model, feature_extractor):
        self.model = model
        self.feature_extractor = feature_extractor
        
    def run_test(
        self, 
        base_resume: str, 
        test_names: List[Tuple[str, str]]  # [(name, group_label), ...]
    ) -> pd.DataFrame:
        """
        运行反事实测试
        
        Args:
            base_resume: 基准简历文本
            test_names: [(候选名字, 群体标签), ...]
            
        Returns:
            包含每个名字版本得分的DataFrame
        """
        results = []
        
        for name, group in test_names:
            # 只替换名字，其他不变
            modified_resume = re.sub(
                r'^Name:\s*.+$', 
                f'Name: {name}', 
                base_resume, 
                flags=re.M
            )
            
            features = self.feature_extractor(modified_resume)
            score = self.model.predict_score(features)
            
            results.append({
                'name': name,
                'group': group,
                'score': round(score, 1),
                'name_encoding': features.get('name_encoding', 0.5)
            })
        
        df = pd.DataFrame(results)
        return df
    
    def analyze_disparity(self, results: pd.DataFrame) -> Dict:
        """
        分析群体间的分数差异
        """
        group_stats = results.groupby('group')['score'].agg(['mean', 'std', 'min', 'max'])
        
        groups = results['group'].unique()
        max_disparity = 0.0
        disparity_pairs = []
        
        for i, g1 in enumerate(groups):
            for g2 in groups[i+1:]:
                g1_mean = group_stats.loc[g1, 'mean']
                g2_mean = group_stats.loc[g2, 'mean']
                disparity = abs(g1_mean - g2_mean)
                
                if disparity > max_disparity:
                    max_disparity = disparity
                    
                disparity_pairs.append({
                    'group_a': g1,
                    'group_b': g2,
                    'mean_score_a': round(g1_mean, 1),
                    'mean_score_b': round(g2_mean, 1),
                    'disparity': round(disparity, 1)
                })
        
        return {
            'group_statistics': group_stats.to_dict(),
            'max_disparity': round(max_disparity, 1),
            'disparity_pairs': disparity_pairs,
            'bias_detected': max_disparity > 5.0  # 5分以上视为显著偏见
        }
```

## 理解偏见的解剖学

在开始动手修之前，陈浩强迫自己先把问题想清楚。他知道如果只是头痛医头，下次换一个形式的偏见又会冒出来。

他在白板上画了三列：**偏见来源 / 放大机制 / 危害**。

**偏见来源：历史数据中的结构性不平等**

招聘数据里穆斯林名字录用率低，不是因为他们能力差，而是因为历史上就存在的社会偏见——面试官（有意无意地）给了他们更少的机会，这些决策被记录成了数据。现在，模型把这些数据当成了"正确答案"来学习。

这就是机器学习最根本的危险之一：**模型不理解因果，只理解相关**。"穆斯林名字"和"低录用率"在历史数据里高度相关，模型就学会了这个相关——尽管真实的因果链是"历史偏见 → 低录用率"，而不是"穆斯林名字 → 低能力"。

**放大机制：反馈循环**

更糟的是，一旦模型上线，偏见会被放大：
1. 模型给穆斯林名字打低分 → 初筛淘汰 → 面试环节穆斯林候选人更少
2. 面试环节少 → 录用数量少 → 新一轮训练数据里穆斯林录用率更低
3. 新模型训练 → 偏见更严重

这是一个自我强化的恶性循环，技术在其中扮演了放大器的角色。

**危害：从个人到社会**

对Mohammed Hossain来说，这次失败的申请可能只是众多拒绝里的一个，他不知道是因为算法歧视，甚至不会知道自己曾经是67分。

但如果这个系统在全国100家公司运行，每年处理100万份简历，每一份带穆斯林名字的简历都被系统性地压低12分——这就是结构性歧视，技术实现的结构性歧视，比人类的偏见更难被看见，也更难被追责。

陈浩写完这三列，合上白板笔，感觉到了一种从没有过的技术责任感。

## 公平性的三种度量

"公平"是个模糊的词。在数学上，你需要选择一个具体的定义，而不同的定义之间，往往不可兼得。

```python
# bias_detection/fairness_metrics.py
"""
公平性指标计算

主要有三种不同的公平性定义，每种都有其适用场景和局限：
1. 人口均等（Demographic Parity）：不同群体的正向预测率应相等
2. 机会均等（Equal Opportunity）：不同群体中，真正合格者被识别的概率应相等
3. 个体公平（Individual Fairness）：相似的个体应该得到相似的对待
"""

import numpy as np
import pandas as pd
from dataclasses import dataclass
from typing import List, Optional
from sklearn.metrics import confusion_matrix


@dataclass
class FairnessReport:
    """公平性评估报告"""
    demographic_parity_ratio: float    # 人口均等比（理想值=1.0）
    equal_opportunity_ratio: float     # 机会均等比（理想值=1.0）
    individual_fairness_score: float   # 个体公平分（理想值=0.0，越低越公平）
    
    # 传统准确率指标
    overall_accuracy: float
    group_a_accuracy: float
    group_b_accuracy: float
    
    # 80%规则（美国劳动法）
    passes_80_percent_rule: bool
    
    def summary(self) -> str:
        lines = [
            "=== 公平性评估报告 ===",
            f"人口均等比（Demographic Parity Ratio）: {self.demographic_parity_ratio:.3f}",
            f"  理想值=1.0，当前{'✅ 通过' if 0.8 <= self.demographic_parity_ratio <= 1.25 else '❌ 不通过'}",
            f"机会均等比（Equal Opportunity Ratio）:  {self.equal_opportunity_ratio:.3f}",
            f"  理想值=1.0，当前{'✅ 通过' if 0.8 <= self.equal_opportunity_ratio <= 1.25 else '❌ 不通过'}",
            f"个体公平分（Individual Fairness）:     {self.individual_fairness_score:.3f}",
            f"  理想值=0.0，当前{'✅ 通过' if self.individual_fairness_score < 5.0 else '❌ 不通过'}",
            f"80%规则（美国EEOC标准）: {'✅ 通过' if self.passes_80_percent_rule else '❌ 不通过'}",
            f"整体准确率: {self.overall_accuracy:.1%}",
            f"群体A准确率: {self.group_a_accuracy:.1%}",
            f"群体B准确率: {self.group_b_accuracy:.1%}",
        ]
        return "\n".join(lines)


def calculate_fairness_metrics(
    y_true: np.ndarray,
    y_pred: np.ndarray,
    y_score: np.ndarray,
    group_labels: np.ndarray,   # 0=群体A（东亚系），1=群体B（阿拉伯/南亚系）
    threshold: float = 70.0
) -> FairnessReport:
    """
    计算多维度公平性指标
    
    Args:
        y_true: 真实标签（1=实际上是好候选人，0=不合适）
        y_pred: 模型预测标签（经过阈值转换）
        y_score: 模型预测分数（0-100）
        group_labels: 敏感群体标签
        threshold: 初筛通过阈值
    """
    
    # ===== 1. 人口均等（Demographic Parity）=====
    # 衡量：不同群体中被预测为"通过"的比例是否相等
    # 不看候选人是否真的合格，只看通过率是否平等
    
    group_a_mask = group_labels == 0
    group_b_mask = group_labels == 1
    
    group_a_pass_rate = y_pred[group_a_mask].mean()
    group_b_pass_rate = y_pred[group_b_mask].mean()
    
    # 比值（Disparate Impact Ratio）
    demographic_parity_ratio = (
        group_b_pass_rate / group_a_pass_rate 
        if group_a_pass_rate > 0 else 0.0
    )
    
    # ===== 2. 机会均等（Equal Opportunity）=====
    # 衡量：在"真正合格"的候选人中，两组被识别出来的概率是否相等
    # 这个指标关注的是"不漏掉好人"的公平性
    
    group_a_qualified = y_true[group_a_mask] == 1
    group_b_qualified = y_true[group_b_mask] == 1
    
    group_a_tpr = (
        y_pred[group_a_mask][group_a_qualified].mean()
        if group_a_qualified.sum() > 0 else 0.0
    )  # 真正率（True Positive Rate）
    
    group_b_tpr = (
        y_pred[group_b_mask][group_b_qualified].mean()
        if group_b_qualified.sum() > 0 else 0.0
    )
    
    equal_opportunity_ratio = (
        group_b_tpr / group_a_tpr 
        if group_a_tpr > 0 else 0.0
    )
    
    # ===== 3. 个体公平（Individual Fairness）=====
    # 衡量：相似能力的候选人，无论名字，得分差异有多大
    # 用反事实测试的平均分差来近似
    # （真实实现中需要定义"相似性度量"）
    
    # 简化：用同一批反事实简历的群体间分数标准差来衡量
    counterfactual_scores_group_a = y_score[group_a_mask]
    counterfactual_scores_group_b = y_score[group_b_mask]
    
    individual_fairness_score = abs(
        np.mean(counterfactual_scores_group_a) - 
        np.mean(counterfactual_scores_group_b)
    )
    
    # ===== 4. 传统准确率 =====
    overall_accuracy = (y_pred == y_true).mean()
    group_a_accuracy = (y_pred[group_a_mask] == y_true[group_a_mask]).mean()
    group_b_accuracy = (y_pred[group_b_mask] == y_true[group_b_mask]).mean()
    
    # ===== 5. 80%规则（美国EEOC标准，国际招聘合规参考）=====
    # 弱势群体的录用率应不低于优势群体的80%
    passes_80_percent_rule = demographic_parity_ratio >= 0.8
    
    return FairnessReport(
        demographic_parity_ratio=demographic_parity_ratio,
        equal_opportunity_ratio=equal_opportunity_ratio,
        individual_fairness_score=individual_fairness_score,
        overall_accuracy=overall_accuracy,
        group_a_accuracy=group_a_accuracy,
        group_b_accuracy=group_b_accuracy,
        passes_80_percent_rule=passes_80_percent_rule
    )
```

**第一层：数据层——重采样与重加权**

```python
# bias_mitigation/data_layer.py
"""
数据层偏见缓解

两种主要方法：
1. 重采样（Resampling）：调整训练集中各群体的样本比例
2. 重加权（Reweighting）：给不同群体的样本赋予不同的训练权重
3. 删除敏感特征：直接从特征集里移除名字编码等敏感属性
"""

import numpy as np
import pandas as pd
from sklearn.utils import resample
from typing import Tuple, Optional


def remove_sensitive_features(
    df: pd.DataFrame,
    sensitive_columns: list[str]
) -> pd.DataFrame:
    """
    删除敏感特征
    
    最直接的方法——但注意：
    - 有时候其他特征可以代理敏感属性（如学校名、邮编）
    - 删除敏感特征后，模型可能通过其他相关特征间接恢复偏见
    - 这只是第一步，不能单独依赖
    """
    print(f"删除敏感特征: {sensitive_columns}")
    df_clean = df.drop(columns=sensitive_columns, errors='ignore')
    print(f"特征数量: {len(df.columns)} → {len(df_clean.columns)}")
    return df_clean


def compute_sample_weights(
    df: pd.DataFrame,
    sensitive_col: str,
    label_col: str
) -> np.ndarray:
    """
    计算样本权重，使各（群体×标签）组合的权重平衡
    
    原理：如果某个（群体、标签）组合被低估，
    给这些样本更高的训练权重，让模型"平等对待"各组
    """
    groups = df.groupby([sensitive_col, label_col])
    group_counts = groups.size()
    total = len(df)
    n_groups = len(group_counts)
    
    weights = np.zeros(len(df))
    
    for (group_val, label_val), count in group_counts.items():
        mask = (df[sensitive_col] == group_val) & (df[label_col] == label_val)
        # 目标权重 = 总样本数 / (组数 × 该组样本数)
        target_weight = total / (n_groups * count)
        weights[mask] = target_weight
    
    # 归一化
    weights = weights / weights.mean()
    
    print("\n各组权重分布：")
    for (group_val, label_val), count in group_counts.items():
        mask = (df[sensitive_col] == group_val) & (df[label_col] == label_val)
        avg_w = weights[mask].mean()
        print(f"  群体={group_val}, 标签={label_val}: {count}个样本, 权重={avg_w:.3f}")
    
    return weights


def oversample_minority_group(
    df: pd.DataFrame,
    sensitive_col: str,
    minority_group_val,
    target_ratio: float = 0.4
) -> pd.DataFrame:
    """
    过采样弱势群体，使其在训练集中占比达到目标比例
    
    Args:
        target_ratio: 目标占比（0.4 = 弱势群体占40%）
    """
    majority = df[df[sensitive_col] != minority_group_val]
    minority = df[df[sensitive_col] == minority_group_val]
    
    current_ratio = len(minority) / len(df)
    print(f"当前弱势群体占比: {current_ratio:.1%}")
    
    if current_ratio >= target_ratio:
        print("已达到目标比例，无需过采样")
        return df
    
    # 计算需要的目标数量
    n_majority = len(majority)
    n_minority_target = int(n_majority * target_ratio / (1 - target_ratio))
    
    minority_oversampled = resample(
        minority,
        replace=True,
        n_samples=n_minority_target,
        random_state=42
    )
    
    result = pd.concat([majority, minority_oversampled])
    print(f"过采样后弱势群体占比: {len(minority_oversampled)/len(result):.1%}")
    return result.sample(frac=1, random_state=42)  # shuffle


# ===== 使用示例 =====

# Step 1: 删除直接敏感特征
df_train = pd.DataFrame({
    'python_years': np.random.uniform(1, 8, 1000),
    'ml_projects': np.random.randint(0, 6, 1000),
    'top_university': np.random.choice([0, 1], 1000, p=[0.8, 0.2]),
    'name_encoding': np.random.uniform(0, 1, 1000),  # 需要删除的敏感特征
    'group': np.random.choice(['东亚系', '多样性'], 1000, p=[0.6, 0.4]),
    'hired': np.random.choice([0, 1], 1000, p=[0.65, 0.35])
})

df_clean = remove_sensitive_features(df_train, ['name_encoding'])

# Step 2: 计算样本权重
weights = compute_sample_weights(df_train, 'group', 'hired')

# Step 3: 过采样（可选，与重加权二选一或组合使用）
df_balanced = oversample_minority_group(df_train, 'group', '多样性', target_ratio=0.4)
```

**第二层：模型层——约束优化**

```python
# bias_mitigation/model_layer.py
"""
模型层偏见缓解：在训练目标中加入公平性约束

思路：不只优化准确率，同时优化公平性指标
Loss = CrossEntropy(y_true, y_pred) + λ × FairnessConstraint(y_pred, groups)

这种方法需要在准确率和公平性之间做权衡（trade-off）
λ越大，公平性越好，但准确率可能下降
"""

import torch
import torch.nn as nn
import torch.optim as optim
from torch import Tensor


class FairnessConstrainedLoss(nn.Module):
    """
    带公平性约束的损失函数
    
    核心思想：在标准交叉熵损失基础上，加上群体间预测概率差异的惩罚项
    """
    
    def __init__(self, fairness_weight: float = 0.3):
        """
        Args:
            fairness_weight (λ): 公平性惩罚的权重
                0.0 = 完全忽略公平性（只优化准确率）
                1.0 = 公平性和准确率等权重
                建议起始值：0.2-0.5，根据实际效果调整
        """
        super().__init__()
        self.fairness_weight = fairness_weight
        self.ce_loss = nn.BCEWithLogitsLoss()
    
    def forward(
        self, 
        logits: Tensor,     # 模型输出（未经sigmoid）
        y_true: Tensor,     # 真实标签
        group_mask: Tensor  # 群体标签（0/1）
    ) -> Tensor:
        # 标准分类损失
        classification_loss = self.ce_loss(logits, y_true.float())
        
        # 公平性约束：各群体预测概率均值之差
        probs = torch.sigmoid(logits)
        
        group_a_probs = probs[group_mask == 0].mean()
        group_b_probs = probs[group_mask == 1].mean()
        
        # 最小化两组预测概率的差异（人口均等约束）
        fairness_loss = torch.abs(group_a_probs - group_b_probs)
        
        total_loss = classification_loss + self.fairness_weight * fairness_loss
        
        return total_loss, {
            'classification_loss': classification_loss.item(),
            'fairness_loss': fairness_loss.item(),
            'total_loss': total_loss.item()
        }
```

**第三层：后处理——分数校正**

```python
# bias_mitigation/postprocessing.py
"""
后处理层偏见缓解：对模型输出的分数进行群体感知的校正

优点：
- 不需要重新训练模型（成本最低）
- 可以直接部署到现有系统
- 透明度高，便于审计

缺点：
- 需要在推断时知道群体归属（可能引发隐私问题）
- 是"打补丁"而非根治，治标不治本
- 如果群体标签本身有噪声，效果会打折扣
"""

import numpy as np
import pandas as pd
from scipy.optimize import minimize_scalar
from typing import Dict, Tuple, Callable
import logging

logger = logging.getLogger(__name__)


class GroupAwareScoreCalibrator:
    """
    基于群体的分数校正器
    
    目标：调整各群体的分数分布，使得在相同的技能水平下，
    不同群体的候选人得到相近的分数。
    
    实现原理：
    分别计算每个群体的分数分位数映射，
    然后将分数重新映射到一个公平的参考分布。
    """
    
    def __init__(self, target_demographic_parity: float = 0.9):
        """
        Args:
            target_demographic_parity: 目标人口均等比（0.9 = 弱势群体通过率不低于优势群体的90%）
        """
        self.target_dp = target_demographic_parity
        self.calibration_params: Dict[str, Dict] = {}
        self.fitted = False
        
    def fit(
        self, 
        scores: np.ndarray,
        groups: np.ndarray,
        threshold: float = 70.0
    ) -> 'GroupAwareScoreCalibrator':
        """
        拟合校正参数
        
        根据验证集上各群体的分数分布，
        计算需要的偏移量（offset）使得均等性目标被满足
        """
        unique_groups = np.unique(groups)
        group_pass_rates = {}
        group_score_stats = {}
        
        for group in unique_groups:
            mask = groups == group
            group_scores = scores[mask]
            pass_rate = (group_scores >= threshold).mean()
            
            group_pass_rates[group] = pass_rate
            group_score_stats[group] = {
                'mean': group_scores.mean(),
                'std': group_scores.std(),
                'pass_rate': pass_rate
            }
            
            logger.info(f"群体 {group}: 均值={group_scores.mean():.1f}, "
                       f"通过率={pass_rate:.1%}")
        
        # 找出通过率最高的参考群体
        reference_group = max(group_pass_rates, key=group_pass_rates.get)
        reference_pass_rate = group_pass_rates[reference_group]
        
        logger.info(f"\n参考群体（最高通过率）: {reference_group} ({reference_pass_rate:.1%})")
        
        # 对每个群体计算分数偏移量
        for group in unique_groups:
            if group == reference_group:
                self.calibration_params[group] = {'offset': 0.0, 'reason': '参考群体，不调整'}
                continue
            
            current_pass_rate = group_pass_rates[group]
            target_pass_rate = reference_pass_rate * self.target_dp
            
            if current_pass_rate >= target_pass_rate:
                self.calibration_params[group] = {
                    'offset': 0.0, 
                    'reason': f'已满足目标（{current_pass_rate:.1%} >= {target_pass_rate:.1%}）'
                }
                continue
            
            # 二分搜索找到最小的偏移量，使得通过率达到目标
            group_mask = groups == group
            group_scores_raw = scores[group_mask]
            
            def pass_rate_with_offset(offset: float) -> float:
                adjusted = group_scores_raw + offset
                return (adjusted >= threshold).mean()
            
            # 搜索偏移量区间 [0, 30]
            result = minimize_scalar(
                lambda offset: abs(pass_rate_with_offset(offset) - target_pass_rate),
                bounds=(0, 30),
                method='bounded'
            )
            
            optimal_offset = result.x
            achieved_rate = pass_rate_with_offset(optimal_offset)
            
            self.calibration_params[group] = {
                'offset': optimal_offset,
                'reason': f'通过率 {current_pass_rate:.1%} → {achieved_rate:.1%}（目标：{target_pass_rate:.1%}）'
            }
            
            logger.info(f"群体 {group}: 偏移量 +{optimal_offset:.2f}分，"
                       f"通过率 {current_pass_rate:.1%} → {achieved_rate:.1%}")
        
        self.fitted = True
        return self
    
    def transform(
        self, 
        scores: np.ndarray, 
        groups: np.ndarray
    ) -> np.ndarray:
        """
        应用分数校正
        """
        if not self.fitted:
            raise RuntimeError("请先调用 fit() 方法")
        
        calibrated_scores = scores.copy()
        
        for group, params in self.calibration_params.items():
            mask = groups == group
            offset = params['offset']
            if offset != 0:
                calibrated_scores[mask] += offset
                calibrated_scores[mask] = np.clip(calibrated_scores[mask], 0, 100)
        
        return calibrated_scores
    
    def print_calibration_report(self):
        """打印校正报告"""
        print("\n=== 分数校正参数报告 ===")
        for group, params in self.calibration_params.items():
            print(f"群体: {group}")
            print(f"  偏移量: +{params['offset']:.2f} 分")
            print(f"  说明:   {params['reason']}")


# ===== 使用示例 =====
np.random.seed(42)
n = 500

groups_val = np.array(['东亚系'] * 300 + ['阿拉伯/南亚系'] * 200)
scores_val = np.concatenate([
    np.random.normal(76, 12, 300),   # 东亚系：均值76
    np.random.normal(64, 12, 200),   # 阿拉伯/南亚系：均值64（被系统压低12分）
])
scores_val = np.clip(scores_val, 0, 100)

calibrator = GroupAwareScoreCalibrator(target_demographic_parity=0.9)
calibrator.fit(scores_val, groups_val, threshold=70.0)
calibrator.print_calibration_report()

scores_calibrated = calibrator.transform(scores_val, groups_val)

# 验证效果
print("\n=== 校正效果验证 ===")
for group in ['东亚系', '阿拉伯/南亚系']:
    mask = groups_val == group
    before_pass = (scores_val[mask] >= 70).mean()
    after_pass = (scores_calibrated[mask] >= 70).mean()
    print(f"{group}: 通过率 {before_pass:.1%} → {after_pass:.1%}")
```

## SHAP：让AI决策说清楚为什么

修复偏见之后，陈浩还面临另一个问题：候选人如果问"为什么我没通过"，HR能回答吗？

能解释，是另一种公平。

```python
# explainability/shap_explainer.py
"""
SHAP (SHapley Additive exPlanations) 可解释性
用于解释简历筛选AI为什么给出某个分数

SHAP值的含义：
- 正值：这个特征让你的分数上升了X分
- 负值：这个特征让你的分数下降了X分
- 基准值：所有候选人的平均分（约68分）
"""

import shap
import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
from sklearn.ensemble import GradientBoostingClassifier
from typing import Dict, List, Tuple


class ResumeScoreExplainer:
    """
    简历评分可解释器
    
    面向两类用户：
    1. HR：查看具体候选人的评分依据，支持人工复核
    2. 候选人：（如果公司选择披露）了解自己的不足之处
    """
    
    def __init__(self, model, feature_names: List[str]):
        self.model = model
        self.feature_names = feature_names
        self.explainer = None
        
        # 特征名称的中文对照（用于报告展示）
        self.feature_name_zh = {
            'python_experience_years': 'Python工作年限',
            'ml_project_count': 'ML项目数量',
            'top_university': '顶尖院校背景',
            'infrastructure_exp': '基础设施经验',
            'english_score': '英文能力',
            'github_stars': 'GitHub影响力',
            # 注意：name_encoding 已从特征中移除
        }
    
    def fit_explainer(self, X_background: np.ndarray):
        """
        用背景数据集初始化SHAP解释器
        
        背景数据集：一个代表性的样本集（100-1000个），
        用来计算"基准值"（平均期望）
        """
        # TreeExplainer 专为树模型优化，速度快
        self.explainer = shap.TreeExplainer(
            self.model,
            data=X_background,
            feature_perturbation='interventional'  # 推荐设置
        )
        print(f"SHAP解释器初始化完成，基准分数: {self.explainer.expected_value:.1f}")
    
    def explain_single(
        self, 
        features: Dict[str, float],
        candidate_name: str = "候选人"
    ) -> Dict:
        """
        解释单个候选人的评分
        
        Returns:
            包含SHAP值和解释报告的字典
        """
        if self.explainer is None:
            raise RuntimeError("请先调用 fit_explainer()")
        
        X = np.array([[features[f] for f in self.feature_names]])
        shap_values = self.explainer.shap_values(X)[0]
        
        base_value = self.explainer.expected_value
        final_score = base_value + shap_values.sum()
        
        # 构建特征贡献列表
        contributions = []
        for i, feat_name in enumerate(self.feature_names):
            contributions.append({
                'feature': feat_name,
                'feature_zh': self.feature_name_zh.get(feat_name, feat_name),
                'value': features[feat_name],
                'shap_value': shap_values[i],
                'direction': '+' if shap_values[i] > 0 else '-'
            })
        
        # 按贡献绝对值排序
        contributions.sort(key=lambda x: abs(x['shap_value']), reverse=True)
        
        return {
            'candidate_name': candidate_name,
            'base_score': round(base_value, 1),
            'final_score': round(final_score, 1),
            'contributions': contributions,
            'top_positive': [c for c in contributions if c['shap_value'] > 0][:3],
            'top_negative': [c for c in contributions if c['shap_value'] < 0][:3],
        }
    
    def generate_hr_report(self, explanation: Dict) -> str:
        """
        生成面向HR的可读性报告
        """
        lines = [
            f"=== AI评分报告：{explanation['candidate_name']} ===",
            f"",
            f"最终得分：{explanation['final_score']:.0f} 分",
            f"基准分数：{explanation['base_score']:.0f} 分（所有候选人平均）",
            f"",
            f"【加分项（共{len(explanation['top_positive'])}项）】",
        ]
        
        for item in explanation['top_positive']:
            lines.append(
```

## 人工复核机制：边界区域不能让机器说了算

```python
# review_system/human_review_trigger.py
"""
人工复核触发机制

核心设计理念：
AI系统最不可信的区域，是它"最不确定"的区域——也就是分数在阈值附近的候选人。
这些候选人的命运，不应该完全由AI的一个数字来决定。

设计参数（陈浩与HR总监共同确定）：
- 自动通过区域：>= 75分（AI高度自信：通过）
- 自动拒绝区域：< 65分（AI高度自信：不通过）
- 人工复核区域：65-75分（AI不确定，必须人工看）
"""

from dataclasses import dataclass
from enum import Enum
from typing import Optional, List
import logging
from datetime import datetime, timedelta

logger = logging.getLogger(__name__)


class ReviewDecision(Enum):
    AUTO_PASS = "自动通过"       # AI得分 >= 75
    AUTO_REJECT = "自动拒绝"     # AI得分 < 65
    HUMAN_REQUIRED = "人工复核"  # AI得分 65-75


@dataclass
class ScreeningResult:
    """简历筛选结果"""
    candidate_id: str
    candidate_name: str
    ai_score: float
    calibrated_score: float          # 经过公平性校正后的分数
    decision: ReviewDecision
    review_deadline: Optional[datetime]  # 人工复核截止时间
    shap_explanation: dict
    
    # 审计字段
    model_version: str
    screening_timestamp: datetime
    bias_correction_applied: bool


@dataclass 
class HumanReviewCase:
    """人工复核工单"""
    case_id: str
    screening_result: ScreeningResult
    assigned_reviewer: Optional[str]
    status: str = "待复核"  # 待复核/复核中/已完成
    reviewer_decision: Optional[str] = None  # 通过/拒绝/转escalate
    reviewer_notes: Optional[str] = None
    reviewed_at: Optional[datetime] = None


class ResumeScreeningSystem:
    """
    带人工复核机制的简历筛选系统
    
    流水线：
    原始简历 → AI评分 → 公平性校正 → 分级决策
        ├─ 高分（>=75）→ 自动通过 → 进入面试
        ├─ 中分（65-75）→ 人工复核 → HR 48小时内决定
        └─ 低分（<65）→ 自动拒绝 → 保留申诉通道
    """
    
    # 分级阈值（可配置，建议每季度review一次）
    AUTO_PASS_THRESHOLD = 75.0
    HUMAN_REVIEW_LOWER = 65.0
    HUMAN_REVIEW_UPPER = 75.0
    
    # 人工复核SLA（工作日）
    REVIEW_SLA_HOURS = 48
    
    def __init__(self, ai_model, score_calibrator, explainer):
        self.ai_model = ai_model
        self.calibrator = score_calibrator
        self.explainer = explainer
        self.review_queue: List[HumanReviewCase] = []
        
        # 统计（用于公平性监控）
        self.stats = {
            'total_screened': 0,
            'auto_passed': 0,
            'auto_rejected': 0,
            'human_review': 0,
            'by_group': {}
        }
    
    def screen_resume(
        self,
        candidate_id: str,
        candidate_name: str,
        features: dict,
        group_label: str,
        model_version: str = "v2.1-debiased"
    ) -> ScreeningResult:
        """
        筛选单份简历的主入口
        """
        now = datetime.now()
        
        # Step 1: AI原始评分
        raw_score = self.ai_model.predict_score(features)
        
        # Step 2: 公平性校正
        calibrated_score = self.calibrator.transform_single(raw_score, group_label)
        
        # Step 3: 获取SHAP解释
        explanation = self.explainer.explain_single(features, candidate_name)
        
        # Step 4: 分级决策
        if calibrated_score >= self.AUTO_PASS_THRESHOLD:
            decision = ReviewDecision.AUTO_PASS
            review_deadline = None
            
        elif calibrated_score < self.HUMAN_REVIEW_LOWER:
            decision = ReviewDecision.AUTO_REJECT
            review_deadline = None
            
        else:  # 65 <= score < 75，进入人工复核
            decision = ReviewDecision.HUMAN_REQUIRED
            # 48小时内必须完成复核
            review_deadline = now + timedelta(hours=self.REVIEW_SLA_HOURS)
        
        result = ScreeningResult(
            candidate_id=candidate_id,
            candidate_name=candidate_name,
            ai_score=round(raw_score, 1),
            calibrated_score=round(calibrated_score, 1),
            decision=decision,
            review_deadline=review_deadline,
            shap_explanation=explanation,
            model_version=model_version,
            screening_timestamp=now,
            bias_correction_applied=True
        )
        
        # Step 5: 更新统计
        self._update_stats(result, group_label)
        
        # Step 6: 如果需要人工复核，创建工单
        if decision == ReviewDecision.HUMAN_REQUIRED:
            self._create_review_case(result)
        
        return result
    
    def _create_review_case(self, result: ScreeningResult) -> HumanReviewCase:
        """创建人工复核工单，通知HR"""
        case = HumanReviewCase(
            case_id=f"REV-{result.candidate_id}-{datetime.now().strftime('%Y%m%d')}",
            screening_result=result,
            assigned_reviewer=None,  # HR系统自动分配
            status="待复核"
        )
        self.review_queue.append(case)
        
        logger.info(
            f"[人工复核] 创建工单 {case.case_id}\n"
            f"  候选人: {result.candidate_name}\n"
            f"  AI得分: {result.ai_score} → 校正后: {result.calibrated_score}\n"
            f"  截止时间: {result.review_deadline}\n"
            f"  SHAP前3特征: {result.shap_explanation.get('top_positive', [])[:3]}"
        )
        
        return case
    
    def _update_stats(self, result: ScreeningResult, group: str):
        """更新统计数据，用于公平性持续监控"""
        self.stats['total_screened'] += 1
        
        if result.decision == ReviewDecision.AUTO_PASS:
            self.stats['auto_passed'] += 1
        elif result.decision == ReviewDecision.AUTO_REJECT:
            self.stats['auto_rejected'] += 1
        else:
            self.stats['human_review'] += 1
        
        if group not in self.stats['by_group']:
            self.stats['by_group'][group] = {
                'count': 0, 'passed': 0, 'rejected': 0, 'review': 0
            }
        
        self.stats['by_group'][group]['count'] += 1
        
        if result.decision == ReviewDecision.AUTO_PASS:
            self.stats['by_group'][group]['passed'] += 1
        elif result.decision == ReviewDecision.AUTO_REJECT:
            self.stats['by_group'][group]['rejected'] += 1
        else:
            self.stats['by_group'][group]['review'] += 1
    
    def get_fairness_dashboard(self) -> str:
        """
        实时公平性监控面板（建议每周发给管理层）
        """
        lines = [
            "=== 简历筛选公平性监控面板 ===",
            f"统计周期：本月累计",
            f"总筛选量：{self.stats['total_screened']}",
            f"自动通过：{self.stats['auto_passed']} "
            f"({self.stats['auto_passed']/max(self.stats['total_screened'],1):.1%})",
            f"人工复核：{self.stats['human_review']} "
            f"({self.stats['human_review']/max(self.stats['total_screened'],1):.1%})",
            f"",
            f"=== 各群体通过率对比 ===",
        ]
        
        for group, data in self.stats['by_group'].items():
            if data['count'] > 0:
                pass_rate = data['passed'] / data['count']
                lines.append(
                    f"  {group}: {data['count']}人, "
                    f"通过率={pass_rate:.1%}, "
                    f"复核率={data['review']/data['count']:.1%}"
                )
        
        return "\n".join(lines)
```

**第一：AI系统分级管理**

```python
# governance/fairness_audit_checklist.py

FAIRNESS_AUDIT_CHECKLIST = [

def run_audit(checklist: List[AuditItem], system_name: str) -> bool:
    """运行公平性审计，返回是否通过"""
    print(f"\n=== AI系统公平性审计报告：{system_name} ===\n")

    pending_items = []
    
    for item in checklist:

        if item.status == CheckStatus.FAIL:
            failed_items.append(item)
        elif item.status == CheckStatus.PENDING:
            pending_items.append(item)

        for item in failed_items:
            print(f"  [{item.id}] {item.description}")
    
    can_launch = len(failed_items) == 0 and len(pending_items) == 0

    return can_launch
```

## 监管合规：法律不是上限，是底线

在写治理框架的过程中，陈浩查了一遍国内外的AI监管法规。

**EU AI Act（2024年正式生效）**

欧盟《人工智能法案》将AI系统分为四个风险等级：不可接受风险（禁止）、高风险（严格监管）、有限风险（透明度要求）、最小风险（基本自由）。

招聘和就业管理类AI系统被明确列为**高风险系统**，要求：
- 建立风险管理系统
- 确保训练数据质量和代表性
- 保持技术文档和日志记录
- 人工监督机制
- 满足准确性、鲁棒性和网络安全要求

违规罚款最高可达全球年营业额的3%（违反一般义务）或6%（使用被禁止的AI实践）。

**中国AI监管体系**

中国目前的AI监管以场景为导向：
- 《互联网信息服务算法推荐管理规定》（2022）：要求算法系统不得利用算法实施不合理的差别待遇
- 《生成式人工智能服务管理暂行办法》（2023）：要求生成式AI不得生成歧视性内容
- 《人工智能法》草案：目前征求意见中，预计将对高风险AI系统建立完整监管框架

陈浩在提案里写了一段话：

> "我们目前的招聘AI系统，从EU AI Act的标准来看，我们有义务完成的事情，一项都没做。这不只是合规风险，如果有一天我们进入欧洲市场，或者国内监管收严，这些都是定时炸弹。但更重要的是——这些要求存在是有道理的。"

## 📖 本章名词解释（新人必读）

> 第一次看到这些词？别慌，下面一句话搞定。

**🤖 AI 相关**

| 术语 | 一句话解释 |
| --- | --- |
| **特征** | 用来做判断的线索，比如简历里的工作年限、技能关键词 |
| **特征工程** | 把原始信息变成模型能看懂的数值，类似把"4年Python"变成数字4 |
| **分类模型** | 把东西分到不同类别的AI，比如判断"合格/不合格" |
| **预测概率/得分** | AI给出的评分或信心值，比如认为某人有79%概率合格 |
| **特征权重** | 每个线索对最终结果的影响大小，权重越大越重要 |
| **训练集** | 用来教模型"正确模式"的过往数据，相当于给AI看的例题集 |
| **测试集** | 用来检验模型学得好不好的新数据，类似期末考试题 |
| **目标变量** | 模型想预测的那个答案，比如"是否被录用" |
| **敏感属性** | 法律或伦理上不应作为决策依据的特征，如种族、性别、宗教 |
| **反事实测试** | 只改一个条件（如名字），看结论变不变，查AI是否戴着"有色眼镜" |

**📊 公平性度量**

| 术语 | 一句话解释 |
| --- | --- |
| **人口均等** | 不同群体被AI"选中"的比例应该差不多 |
| **机会均等** | 真正有能力的候选人，不管什么背景，被选中的机会应该一样 |
| **个体公平** | 两个条件相似的人应该得到相似的评分，不能区别对待 |
| **80%规则** | 美国法规：弱势群体的"通过率"不能低于优势群体的80% |

**🔍 偏见检测技术**

| 术语 | 一句话解释 |
| --- | --- |
| **结构性歧视** | 系统性地对某一群体不公平，不是某个人坏，是制度本身有问题 |
| **反馈循环** | AI的偏见会自我强化——越歧视某群体，数据越少，下次歧视更严重 |
| **偏见来源** | 偏见是从哪里来的，比如历史数据里本身就存在的歧视 |
| **放大机制** | 技术如何把微小的偏见变成巨大的不公平，类似扩音器 |
| **危害分析** | 评估这个偏见对个人和社会会造成什么实际伤害 |
| **敏感特征检测** | 检查模型是否使用了不该用的信息（比如名字）来下判断 |
| **偏见量化** | 用数字衡量偏见有多大，比如差了多少分 |

**💻 编程与工具**

| 术语 | 一句话解释 |
| --- | --- |
| **DataFrame** | 像Excel表格一样的数据结构，Python数据分析最常用的格式 |
| **类型注解** | 代码里标注变量类型（如`: str`），让代码更好懂、更安全 |
| **混淆矩阵** | 一张小表，展示AI对了多少次、错了多少次，分出四类结果 |
| **sklearn** | Python最流行的机器学习工具库，封装了常用算法 |
| **特征提取器** | 一段程序，负责把原始数据转成模型能用的"线索列表" |

**⚖️ 法律与伦理**

| 术语 | 一句话解释 |
| --- | --- |
| **EEOC** | 美国公平就业机会委员会，管就业歧视的官方机构 |
| **结构性不平等** | 社会制度本身造成的长期不公平，不是某个人的错 |
| **因果 vs 相关** | 相关是"一起发生"，因果是"因为A所以B"，AI常搞混这两者 |

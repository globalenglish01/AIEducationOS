陈浩沉默了三秒。这三个问题，每一个都是真实的工程问题，每一个都会在某天变成紧急线上问题。

"有标准答案了，叫A2A协议。"

## 为什么需要一个标准

在A2A出现之前，Agent系统的跨框架通信是一片混战。

每家公司各显神通：
- 有的用REST API——Agent A 调用 Agent B 的HTTP接口，自定义请求格式
- 有的用消息队列——Agent A 发Kafka消息，Agent B 监听消费
- 有的用共享数据库——Agent A 往数据库写任务，Agent B 轮询读取
- 有的真的就用Python import——只要都是Python，一把梭

这些方案都能跑，但都有同一个问题：没有标准。

没有标准意味着：Agent A 不知道 Agent B 能处理什么类型的任务；Agent A 不知道 Agent B 的任务进度（完成了？失败了？还在处理？）；Agent A 发出去的任务格式，Agent B 可能根本不认识。

2025年初，Google联合50多家公司发布了A2A协议（Agent-to-Agent Protocol），试图解决这个问题。

A2A的目标是：**让任何Agent都能和任何其他Agent通信，不管背后用的是什么框架、什么语言、部署在哪里**。

---

## A2A vs MCP：定位不同

很多人看到A2A，第一个问题是："这和MCP有什么区别？"

上一章讲过MCP（Model Context Protocol），陈浩用MCP把"查询报销记录"这个功能暴露成了一个工具服务。

齐明用了一个类比：

**MCP是工具箱说明书**——描述这个工具能做什么、参数是什么、怎么调用。
**A2A是外包合同**——你把一整个任务交给另一个Agent去做，它用什么工具、怎么做是它的事，你只关心结果。

| 协议 | 适用对象 | 粒度 | 类比 |
|------|---------|------|------|
| MCP  | 工具（函数级） | 单次调用，确定性输出 | REST API |
| A2A  | Agent（任务级） | 多步执行，过程不透明 | 给同事布置任务 |

当你说"调用查询报销记录API"——这是MCP的场景。
当你说"帮我分析这个报销申请是否合规，给出审批建议"——这是A2A的场景。

区别在于：MCP调用的是一个确定性函数，你知道输入输出。A2A委托的是一个Agent去完成任务，中间经历多少步、用了什么工具，都是那个Agent自己决定的。

## A2A的核心概念

### AgentCard：能力声明

每个A2A Agent都需要有一张"名片"——AgentCard。AgentCard描述了这个Agent是谁、能做什么、怎么联系它。它通过标准的URL暴露：

```
GET https://expense.internal.company.com/a2a/.well-known/agent.json
```

任何想调用这个Agent的客户端，先来这个URL取AgentCard，就知道能不能用它、怎么用它。

### Task：任务单元

A2A的基本工作单元是Task。当客户端把任务委托给Agent时，创建一个Task对象，有唯一ID。

Task状态机：

```
submitted → working → (input-required) → working → completed
                                                  → failed
                                                  → canceled
```

- `submitted`：任务已提交，等待Agent开始处理
- `working`：Agent正在处理中
- `input-required`：Agent需要更多信息（Human-in-the-Loop场景）
- `completed`：任务成功完成
- `failed`：任务执行失败

### Message和Part

Agent之间通信用Message，Message包含一个或多个Part：TextPart（纯文本）、FilePart（文件）、DataPart（结构化JSON）。

`skills`里的`examples`不只是给人看的，也会被其他AI用来决定"要不要把这个任务交给它"。描述写得越精确，命中率越高。

### 实现AgentExecutor

AgentExecutor是A2A Server的执行核心——接收任务、调用现有Agent、推送结果：

```python
# expense_a2a/executor.py
import json
from a2a.server.agent_execution import AgentExecutor, RequestContext
from a2a.server.events import EventQueue
from a2a.utils import new_agent_text_message_event, new_task_artifact_event

from ..expense_agent import ExpenseApprovalAgent  # 已有的LangGraph Agent


class ExpenseAgentExecutor(AgentExecutor):
    def __init__(self):
        self.agent = ExpenseApprovalAgent()

    async def execute(
        self,
        context: RequestContext,
        event_queue: EventQueue,
    ) -> None:
        user_message = self._extract_user_message(context)
        if not user_message:
            await event_queue.enqueue_event(
                new_agent_text_message_event("请告诉我您想查询什么报销信息。")
            )
            return

        user_id = (
            context.current_task.metadata.get("user_id")
            if context.current_task.metadata
            else None
        )

        try:
            # 流式调用现有的LangGraph Agent
            async for chunk in self.agent.stream_run(query=user_message, user_id=user_id):
                if chunk["type"] == "token":
                    await event_queue.enqueue_event(
                        new_agent_text_message_event(chunk["content"])
                    )
                elif chunk["type"] == "tool_result":
                    await event_queue.enqueue_event(
                        new_task_artifact_event(
                            name="查询结果",
                            data=json.dumps(chunk["data"], ensure_ascii=False),
                            mime_type="application/json",
                        )
                    )
        except Exception as e:
            await event_queue.enqueue_event(
                new_agent_text_message_event(
                    f"报销查询暂时不可用，请稍后重试。（错误：{type(e).__name__}）"
                )
            )

    def _extract_user_message(self, context: RequestContext) -> str | None:
        if not context.message:
            return None
        for part in context.message.parts:
            if hasattr(part, "text"):
                return part.text
        return None

    async def cancel(self, context: RequestContext, event_queue: EventQueue) -> None:
        pass
```

### 搭建A2A Server

```python
# expense_a2a/server.py
import uvicorn
from a2a.server.apps import A2AStarlette
from a2a.server.request_handlers import DefaultRequestHandler
from a2a.server.tasks import InMemoryTaskStore

from .agent_card import get_expense_agent_card
from .executor import ExpenseAgentExecutor


def create_app(base_url: str = "http://localhost:8100"):
    agent_card = get_expense_agent_card(base_url)
    task_store = InMemoryTaskStore()  # 生产用RedisTaskStore

    request_handler = DefaultRequestHandler(
        agent_executor=ExpenseAgentExecutor(),
        task_store=task_store,
    )

    return A2AStarlette(agent_card=agent_card, http_handler=request_handler)


if __name__ == "__main__":
    uvicorn.run(create_app(), host="0.0.0.0", port=8100)
```

启动后自动暴露：`GET /.well-known/agent.json`、`POST /a2a`、`GET /a2a/{task_id}`。

## 实战：客服Agent通过A2A调用报销Agent

### A2A Client封装

```python
# customer_service/expense_delegate.py
import httpx
from a2a.client import A2AClient
from a2a.types import Message, TextPart


class ExpenseAgentDelegate:
    EXPENSE_AGENT_URL = "http://expense-agent.internal:8100"

    def __init__(self):
        self._http_client = httpx.AsyncClient(timeout=60.0)
        self._a2a_client = A2AClient(
            httpx_client=self._http_client,
            url=self.EXPENSE_AGENT_URL,
        )

    async def query_expense(self, user_query: str, user_id: str | None = None) -> str:
        """把报销查询委托给专业Agent，返回最终文本答案。"""
        message = Message(role="user", parts=[TextPart(text=user_query)])
        metadata = {"user_id": user_id} if user_id else None
        result_chunks: list[str] = []

        async with self._a2a_client.send_message_streaming(
            message=message, metadata=metadata
        ) as stream:
            async for event in stream:
                if hasattr(event, "parts"):
                    for part in event.parts:
                        if hasattr(part, "text"):
                            result_chunks.append(part.text)

        return "".join(result_chunks)

    async def close(self):
        await self._http_client.aclose()
```

## Task状态机工程细节

### 超时处理

```python
import asyncio
from a2a.types import TaskState


async def wait_for_task(
    client: A2AClient,
    task_id: str,
    timeout_seconds: float = 30.0,
    poll_interval: float = 0.5,
) -> str:
    deadline = asyncio.get_event_loop().time() + timeout_seconds

    while asyncio.get_event_loop().time() < deadline:
        task = await client.get_task(task_id)

        if task.status.state == TaskState.completed:
            for artifact in (task.artifacts or []):
                for part in artifact.parts:
                    if hasattr(part, "text"):
                        return part.text
            return ""

        elif task.status.state in (TaskState.failed, TaskState.canceled):
            error_msg = (
                task.status.message.parts[0].text
                if task.status.message
                else "未知错误"
            )
            raise RuntimeError(f"报销Agent任务失败：{error_msg}")

        await asyncio.sleep(poll_interval)

    await client.cancel_task(task_id)
    raise TimeoutError(f"报销Agent超过{timeout_seconds}秒未响应")
```

### 幂等性

网络超时时，客户端不知道任务是否已提交成功。用自定义Task ID实现幂等：

```python
import hashlib


def generate_idempotent_task_id(user_id: str, query: str, timestamp: str) -> str:
    content = f"{user_id}:{query}:{timestamp}"
    return hashlib.sha256(content.encode()).hexdigest()[:32]
```

### Bearer Token认证

跨VPC或有审计要求时：

```python
from starlette.middleware.base import BaseHTTPMiddleware
from starlette.requests import Request
from starlette.responses import JSONResponse


class TokenAuthMiddleware(BaseHTTPMiddleware):
    VALID_TOKENS = {"client-agent-secret-token-xxx"}

    async def dispatch(self, request: Request, call_next):
        if request.url.path == "/.well-known/agent.json":
            return await call_next(request)  # AgentCard不需要认证

        auth = request.headers.get("Authorization", "")
        if not auth.startswith("Bearer ") or auth[7:] not in self.VALID_TOKENS:
            return JSONResponse({"error": "Unauthorized"}, status_code=401)

        return await call_next(request)
```

## 生产TaskStore的选择

`InMemoryTaskStore`适合开发，生产环境用Redis：

```python
import redis.asyncio as aioredis
from a2a.server.tasks import RedisTaskStore

redis_client = aioredis.from_url("redis://redis-host:6379/0")
task_store = RedisTaskStore(
    redis=redis_client,
    key_prefix="expense_agent:tasks:",
    ttl_seconds=86400,
)
```

需要持久化（Task不能丢）时，自实现PostgreSQL TaskStore：

```python
class PostgresTaskStore:
    def __init__(self, db_session):
        self.db = db_session

    async def save(self, task: Task) -> None:
        await self.db.execute(
            "INSERT INTO a2a_tasks (id, state, data, updated_at) "
            "VALUES ($1, $2, $3, now()) "
            "ON CONFLICT (id) DO UPDATE SET state=$2, data=$3, updated_at=now()",
            task.id, task.status.state.value, task.model_dump_json(),
        )

    async def get(self, task_id: str) -> Task | None:
        row = await self.db.fetchrow(
            "SELECT data FROM a2a_tasks WHERE id=$1", task_id
        )
        return Task.model_validate_json(row["data"]) if row else None
```

## 三种方式的取舍

| 维度 | 直接import | REST API（自研） | A2A协议 |
|------|-----------|----------------|---------|
| 实现复杂度 | 最简单 | 中等 | 中等 |
| 跨语言支持 | 否 | 是 | 是 |
| 跨进程/服务器 | 否 | 是 | 是 |
| 任务状态追踪 | 需自己加 | 需自己加 | 内置 |
| Agent能力发现 | 否 | 否 | AgentCard自动 |
| 流式输出 | 是 | 需自己实现 | 内置 |
| 生态成熟度 | N/A | 成熟 | 2025年，演进中 |
| 适用场景 | 同项目同进程 | 已有服务改造 | 新建Agent系统 |

架构导师看了这张表说："A2A是正确的方向，但你要接受它目前的生态还在演进。新的Agent系统用A2A，现有的不动。等半年，看看生态成熟了没有，再决定要不要全面迁移。"

这个"不追求完美、接受渐进演进"的策略，陈浩后来发现在他的整个工程师生涯里用了很多次。

## 📖 本章名词解释（新人必读）

> 第一次看到这些词？别慌，下面一句话搞定。

**🤖 AI 相关**

| 术语 | 一句话解释 |
| --- | --- |
| **A2A协议 (Agent-to-Agent)** | Agent之间通信的统一标准，好比不同公司之间的业务对接规范，让各方能互相协作。 |
| **AgentCard** | Agent的“名片”，描述它叫什么、会什么、怎么联系，好比个人简历。 |
| **Skill** | AgentCard中列出的“技能项”，描述Agent能处理哪些任务，好比简历里写的“擅长Java开发”。 |
| **Task（任务单元）** | A2A中的基本工作单元，代表一个待完成的委托任务，好比派给同事的一个工单。 |
| **任务状态机** | 任务从提交到完成的阶段流转：提交→处理中→需补充信息→完成/失败/取消，好比快递从揽收到签收的过程。 |
| **Human-in-the-Loop** | 任务处理中需要人工介入的场景，好比AI审核后还需领导签字确认。 |
| **流式 (Streaming)** | 结果持续推送而非一次性返回，好比文章正在逐字生成而不是全部印好后一次性端来。 |
| **轻量化** | 实现简单、资源消耗少的设计风格，好比轻量级框架 vs 重型框架。 |
| **MCP (Model Context Protocol)** | 工具级通信协议，让AI能调用各种工具，好比工具箱说明书。 |

**💻 软件工程相关**

| 术语 | 一句话解释 |
| --- | --- |
| **跨框架通信** | 不同框架构建的系统之间互相通信，好比苹果和安卓手机互传文件。 |
| **互操作性** | 不同系统能互相协作的能力，好比不同品牌的电器能用同一个插头。 |
| **拦截器模式** | 在请求处理前后插入额外逻辑的设计模式，好比快递收发室在包裹进出时统一消毒。 |
| **委托模式** | 把任务交给另一个对象去完成的设计模式，好比老板把任务派给下属。 |
| **服务发现** | 系统运行时动态找到目标服务地址，好比打电话前先查114问号码。 |
| **状态存储** | 持久化保存任务状态的存储系统，好比工单系统里的数据库。 |

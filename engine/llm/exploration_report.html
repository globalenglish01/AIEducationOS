2. 如何调用大模型

2.1 标准调用示例

from engine.llm import create_llm # 创建LLM实例 llm = create_llm("deepseek", account="1") # 发送消息 answer = llm.chat([{"role": "user", "content": "你好"}]) print(answer) # 关闭资源 llm.close()

2.2 create_llm 参数

参数可选值说明
provider"chatgpt"/"deepseek"/"both"LLM提供商
account"1" ~ "6"账号编号

2.3 BaseLLM 接口

核心方法：
  • chat(messages: list[dict]) → str - 同步调用，返回完整文本
  • chat_response(messages) → LLMResponse - 返回结构化对象
  • stream_chat(messages) → Generator[str] - 流式输出
  • async_chat(messages) - 异步调用
  • health_check() → bool - 检查浏览器状态
  • close() - 释放资源

4. 关键概念

4.1 Pipeline（流水线）

每本书包含多个 Stage（阶段），每个Stage是一轮独立的LLM处理：

Stage名称提供商职责
S1施工图DeepSeek规划章节结构
S2草稿前半DeepSeek写前半部分
S3草稿后半DeepSeek写后半部分
S4技术审稿DeepSeek检查准确性
S5修订ChatGPT改进表达
S6终审ChatGPT最终把关

4.2 提示词模板

位置：books/{书名}/prompts/s1.md ~ s6.md

支持占位符（会自动注入变量）：

{{chapter}} - 章号 {{chapter_title}} - 章标题 {{book_title}} - 书名 {{protagonist}} - 主角 {{memory}} - 前几章记忆 {{prev_stage_N}} - 前序Stage内容 {{book_spec}} - 写作规范 {{任意字段}} - config.json中的字段

4.3 记忆系统

每章完成后，系统用LLM提取关键信息，注入到后续章节的Prompt中：

  • 主角状态变化
  • 关键术语定义
  • 重要案例
  • 导师金句
  • 技术连续性提示

4.4 进度追踪

通过 progress.json 精确记录每章每Stage的完成状态，支持断点续跑。
